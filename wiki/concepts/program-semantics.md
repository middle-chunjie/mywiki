## Definition

**Program Semantics** (程序语义) — the meaning of a program in terms of its intended function, constraints, state transitions, and observable input/output behavior.

## Key Points

- [[ding-2024-semcoder-2406-01006]] decomposes program semantics into approximate, symbolic, abstract, and operational views rather than treating semantics as only final execution output.
- The paper uses high-level task descriptions, key properties, and execution narratives to connect static code tokens with dynamic runtime behavior.
- Abstract semantics capture invariants and constraints that hold across executions, while operational semantics describe concrete state changes for specific inputs.
- The authors argue that debugging and repair require richer semantic understanding than plain next-token code generation.
