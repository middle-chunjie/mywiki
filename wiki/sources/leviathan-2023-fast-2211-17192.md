## Summary

⚠ Possibly outdated: published 2023; re-verify against recent literature.

The paper introduces speculative decoding, an exact acceleration method for autoregressive Transformer inference that uses a cheaper approximation model `M_q` to draft `γ` tokens and a larger target model `M_p` to verify those drafts in parallel. Its key contribution is speculative sampling, which preserves the target model's original output distribution by accepting draft tokens with probability `min(1, p(x)/q(x))` and repairing the first rejected token with a residual distribution. The analysis links speedup to the draft-target agreement rate `α = E(min(p, q))` and draft cost ratio `c`. Empirically, the method speeds up T5-XXL decoding by roughly `2x-3x` without retraining, architecture changes, or output drift, making it a practical systems technique for memory-bandwidth-bound LLM serving.

## Problem & Motivation

Autoregressive decoding from large Transformers is inherently serial: generating `K` tokens requires `K` sequential target-model evaluations. As model size grows, decode latency becomes dominated less by raw arithmetic and more by reading model weights, KV cache, and communication-heavy execution, so simply adding more parallel arithmetic is often possible but underused. Prior acceleration approaches such as distillation, sparse/efficient architectures, or adaptive-computation methods usually require retraining, architecture changes, or heuristic shortcuts that alter outputs. This paper asks whether one can exploit easier next-token decisions with a smaller model while still sampling exactly from the large model's distribution. The answer is yes: use a cheap draft model to speculate on several future tokens, then verify them in one parallel target pass and only repair the first mismatch.

## Method

- **Setup**: define the slow target model `M_p` with token distribution `p(x_t | x_{<t})` and a faster approximation model `M_q` with distribution `q(x_t | x_{<t})`. The draft model autoregressively proposes `γ` guesses `x_1, ..., x_γ`.
- **Standardized sampling view**: argmax, top-`k`, nucleus sampling, and temperature scaling are all treated as standard sampling from adjusted token distributions, so the algorithm operates on normalized `p(x)` and `q(x)` after any decoding transform.
- **Speculative sampling**: sample `x ~ q(x)`, accept it if `q(x) <= p(x)`, otherwise reject with probability `1 - p(x)/q(x)`. When rejected, resample from the residual distribution ``p'(x) = norm(max(0, p(x) - q(x)))``. This preserves exact sampling from `p`.
- **Parallel verification step**: after drafting `γ` tokens with `M_q`, run `M_p` in parallel on `prefix`, `prefix + [x_1]`, ..., `prefix + [x_1, ..., x_γ]` to obtain `p_1(x), ..., p_{γ+1}(x)` in one serial round of the target model.
- **Acceptance rule**: draw `r_i ~ U(0, 1)` and accept the longest prefix whose guessed token satisfies ``r_i <= p_i(x_i) / q_i(x_i)``. If the first rejection occurs at position `n + 1`, keep `x_1, ..., x_n` and sample the correction token from ``p'(x) = norm(max(0, p_{n+1}(x) - q_{n+1}(x)))``; if all `γ` guesses are accepted, sample one extra token from `p_{γ+1}(x)`.
- **Generated-token analysis**: if per-step acceptance rates are approximated as i.i.d. with mean `α = E(β)`, then one speculative step emits a capped geometric number of tokens with expectation ``E[#tokens] = (1 - α^(γ + 1)) / (1 - α)``.
- **Agreement metric**: the paper defines ``D_LK(p, q) = 1 - Σ_x min(p(x), q(x))`` and shows ``β = 1 - D_LK(p, q)``, hence ``α = E(min(p, q))``. This gives a concrete target-draft similarity statistic for choosing `M_q`.
- **Walltime model**: with draft/target step-time ratio `c`, expected speedup is ``(1 - α^(γ + 1)) / ((1 - α)(γc + 1))``. Improvement exists whenever the draft model is cheap enough relative to its acceptance rate, e.g. `α > c` is sufficient.
- **Operation trade-off**: speculative decoding raises concurrent arithmetic by up to `γ + 1` target evaluations and can increase total operations by ``((1 - α)(γ\hat{c} + γ + 1)) / (1 - α^(γ + 1))``, but can still reduce weight/KV-cache memory reads per output token.
- **Practical approximation models**: the paper tests much smaller off-the-shelf Transformers, but also discusses negligible-cost drafts such as unigram/bigram models, copy heuristics, and even non-autoregressive models as valid `M_q` choices.

## Key Results

- T5-XXL (`11B`) translation on EnDe reaches `3.4x` speedup at `temp = 0` with T5-small draft (`γ = 7`, `α = 0.75`) and `2.6x` at `temp = 1` (`α = 0.62`), with identical outputs to baseline T5X decoding.
- T5-XXL summarization on CNN/DailyMail reaches `3.1x` speedup at `temp = 0` with T5-small (`γ = 5`, `α = 0.65`) and `2.3x` at `temp = 1` (`α = 0.53`).
- Larger draft models improve agreement but not always latency: on EnDe with `temp = 0`, T5-base gets `α = 0.80` and `2.8x`, while T5-large gets `α = 0.82` but only `1.7x` because the draft cost `c` is higher.
- For GPT-like language modeling, a `6M` draft model paired with a `97M` target reaches `α = 0.88` (`t = 0`) and `α = 0.89` (`t = 1`), showing that exact speculative decoding can work well even with small decoder-only drafts.
- Even trivial drafts help: for T5-XXL EnDe, a bigram draft reaches `α ≈ 0.20`, which the paper notes is enough for about `1.25x` speedup with `γ = 3` when `c = 0`.
- Theoretical examples show the systems trade-off clearly: at `α = 0.9, γ = 10, c = 0`, the predicted speed is `6.86x` while total arithmetic grows to `1.60x` baseline.

## Limitations

- The method improves latency by trading for more concurrency; if spare compute is unavailable, speculative decoding may not help and can increase total arithmetic.
- Quality of speedup depends heavily on the approximation model through `α` and `c`; a more accurate but slower draft can reduce net gains.
- The empirical implementation is only shown on text tasks and mainly on T5-XXL walltime; broader deployment evidence for other architectures is limited.
- Beam-search support is only discussed in the appendix and incurs extra parallel budget `(w + uγ)`; the paper does not provide a full experimental evaluation there.
- The analysis uses simplifying assumptions such as i.i.d. acceptance rates across positions, so closed-form predictions are approximate rather than exact system models.
