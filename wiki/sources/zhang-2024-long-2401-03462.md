## Summary

Activation Beacon is a plug-in module for extending LLM context windows without modifying the base model's parameters. It introduces special beacon tokens (`<bcn>`) that condense raw KV activations from a context interval into compact representations, which are then accumulated in a sliding window for autoregressive generation. Trained in only 10K steps (~9 hours on 8×A800) using short sequences up to 8K with randomly sampled condensing ratios, Activation Beacon extends Llama-2-7B from 4K to 400K context length. It achieves competitive perplexity against fine-tuned full-attention baselines on PG19, Proof-Pile, and CodeParrot, and comparable performance on LongBench tasks, while maintaining linear time complexity and near-constant GPU memory (~21 GB) across all tested context lengths up to 100K.

## Problem & Motivation

LLMs are constrained by fixed context windows (e.g., 4K for Llama-2). Fine-tuning on long sequences extends the window but incurs quadratic compute cost (due to full self-attention), requires expensive long-sequence training data, and often degrades the LLM's short-context capabilities. Existing alternatives — position encoding tricks (PI, NTK, YaRN), sliding windows (StreamingLLM), and context compression (AutoCompressor, Gist tokens) — each suffer from limitations: context loss beyond the window, inability to generalize to diverse lengths, or incompatibility with existing pretrained models. The paper seeks a method that is simultaneously effective, efficient, compatible with the original LLM, and cheap to train.

## Method

**Architecture — Beacon Token Mechanism:**
- Special tokens `<bcn>` are appended to each context interval of length `l`. For interval length `l` and `k` beacon tokens, the condensing ratio is `α = l/k`.
- A separate copy of the LLM's multi-head attention (MHA) parameters — `W^b_Q, W^b_K, W^b_V, W^b_O` — is introduced as the only learnable module (~2B parameters for Llama-2-7B, i.e., less than 1/3 of the base model size). All original LLM parameters are frozen.
- Beacon tokens attend to both their own activations and all raw activations of normal tokens in the interval. The condensed output is: `O^b ← W^b_O · A · (V^r ⊕ V^b)` where `A = softmax(mask(Q^b (K^r ⊕ K^b)^T / √D))`.

**Attention Scheme — Stepwise Expansion (best variant):**
- Three schemes tested: Segmentation, Full Coverage, Stepwise Expansion. Stepwise expansion is optimal: beacon `i` can attend to spans `1..i`, so the last beacon covers the full interval while earlier ones capture progressively narrower local views. This provides hierarchical local-to-global coverage within each interval.

**Stream Processing (Sliding Window):**
- Long context is split into intervals of length `l = 1024`. The sliding window contains: (a) accumulated condensed activations from all previous intervals (beacon tokens), and (b) raw activations from the current interval.
- Raw activations from past intervals are discarded; condensed activations are kept. Window size is bounded by the LLM's original context limit (4K for Llama-2), giving `O(l)` memory per interval and linear overall time complexity.
- The LLM predicts: `p(x_n | <bcn>_1, …, <bcn>_m, x_{m+1}, …, x_{n-1}; Θ, Θ^b)`.

**Training — Step-wise Randomized Condensing Ratio:**
- Auto-regression loss on next-token prediction is minimized over frozen LLM + trainable beacon parameters.
- Training data: 80K samples from [[redpajama]] (70K) + [[longalpaca]] (10K), all between 1024–8192 tokens.
- Condensing ratio `α_j` is randomly sampled per interval from `{2, 4, 8, 16, 32, 64, 128}`, introducing diversity that enables generalization to arbitrary context lengths at inference.
- `10,000` steps, batch size `8`, learning rate `5e-5` with linear scheduler, 8×A800 GPUs, ~9 hours.

**Compatibility with Other Methods:**
- Since Activation Beacon does not modify positional encoding, it can be combined with PI or NTK to further increase the effective window size, pushing context to ~1M tokens.
- BM25-based retrieval over condensed activation intervals dramatically improves fine-grained memory accuracy (100% passkey recall at all lengths).

## Key Results

- **Language Modeling (Perplexity):** On PG19 at 32K context, Activation Beacon achieves PPL=8.27, outperforming fine-tuning-free baselines (NTK: 37.8) and comparable to LongChat-32K (8.81), using only short-sequence training data.
- **100K context:** Activation Beacon maintains PPL=8.50 (PG19), 3.31 (Proof-Pile), 2.62 (CodeParrot) at 100K tokens; all full-attention baselines are OOM.
- **400K context:** With condensing ratio α=128, context extends to 400K (100× Llama-2-7B's original 4K), shown via Figure 1 perplexity curves.
- **LongBench:** Activation Beacon scores 28.27 Single-Doc QA, 28.44 Multi-Doc QA, 25.15 Summarization, 61.00 Few-Shot, 57.75 Code — on par with LongAlpaca-16K (28.70/28.10/27.80/63.70/56.00).
- **GPU Memory:** ~21.6 GB at 100K context vs. LongChat OOM at 32K; memory nearly flat across all lengths.
- **Training Efficiency:** 9 hours on 8×A800 vs. LongAlpaca-16K's 20.8 hours on 8×A100.
- **Ablations:** Stepwise expansion > Segmentation > Full Coverage; Step-wise randomized ratio > Instance-wise > Monotonous (α=4, which fails at >16K); Q,K,V,O (2B) is the optimal parameter budget.

## Limitations

- **Stride equals interval length:** The sliding window advances by exactly one interval, so tokens at the beginning of each new interval have no raw-activation context — they attend only to condensed summaries from prior intervals, potentially degrading generation quality at interval boundaries.
- **Memory-accuracy trade-off:** Higher condensing ratios (needed for very long contexts) cause vague representations; Activation Beacon cannot reliably recall fine-grained details (e.g., passkeys) beyond ~8K without retrieval augmentation.
- **FlashAttention-2 incompatibility:** The custom attention masking scheme precludes use of FlashAttention-2, limiting inference speed relative to full-attention models at short contexts.
- **Single base model evaluated:** All experiments use Llama-2-7B (chat); generalization to other architectures (GQA, newer positional encodings) is not demonstrated.
- **Short-context overhead:** At short contexts (4K), Activation Beacon is slower than full-attention (LongChat) because of the sequential stream-processing design.
