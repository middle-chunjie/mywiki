## Summary

MemAgent proposes an RL-trained memory agent that turns a standard dense-attention LLM into a linear-cost long-context reasoner without changing the underlying decoder architecture. Instead of attending over the full document, the model reads a long text chunk by chunk, repeatedly overwriting a fixed-length token memory and answering only after the stream ends. Training is done end-to-end with Multi-Conv DAPO, which propagates a final answer reward back across all memory-update conversations in the trajectory. On synthetic HotpotQA-based RULER evaluation, models trained on `32K` documents with only an `8K` working window extrapolate to `3.5M`-token QA with limited degradation, and the `14B` system stays around `76-84%` accuracy from `7K` through `3.5M` context lengths.

## Problem & Motivation

The paper targets the long-context trilemma for dense-attention LLMs: standard extrapolation methods still incur `O(n^2)` attention cost, sparse or linear-attention variants often require architectural changes or retraining from scratch, and external compression/memory mechanisms can disrupt the native generation pipeline. The authors want a method that can process effectively unbounded documents, retain task-critical evidence during extrapolation, and keep inference cost linear in document length. Their core intuition is to mimic human note-taking: store only answer-relevant information in a compact textual memory and discard irrelevant detail.

## Method

- **Inference workflow**: split a document into chunks of length `C = 5000` and maintain a fixed textual memory of `M = 1024` tokens; the model iteratively maps `(problem, previous memory, current chunk)` to a new memory, then performs a final answer-only generation step from `(problem, final memory)`.
- **Prompt design**: the context-processing prompt explicitly asks the model to "update the memory" after each section, while the final prompt asks it to answer from memory and place the result in `` `\boxed{}` ``.
- **Working context budget**: training intentionally uses an `8K` window allocated roughly as query `1024` + chunk `5000` + memory `1024` + output `1024`, with the remainder reserved for the chat template; typical trajectories require `5-7` conversations per sample.
- **RL objective**: starting from GRPO with group size `G = 16`, the method treats each context-independent conversation in a rollout as a separate optimization target and distributes a sample-level final-answer reward across all of that sample's memory-update turns.
- **Multi-Conv DAPO**: for sample `i` with conversations `o_{i,1}, ..., o_{i,n_i}`, the paper uses `\hat{A}_{i,j,t} = r_i - mean({R_i}_{i=1}^G)` and a token-averaged clipped DAPO loss over the full `(group, conversation, token)` tensor instead of a single-turn `(group, token)` layout.
- **Reward modeling**: exact-answer tasks use `R(\hat{y}, Y) = max_{y in Y} I(is_equiv(y, \hat{y}))`; multi-value retrieval tasks use `R(\hat{y}, Y) = |{y in Y : y in \hat{y}}| / |Y|`.
- **Optimization details**: AdamW, learning rate `1e-6`, constant schedule with linear warm-up, KL coefficient `1e-3`, entropy loss disabled, rollout batch size `128` for `7B` and `256` for `14B`, and sample-to-backprop batch ratio `16`.
- **Autoregressive reinterpretation**: the model introduces latent memories `m^1:K-1` so that `p(x_{1:N}) = sum_{m^{1:K-1}} prod_k p(c^k | m^{k-1}) p(m^k | c^k, m^{k-1})`, giving per-step cost `O(C + M)` and overall linear complexity `O(N)` for streamed inputs.

## Key Results

- On the main HotpotQA-synthesized RULER-QA benchmark, `RL-MemAgent-14B` scores `83.59` at `7K`, `84.38` at `28K`, `77.34` at `896K`, `76.56` at `1.75M`, and `78.12` at `3.5M`.
- `RL-MemAgent-7B` also extrapolates strongly: `82.03` at `7K`, `79.69` at `112K`, `76.56` at `896K`, `75.78` at `1.75M`, and `71.09` at `3.5M`.
- At `896K`, major long-context baselines collapse well before their nominal windows: `Qwen2.5-Instruct-14B-1M` and `Qwen2.5-Instruct-7B-1M` both reach `0.00`, while `QwenLong-L1-32B` falls to `11.72`.
- On out-of-distribution RULER tasks, the paper reports `95%+` average accuracy for MemAgent-14B up to `512K` context, with stable performance on NIAH, variable tracking, frequent-word extraction, and SQuAD-derived QA.
- The appendix-level FLOP analysis shows the baseline dense-attention setup grows as `O(n^2)`, whereas MemAgent scales as `O(n)` by fixing memory size and streaming the document.

## Limitations

- The empirical study is concentrated on long-document QA and synthetic RULER-style tasks; it does not show equally detailed evidence for open-ended generation, summarization, or interactive agent environments.
- Training and comparisons are limited to Qwen2.5 `7B` and `14B` backbones plus a small set of long-context baselines, so cross-architecture generality remains partly untested.
- The fixed `1024`-token memory is a hard information bottleneck; the paper shows it works well for the tested tasks, but does not characterize failure modes when too many answer-critical facts must be retained simultaneously.
- The method is still RL-dependent and uses multi-conversation rollouts, which may be expensive or unstable relative to simpler supervised long-context adaptation recipes.
- The work is an arXiv preprint and does not include a broad real-world downstream evaluation beyond the synthetic-to-semi-real QA setup built from HotpotQA and RULER.
