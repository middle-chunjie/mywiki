## Summary

⚠ Possibly outdated: published 2023; re-verify against recent literature.

He and Hofmann ask how much of the standard Pre-LN Transformer block is actually necessary for fast training. Starting from signal propagation arguments and validating them empirically, they remove the attention skip connection, then value and projection matrices, then the remaining MLP skip via a parallel block layout, and finally optionally normalization layers. The resulting SAS and SAS-P blocks match standard Pre-LN training dynamics on decoder-only CodeParrot and encoder-only BERT settings while reducing parameters and improving throughput. In their reported configurations, the simplified models preserve downstream GLUE performance, remove roughly 15-16% of parameters, and deliver about 9-16% faster training throughput, suggesting that standard Transformer blocks contain substantial architectural redundancy.

## Problem & Motivation

Standard Transformer blocks are built from tightly coupled attention and MLP sub-blocks, residual connections, normalization, and value/projection matrices whose exact arrangement is known to be brittle. The paper studies whether these components are all necessary for trainability, or whether a simpler block can preserve the per-update and wall-clock training speed of Pre-LN Transformers. The motivation is both theoretical and practical: simpler blocks make signal-propagation analyses closer to real architectures, and they also reduce parameter count and matrix multiplications in large language-model and BERT-style training pipelines.

## Method

- **Baseline formulation**: the Pre-LN block is written as `` `X_out = alpha_FF * X_hat + beta_FF * MLP(Norm(X_hat))` `` with `` `X_hat = alpha_SA * X_in + beta_SA * MHA(Norm(X_in))` ``, where default gains are `` `alpha_* = beta_* = 1` ``.
- **Attention simplification**: removing the attention skip sets `` `alpha_SA = 0` ``, which would normally cause rank-collapse-related optimization issues in deep skipless Transformers.
- **Shaped Attention**: they replace the attention matrix with `` `A(X) <- alpha * I_T + beta * A(X) - gamma * C` ``, initialize `` `alpha = beta = gamma = 1` ``, and set `` `W^Q = 0` `` at initialization so the identity component dominates.
- **Restricted value/projection updates**: they motivate the reparameterization `` `W^V = alpha_V * W_init^V + beta_V * Delta W^V` `` and `` `W^P = alpha_P * W_init^P + beta_P * Delta W^P` `` as an analogue of residual-branch downweighting; under AdamW this behaves like smaller effective learning rates for value/projection parameters.
- **Simplified Attention Sub-block (SAS)**: empirical trajectories show most value/projection parameters converge toward identity, so the final SAS design fixes all projection matrices and all value matrices except the first-layer value matrix to identity. This removes half the attention-sub-block parameters and matrix multiplications.
- **Parallel skipless block (SAS-P)**: to remove the remaining MLP skip, they adopt the parallel block `` `X_out = alpha_comb * X_in + beta_FF * MLP(Norm(X_in)) + beta_SA * MHA(Norm(X_in))` `` and set `` `alpha_comb = 0` `` with downweighted `` `beta_FF < 1` ``.
- **Decoder-only experimental setup**: CodeParrot experiments use an `` `L = 18` `` layer GPT-style causal model with `` `d = 768` ``, `` `H = 12` `` heads, `` `d_ff = 3072` ``, sequence length `` `T = 128` ``, vocabulary size `` `50K` ``, ReLU MLPs, no dropout, sinusoidal positional encodings, RMSNorm with `` `eps = 1e-8` `` when normalization is used, and trainable `` `beta_FF = 0.1` `` in simplified blocks.
- **Optimization details**: CodeParrot training uses AdamW with weight decay `` `0.1` ``, betas `` `(0.9, 0.999)` ``, epsilon `` `1e-8` ``, gradient clipping `` `1` ``, linear decay with `` `5%` `` warmup, batch size `` `128` `` with microbatch `` `32` ``, and tuned max learning rate `` `1e-3` `` for SAS / SAS-P.
- **Encoder-only setup**: BERT experiments use `` `16` `` layers, `` `d = 768` ``, `` `12` `` heads, GLU with GeLU, LayerNorm with `` `eps = 1e-12` ``, WordPiece vocabulary `` `32768` ``, sequence length `` `128` ``, masking probability `` `0.25` ``, and simplified blocks with smaller `` `beta_FF = 0.05` ``.

## Key Results

- Abstract-level headline: simplified Transformers emulate standard-model training dynamics while using about `15%` fewer parameters and achieving about `15%` faster training throughput.
- In the 18-layer CodeParrot setting, replacing trainable value/projection matrices with identity improves final evaluation loss from `1.194` to `1.178` and reduces total parameters from `167M` to `146M` (`13%` fewer).
- In the BERT/GLUE setup, Pre-LN scores `78.9 ± 0.7` GLUE with `120M` parameters at normalized speed `1.00`; SAS scores `78.4 ± 0.8` with `101M` parameters at speed `1.09`; SAS-P scores `78.3 ± 0.4` with `101M` parameters at speed `1.16`.
- The no-normalization SAS-P variant reaches the highest reported training-speed ratio, `1.20`, but is excluded from GLUE reporting because it is unstable for downstream fine-tuning.
- Longer-training experiments scale the decoder-only setup from about `700M` tokens (`~43K` steps) to about `2B` tokens (`~120K` steps) and show SAS / SAS-P continue to match or outperform Pre-LN training speed.
- Depth-scaling experiments increase model depth from `18` to `72` blocks and show SAS / SAS-P maintain near-indistinguishable per-update trajectories from Pre-LN, unlike the earlier V-SkipInit baseline.

## Limitations

- The largest models studied are only in the `100M-300M` parameter range, so the paper does not establish that the same simplifications remain optimal at frontier LLM scale.
- Several conclusions rely on fixed inherited training stacks; the authors explicitly note that a fuller hyperparameter search might further change the relative ranking of blocks.
- Removing normalization entirely improves wall-clock throughput but slightly hurts loss-per-step and causes fine-tuning instabilities, including NaNs for a small minority of downstream sequences.
- Empirical coverage is still narrow: decoder-only next-token prediction on CodeParrot and encoder-only masked language modeling / GLUE do not cover broader modalities or long-context regimes.
