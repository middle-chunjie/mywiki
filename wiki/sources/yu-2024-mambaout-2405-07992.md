## Summary

MambaOut asks whether the state-space-model (SSM) token mixer that defines Mamba is actually needed for mainstream vision workloads. Yu and Wang argue that Mamba is most appropriate for tasks that are both long-sequence and autoregressive, while standard visual recognition tasks often satisfy neither condition. They derive a simple Transformer-based sequence-length criterion, then remove SSM from Mamba-style blocks and build a family of Gated CNN backbones called MambaOut. The resulting models outperform visual Mamba baselines on ImageNet classification, but fall behind the strongest visual Mamba systems on COCO detection and ADE20K segmentation. The paper is therefore best read as a negative-result-style baseline paper: SSM appears unnecessary for standard image classification, but may still help dense visual tasks with longer token sequences.

## Problem & Motivation

The paper studies a narrow but important question in visual backbone design: whether Mamba's SSM-based recurrent token mixer provides real benefits for vision, or whether recent visual Mamba results mostly come from the surrounding architecture rather than SSM itself. The motivation is that Mamba was originally attractive because it offers linear-time sequence processing and causal recurrent memory, properties that matter in long-sequence autoregressive settings. The authors argue that common visual recognition tasks are understanding problems with fully visible inputs, not causal generation problems, and that standard ImageNet classification also uses relatively short token sequences. If that reasoning is correct, then injecting SSM into visual classifiers may add complexity without adding the task properties that make Mamba valuable.

## Method

- **Conceptual criterion for Mamba's usefulness**: the paper revisits selective SSM with recurrence `h_t = \bar{A} h_{t-1} + \bar{B} x_t`, `y_t = C h_t`, arguing that its fixed-size recurrent state is inherently causal and lossy, unlike fully visible attention.
- **Two-task-characteristic hypothesis**: the authors claim Mamba is best suited to tasks that are both long-sequence and causal/autoregressive. They contrast causal token mixing `y_t = f(x_1, ..., x_t)` with fully visible mixing `y_t = f(x_1, ..., x_T)`.
- **Long-sequence test**: using a Transformer block with MLP ratio `4`, they derive `FLOPs = 24 D^2 L + 4 D L^2` and the quadratic-to-linear ratio `r_L = L / (6D)`. They treat `L > 6D` as the regime where sequence length is genuinely dominant.
- **Task diagnosis**: for ImageNet classification with `224 x 224` inputs and `16 x 16` patches, `L = 196`, well below thresholds `2304` for `D = 384` and `4608` for `D = 768`; for COCO detection at roughly `800 x 1280` and ADE20K segmentation at roughly `512 x 2048`, token counts are about `4K`, which the authors treat as long-sequence.
- **MambaOut block construction**: starting from the shared meta-architecture `X' = Norm(X)` and `Y = (TokenMixer(X'W_1) \odot \sigma(X'W_2)) W_3 + X`, they replace Mamba's `TokenMixer_Mamba(Z) = SSM(\sigma(Conv(Z)))` with `TokenMixer_GatedCNN(Z) = Conv(Z)`.
- **Token mixer details**: the convolution is a `7 x 7` depthwise convolution, following ConvNeXt-style design, and is applied only to partial channels for practical speed following InceptionNeXt-style partial convolution.
- **Model family**: MambaOut uses a four-stage hierarchical vision backbone with stem `3 x 3` conv stride `2`, GELU, and another `3 x 3` conv stride `2`, plus strided `3 x 3` downsampling between stages. Tiny/Small use channels `(96, 192, 384, 576)`; Base uses `(128, 256, 512, 768)`. Tiny has blocks `(3, 3, 9, 3)`; Small/Base use `(3, 4, 27, 3)`.
- **Block hyperparameters**: the Gated CNN block uses MLP ratio `8/3`, global average pooling plus normalization and MLP for classification, and residual-style stacking over the four stages.
- **ImageNet training**: all models train for `300` epochs with AdamW, batch size `4096`, learning rate `4e-3`, Adam `eps = 1e-8`, betas `(0.9, 0.999)`, weight decay `0.05`, cosine decay, warmup `20` epochs, label smoothing `0.1`, Mixup `0.8`, CutMix `1.0`, random erasing `0.25`, and peak stochastic depth from `0.025` to `0.6` by model size.
- **Dense prediction transfer**: for COCO, MambaOut is used as a Mask R-CNN backbone with a `1x` (`12`-epoch) schedule, AdamW `lr = 1e-4`, batch size `16`, resized inputs with short side `800` and max long side `1333`; for ADE20K, MambaOut is used as a UperNet backbone with AdamW `lr = 1e-4`, batch size `16`, and `160000` iterations. Both use ImageNet pretraining and FP16.

## Key Results

- **ImageNet classification**: MambaOut consistently beats visual Mamba baselines across scales. `MambaOut-Tiny` reaches `82.7%` top-1 vs `82.2%` for `VMamba-T`; `MambaOut-Small` reaches `84.1%` vs `83.7%` for `LocalVMamba-S`; `MambaOut-Base` reaches `84.2%` vs `83.7%` for `VMamba-B`.
- **Efficiency on ImageNet**: `MambaOut-Small` achieves `84.1%` at `9.0` GMACs, compared with `LocalVMamba-S` at `83.7%` and `11.4` GMACs, supporting the paper's claim that SSM is unnecessary for this task.
- **COCO detection / instance segmentation**: `MambaOut-Tiny` scores `45.1 AP^b / 41.0 AP^m`, trailing `VMamba-T` by `1.4 AP^b` and `1.1 AP^m`; `MambaOut-Small` gets `47.4 / 42.7`, below `VMambaV9-S` at `48.7 / 43.7`; `MambaOut-Base` gets `47.4 / 43.0`, below `VMambaV9-B` at `49.2 / 43.9`.
- **ADE20K segmentation**: `MambaOut-Tiny` reports `47.4` mIoU single-scale and `48.6` multi-scale, slightly above `VMamba-T` (`47.3 / 48.3`) but below `LocalVMamba-T` (`47.9 / 49.1`); `MambaOut-Base` reaches `49.6 / 51.0`, below `VMambaV9-B` at `51.0 / 51.6`.
- **Stronger non-Mamba baselines remain ahead**: on ImageNet, `TransNeXt-Tiny` reaches `84.0%` and `CAFormer-M36` reaches `85.2%`; on COCO, `TransNeXt-Tiny` reaches `49.9 AP^b`; on ADE20K, `TransNeXt-Base` reaches `53.0 / 53.4` mIoU. The paper's contribution is therefore diagnostic rather than state-of-the-art.

## Limitations

- The central claim is conceptual and empirical, not a formal proof that SSM is unnecessary for vision.
- The "long-sequence" criterion `L > 6D` is derived from a simplified Transformer FLOPs decomposition and may not transfer cleanly across all backbone families, resolutions, or implementation regimes.
- MambaOut does not beat the strongest visual Mamba models on COCO or ADE20K, so the paper only supports a narrow negative claim for ImageNet-style classification.
- The work studies visual tasks only; the conclusion section explicitly leaves LLM and multimodal extensions as future work rather than evidence-backed claims.
- Even outside Mamba, the best-performing dense prediction systems in the tables are stronger hybrid conv-attention models, so the paper does not establish MambaOut as a new overall visual SOTA.
