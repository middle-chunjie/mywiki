## Summary

DRAGIN proposes a dynamic [[retrieval-augmented-generation]] framework that decides both when to retrieve and what query to issue during decoding, instead of relying on fixed retrieval intervals or only the most recent sentence. The method combines [[real-time-information-needs-detection]] (RIND), which triggers retrieval from token entropy, downstream attention importance, and stopword filtering, with [[query-formulation-based-on-self-attention]] (QFS), which extracts high-attention query terms from the full context. Across 2WikiMultihopQA, HotpotQA, StrategyQA, and IIRC, the framework improves over prior dynamic RAG baselines on all evaluated model families, with especially large gains on multihop settings. The paper is useful because it turns [[self-attention]] from an internal diagnostic into a lightweight control signal for test-time retrieval in [[large-language-model]] generation.

## Problem & Motivation

The paper targets factual errors and missed evidence in [[large-language-model]] outputs on [[knowledge-intensive-generation]] tasks. Prior dynamic RAG systems make brittle design choices in both dimensions of retrieval control: they often retrieve on fixed schedules and formulate queries only from the latest sentence or token window. The authors argue that this can inject irrelevant context, waste inference budget, and fail to capture the model's actual information needs when those needs depend on earlier parts of the prompt or emerging chain-of-thought. DRAGIN is motivated by the idea that the model's own uncertainty and attention patterns can provide better test-time signals for retrieval timing and query construction.

## Method

- **Framework scope**: DRAGIN is a training-free dynamic [[retrieval-augmented-generation]] method for Transformer-based LLMs; it does not require additional fine-tuning, prompt engineering, or retriever training.
- **RIND uncertainty term**: for generated sequence `T = {t_1, ..., t_n}`, token uncertainty is measured by entropy `H_i = -Σ_{v in V} p_i(v) log p_i(v)`, where `p_i(v)` is the next-token distribution at position `i`.
- **RIND importance term**: token importance is estimated from last-layer self-attention via `A_{i,j} = softmax(Q_i K_j^T / sqrt(d_k))` and `a_max(i) = max_{j>i} A_{i,j}`, so a token is scored by how strongly later tokens depend on it.
- **RIND semantic filter**: stopwords are masked with a binary indicator `s_i = 0` if `t_i in S` else `1`, using spaCy `en_core_web_sm`; the final trigger score is `S_RIND(t_i) = H_i * a_max(i) * s_i`, and retrieval fires when any token exceeds threshold `theta`.
- **QFS query formulation**: once token `t_i` triggers retrieval, QFS ranks all previous tokens by the last-layer attention vector from position `i`, selects the top `n` tokens, restores them to original text order, and uses those words as the retrieval query. The evaluated `n` values are dataset/model dependent, typically `25`, `35`, or `40`.
- **Generation after retrieval**: the partial output is truncated at the trigger point as `T' = truncate(T, t_i)`, retrieved passages are injected with the FLARE-style prompt template, and decoding resumes from the truncation point.
- **Implementation details**: the retriever is [[bm25]] over Wikipedia segmented into `100`-token passages with `top-k = 3` documents. Evaluated LLMs are [[llama-2]] `7B/13B` Chat and [[vicuna-13b-v1-5]]. Appendix hyperparameters set generation length to `64/100/128` depending on dataset and `theta` in the range `0.75-1.5`.

## Key Results

- **Overall performance, Llama2-13B-Chat**: DRAGIN reaches `0.309/0.3971` EM/F1 on [[2wiki-multihopqa]], `0.337/0.4415` on [[hotpotqa]], `0.669` accuracy on [[strategyqa]], and `0.194/0.2352` on [[iirc]].
- **Comparison to dynamic baselines**: on 2WikiMultihopQA with Llama2-13B, DRAGIN improves over FS-RAG from `0.270/0.3610` to `0.309/0.3971`; on HotpotQA it improves over FS-RAG from `0.267/0.3715` to `0.337/0.4415`.
- **Vicuna gains**: with [[vicuna-13b-v1-5]], DRAGIN achieves `0.266/0.3628` on 2WikiMultihopQA, `0.296/0.4185` on HotpotQA, `0.687` on StrategyQA, and `0.2379/0.2829` on IIRC, outperforming the compared baselines across all four tasks.
- **Ablations**: on IIRC, replacing only the retrieval-timing policy shows DRAGIN outperforming FLARE / FL-RAG / FS-RAG for both Llama2-13B (`0.187/0.2242`) and Vicuna-13B (`0.196/0.2367`). For query formulation, DRAGIN reaches `0.315/0.4274` on HotpotQA with Llama2-13B versus `0.262/0.3674` for FLARE queries.
- **Retriever and threshold analysis**: BM25 beats SGPT on all three reported datasets, e.g. HotpotQA `0.442` vs `0.371` F1 and IIRC `0.235` vs `0.201` F1. On HotpotQA with Llama2-13B, varying `theta` from `1.0` to `1.4` keeps F1 in a narrow `0.4339-0.4415` band while retrieval calls fall from `3.840` to `3.505`.

## Limitations

- The method depends on access to self-attention scores, so it is not applicable to black-box APIs that expose only text completions.
- Evaluation is restricted to four QA-style [[knowledge-intensive-generation]] benchmarks over Wikipedia; the paper does not test long-form generation, dialogue, or non-Wikipedia corpora.
- RIND still relies on manually chosen thresholds `theta`, stopword filtering, and top-`n` query selection, even though the sensitivity analysis suggests moderate robustness.
- DRAGIN does not dominate every single metric against every baseline: for example, on IIRC with Llama2-13B, SR-RAG has slightly higher EM (`0.196` vs `0.194`) although DRAGIN has better F1 (`0.2352` vs `0.2303`).
