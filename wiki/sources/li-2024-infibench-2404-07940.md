## Summary

InfiBench introduces a benchmark for free-form code question answering rather than conventional code generation. The dataset contains `234` carefully curated Stack Overflow questions spanning `15` programming languages and `5` broad areas, with prompts paraphrased by domain experts and paired with question-specific automatic grading criteria. Instead of relying on an LLM judge, the benchmark mixes keyword matching, blank filling, unit testing, and dialogue similarity, aiming to score diverse real-world coding questions while remaining model-independent. The authors evaluate more than `100` proprietary and open-source code LLMs and show that even GPT-4-0613 reaches only `70.64%`, while strong open models such as DeepSeek-Coder-V2 and Codestral-22B remain noticeably behind. The paper positions InfiBench as an unsaturated, contamination-aware benchmark for code LLM QA ability.

## Problem & Motivation

Existing code benchmarks disproportionately emphasize code generation, often reuse overlapping source problems, and are increasingly saturated by frontier models. This makes them weak proxies for the broader question-answering behavior developers expect from code LLMs in real usage, such as debugging explanations, API knowledge, or environment troubleshooting. The authors therefore target a benchmark built from natural Stack Overflow questions, with broader task diversity, stronger resistance to benchmark memorization, and automatic scoring that does not depend on a costly or biased LLM judge.

## Method

- **Source pool construction**: start from `23.54M` Stack Overflow questions and `34.68M` answers; keep questions with at least `3` positively voted answers and `1` accepted answer, yielding `1,090,238` candidates.
- **Popularity and freshness filtering**: retain questions above the empirical boundary connecting `(0, 5)` and `(3000, 15.5)` in the space of days-since-creation vs. `log(view_count)`, leaving `17,402` questions.
- **Seed set and taxonomy**: manually build a tag tree over the top `200` tags, map questions into `5` areas, exclude `6` data/domain-specific languages (`JSON`, `regex`, `Markdown`, `YAML`, `CSV`, `SQL`), and obtain a seed set of `13,854` questions across `24` programming languages.
- **Sampling design**: allocate area quotas of `25% / 25% / 25% / 15% / 10%` for front-end, back-end, DS&ML, mobile-desktop, and ITOps; enforce minimum per-language quota for low-frequency languages; expand the tentative sample to `270` questions before post-filtering to the final `N = 234`.
- **Human annotation**: `5` domain experts each own one area and perform `3` steps: question selection plus type labeling, prompt paraphrasing, and YAML correctness-criterion annotation with cross-validation and calibration.
- **Automatic evaluation framework**: support `4` model-free metric families: keyword matching, blank filling, unit testing, and dialogue similarity. Blank filling uses longest common subsequence matching; unit-testing uses executable runtimes for `9` languages.
- **Scoring and protocol**: each question contributes `1` point, which can be decomposed into weighted subcriteria such as keyword hits. Main leaderboard uses `best@10` with sampling temperature `T = 0.2` and nucleus parameter `p = 0.9`.
- **Evaluation setup**: benchmark more than `100` code LLMs from proprietary and open-source families; proprietary models are queried by API, while open-source models are run on an `8 x A100` server through `bigcode-evaluation-harness`.

## Key Results

- Final benchmark size is `234` questions over `15` languages and `5` areas; question types are `30.37%` code completion, `27.04%` knowledge QA, `26.67%` code debugging, and `15.93%` config/environment debugging.
- Metric mix is `57.41%` keyword matching, `12.22%` blank filling, `19.26%` unit testing, and `11.85%` dialogue similarity.
- Human-alignment study shows `85.1%` agreement between InfiBench grading and human experts, versus `77.8%` for GPT-4o-based judging.
- GPT-4-0613 reaches `70.64% ± 0.82%`, Claude 3 Opus `63.89%`, DeepSeek-Coder-V2 `65.49%`, and Codestral-22B `62.98% ± 0.56%`; copied Stack Overflow answers score only `65.18%`.
- Open models can beat GPT-3.5-turbo (`56.47%`), but a visible gap to GPT-4 remains. The paper also reports a non-trivial scaling-law break beyond roughly `40B` parameters for open models.
- Additional fine-tuning on sanitized Stack Overflow data improves CodeLlama-13B-Instruct from `46.37%` to `60.74%` and Mixtral-8x7B-Instruct from `55.55%` to `62.61%`.

## Limitations

- Correctness criteria are expert-authored and calibrated, but still subjective rather than universally objective.
- Benchmark size is modest because annotation is labor-intensive; less popular languages remain underrepresented and some were removed after filtering.
- Data contamination is mitigated through paraphrasing and analysis, but not eliminated; future public benchmark exposure can itself induce contamination.
- The score mainly measures answer correctness, not broader usability dimensions such as naturalness, trustworthiness, latency, or safety behavior.
