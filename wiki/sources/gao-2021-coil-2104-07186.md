## Summary

⚠ Possibly outdated: published 2021; re-verify against recent literature. COIL revisits first-stage lexical retrieval by replacing term-frequency heuristics with contextualized token-vector scoring while preserving exact term overlap and inverted-list efficiency. Queries and documents are encoded with BERT; for each overlapping token, COIL scores the maximum dot product between the query token vector and matching document token vectors stored in that token's contextualized inverted list, and optionally adds a global `[CLS]` similarity term. This design targets semantic mismatch without paying ColBERT-style all-to-all late-interaction cost. On MS MARCO passage and document ranking, COIL-tok strongly surpasses BM25, DeepCT, DocT5Query, and competitive dense retrievers, while COIL-full approaches ColBERT-level effectiveness with materially lower latency and much simpler indexing.

## Problem & Motivation

Classical exact-match retrievers such as [[bm25]] are efficient because they score only documents appearing in query-term inverted lists, but they fail on vocabulary mismatch and semantic mismatch. Neural rerankers and dense retrievers improve semantic matching, yet full cross-attention or all-to-all token interaction is too expensive for first-stage retrieval. The paper asks whether exact lexical matching itself can be made semantically richer by contextualizing matched tokens, so that a system keeps the efficiency, explainability, and controllability of lexical search while recovering much of the effectiveness of modern neural IR.

## Method

- **Lexical starting point**: COIL keeps the classical exact-match decomposition `s = Σ_{t in q cap d} σ_t(h_q(q, t), h_d(d, t))` and changes the per-term signal from heuristic statistics to contextualized vector similarity.
- **Token encoding**: a Transformer LM, instantiated with BERT base uncased, encodes the `i`-th query token as ``v_i^q = W_tok LM(q, i) + b_tok`` and the `j`-th document token as ``v_j^d = W_tok LM(d, j) + b_tok``. The token projection dimension is reduced from the LM hidden size to `n_t`.
- **Contextualized exact match**: COIL-tok scores only exact lexical overlaps via ``s_tok(q, d) = Σ_{q_i in q cap d} max_{d_j = q_i} (v_i^{qT} v_j^d)``. The `max` operator keeps the strongest matched mention for each query token.
- **CLS semantic backoff**: COIL-full adds a global sequence-level similarity term using projected `[CLS]` vectors, ``s_full(q, d) = s_tok(q, d) + v_cls^{qT} v_cls^d``, to mitigate vocabulary mismatch not covered by exact token overlap.
- **Training objective**: the model is optimized with negative log likelihood, ``L = -log exp(s(q, d+)) / (exp(s(q, d+)) + Σ_l exp(s(q, d_l^-)))``, using in-batch negatives plus hard negatives sampled from top-`1000` BM25 results.
- **Index organization**: for each vocabulary term `t`, COIL stores all contextualized document vectors with surface form `t` in a contextualized inverted list ``I^t = {v_j^d | d_j = t, d in C}``. Each list is stacked into a matrix `M^{n_t x |I^t|}` so query-time scoring becomes matrix-vector products over only the query's touched lists.
- **Default architecture**: the main system uses `n_t = 32`; COIL-full uses `n_c = 768` for the CLS branch, giving about `110M` parameters. The paper also studies smaller settings such as `n_c = 128` and `n_t = 8`.
- **Optimization and batching**: models are trained for `5` epochs with AdamW, learning rate `3e-6`, warm-up ratio `0.1`, and linear decay. Each query uses `1` positive plus `7` hard negatives; batch size is `8` on MS MARCO passages and `4` on documents.
- **Practical constraints**: documents are truncated to the first `512` tokens. Latency is measured on dual Xeon E5-2630 v3 CPUs and an RTX 2080 Ti GPU; the implementation uses NumPy on CPU and PyTorch on GPU for the matrix products and scatter ranking.

## Key Results

- **MS MARCO passage retrieval**: COIL-tok reaches `MRR@10 = 0.341`, `Recall@1K = 0.949`, `NDCG@10 = 0.660`, and `MRR@1K = 0.915`; COIL-full improves to `0.355 / 0.963 / 0.704 / 0.924`, versus BM25 `0.184 / 0.853 / 0.506 / 0.825` and the paper's dense baseline `0.304 / 0.932 / 0.635 / 0.898`.
- **MS MARCO document retrieval**: COIL-tok gets `MRR@10 = 0.385`, `Recall@1K = 0.952`, `NDCG@10 = 0.626`, and `MRR@1K = 0.921`; COIL-full reaches `0.397 / 0.962 / 0.636 / 0.913`, outperforming BM25 `0.230 / 0.886 / 0.519 / 0.805` and the dense baseline `0.340 / 0.883 / 0.546 / 0.785`.
- **Reranking quality**: on passage reranking, COIL-full attains `MRR@10 = 0.348`, essentially on par with ColBERT `0.349` and slightly above BM25+BERT reranker `0.347`; on document reranking, COIL-full reaches `0.388` versus BM25+BERT reranker `0.383`.
- **Against late interaction**: on passage full retrieval, ColBERT reports `MRR@10 = 0.360` and `Recall@1K = 0.968`; COIL-full is close at `0.355` and `0.963` while avoiding all-to-all token interaction over the whole corpus.
- **Dimension-efficiency tradeoff**: reducing COIL-full from `n_c = 768, n_t = 32` to `n_c = 128, n_t = 8` lowers dev `MRR@10` only from `0.355` to `0.347`, but improves latency from `380ms` to `113ms` on CPU and from `41ms` to `21ms` on GPU.
- **Near-BM25 speed regime**: COIL-tok with `n_t = 8` runs at `55ms` CPU and `16ms` GPU, versus BM25 `36ms` CPU, while still exceeding dense retrieval on the paper's DL2019 MRR metric (`0.953` vs. `0.898`).

## Limitations

- COIL-tok still depends on exact lexical overlap, so vocabulary mismatch is not solved unless the CLS branch in COIL-full can recover it, and that recovery is coarser than token-level matching.
- Documents are truncated to the first `512` tokens, which can drop relevant evidence for long documents and likely matters for the document-ranking setting.
- The search implementation is still exhaustive within each inverted list matrix; approximate search and quantization are mentioned as future work rather than evaluated components.
- Evaluation is limited to English MS MARCO passage/document retrieval and TREC DL 2019, so cross-domain and multilingual generalization are not established.
- The paper is a 2021 preprint in a fast-moving IR area; later sparse, late-interaction, and hybrid retrievers may change the practical comparison frontier.
