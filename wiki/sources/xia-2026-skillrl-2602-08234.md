## Summary

SkillRL proposes a reinforcement-learning framework for LLM agents that replaces raw episodic memory with a structured, evolving skill library. The method first distills successful and failed trajectories into concise reusable skills and failure lessons, then organizes them into a hierarchical SkillBank with always-on general skills and retrieved task-specific skills. A cold-start supervised fine-tuning phase teaches the base model to use these skills before GRPO training begins, and the library is recursively expanded from validation failures during RL. Across ALFWorld, WebShop, and seven search-augmented QA benchmarks, the framework improves both sample efficiency and final performance, showing that skill abstraction is more effective than storing or replaying long noisy trajectories.

## Problem & Motivation

Existing memory-augmented LLM agents usually cache raw trajectories, but those traces are verbose, noisy, and difficult to reuse as compact decision guidance. The paper argues that effective experience transfer requires abstraction rather than trajectory replay: successful and failed episodes should be compressed into reusable skills that can guide future decisions while staying within context limits. The authors also target a second gap in prior work: memory is often static, whereas agents trained with RL should keep expanding and refining their reusable skill set as new failure modes appear.

## Method

- **Agentic RL setup**: the agent follows `\pi_\theta(a_t \mid o_{\le t}, d, c)` in environment `\mathcal{E}` and optimizes `\max_\theta \mathbb{E}_{\tau \sim \pi_\theta}[\sum_{t=0}^{T} \gamma^t r_t]` under a context budget `|c| \le L_{\max}`.
- **Base optimizer**: SkillRL uses GRPO with clipped PPO-style updates, `\mathcal{J}_{\mathrm{GRPO}}(\theta) = \mathbb{E}[\frac{1}{G}\sum_i \min(r_i A_i, \mathrm{clip}(r_i, 1-\epsilon, 1+\epsilon)A_i) - \beta D_{\mathrm{KL}}(\pi_\theta \Vert \pi_{\mathrm{ref}})]`, where `A_i` is group-normalized reward.
- **Experience-based skill distillation**: a teacher model `\mathcal{M}_T` converts successful trajectories into reusable strategies `s^+ = \mathcal{M}_T(\tau^+, d)` and failed trajectories into compact failure lessons `s^- = \mathcal{M}_T(\tau^-, d)`. Failed cases are summarized as failure point, mistaken reasoning/action, corrective action, and general prevention principle.
- **Hierarchical SkillBank**: the library contains general skills `\mathcal{S}_g` shared across task types and task-specific skills `\mathcal{S}_k` specialized to category `k`. Each skill stores a concise name, a principle, and `when_to_apply` conditions.
- **Skill retrieval**: at inference time, general skills are always injected and task-specific skills are retrieved by semantic similarity, `\mathcal{S}_{\mathrm{ret}} = \mathrm{TopK}(\{s \in \mathcal{S}_k : \mathrm{sim}(e_d, e_s) > \delta\}, K)`, followed by action generation from `\pi_\theta(a_t \mid o_{\le t}, d, \mathcal{S}_g, \mathcal{S}_{\mathrm{ret}})`.
- **Cold-start SFT**: before RL, the teacher generates skill-augmented demonstrations `\mathcal{D}_{\mathrm{SFT}} = \{(d_i, \mathcal{S}_i, \tau_i^*)\}` and the policy is initialized by `\theta_{\mathrm{sft}} = \arg\min_\theta \mathcal{L}_{\mathrm{CE}}(\mathcal{D}_{\mathrm{SFT}}; \theta)`. This trained model also becomes the KL reference policy.
- **Recursive skill evolution**: after each validation epoch, categories with success rate `Acc(C) < \delta` trigger failure collection. The teacher analyzes sampled failed trajectories and proposes `\mathcal{S}_{\mathrm{new}} = \mathcal{M}_T(\mathcal{T}_{\mathrm{val}}^-, \mathrm{SkillBank})`, which is merged into the library.
- **Implementation details**: base model `qwen2.5-7b-instruct`; teacher `openai-o3`; RL uses `lr = 1e-6`, effective batch size `64` (`16` with `4` gradient accumulation steps in the main text), group size `8`, `KL coef = 0.01`, invalid-action penalty `0.1`, `K = 6`, threshold `\delta = 0.4`, max prompt length `6000`, max response length `1024`, validation interval `5` steps, and up to `3` new skills per evolution step.
- **Training budget**: cold-start SFT uses `lr = 1e-4`, batch size `16`, `3` epochs, and `7500 / 2400` SFT examples for ALFWorld / WebShop. Full training runs for `150` RL epochs on `8 x NVIDIA H100 80GB` GPUs with about `30` hours wall-clock time per experiment.

## Key Results

- **ALFWorld / WebShop headline numbers**: SkillRL reaches `89.9%` overall success on ALFWorld and `85.2` average score with `72.7%` success on WebShop.
- **Gain over vanilla GRPO**: ALFWorld improves from `77.6` to `89.9` (`+12.3` absolute), with especially large gains on `Cool` (`72.5 -> 95.5`, `+23.0`) and `Pick2` (`64.7 -> 87.5`, `+22.8`).
- **Gain over memory-augmented RL baselines**: Mem0+GRPO reaches `54.7` on ALFWorld and `37.5` WebShop success, far below SkillRL's `89.9 / 72.7`; MemRL is even lower at `21.4` ALFWorld success.
- **Closed-source comparison**: on ALFWorld, SkillRL outperforms `GPT-4o` (`48.0`) by `+41.9` and `Gemini-2.5-Pro` (`60.3`) by `+29.6`.
- **Search-augmented QA**: average score is `47.1`, ahead of `Search-R1` (`38.5`) and `EvolveR` (`43.1`); on `Bamboogle`, SkillRL reaches `73.8` versus `54.4` for EvolveR.
- **Ablations**: removing hierarchy drops performance to `76.8 / 61.4` on ALFWorld / WebShop; replacing the skill library with raw trajectories drops to `61.7 / 50.2`; removing cold-start SFT drops to `65.2 / 46.5`; removing dynamic evolution drops to `84.4 / 70.3`.
- **Efficiency and growth**: the skill library grows from `55` to `100` skills (`12 -> 20` general, `43 -> 80` task-specific), while prompt length falls from roughly `1450` tokens for raw memory to `<1300` for SkillRL, a `10.3%` reduction. The evolving version passes `80%` ALFWorld success within about `60` training steps, versus roughly `90` for the non-evolving variant.

## Limitations

- The method depends on a strong teacher model (`openai-o3`) for trajectory distillation and SFT data generation, so quality and cost may be sensitive to teacher choice.
- Rewards are binary task-success signals, which simplify optimization but may provide weak learning signals for nuanced or partially successful behaviors.
- Dynamic evolution is triggered by heuristic thresholds such as `Acc(C) < 0.4`, and the paper does not deeply analyze sensitivity to these control parameters.
- Evaluation focuses on text/web/search benchmarks with a single open base model (`qwen2.5-7b-instruct`), leaving cross-model robustness and broader open-ended agent settings less established.
- The paper demonstrates strong benchmark gains, but it does not test whether the skill library remains stable, compact, and non-redundant over much longer continual-learning horizons.
