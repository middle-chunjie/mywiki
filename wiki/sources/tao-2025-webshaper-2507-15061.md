## Summary

WebShaper proposes a formalization-driven pipeline for synthesizing training data for web information-seeking agents. Instead of retrieving arbitrary evidence first and asking an LLM to phrase questions afterward, the paper models tasks as compositions of set-theoretic Knowledge Projections (KPs), then uses an agentic Expander to retrieve, validate, and rewrite questions so their reasoning structure matches the intended formal specification. The synthesis pipeline builds `18k` seed questions from offline Wikipedia random walks, expands them layer by layer, and converts successful executions into `5,000` filtered ReAct-style trajectories for supervised fine-tuning and GRPO-based reinforcement learning. Models trained on WebShaper achieve the strongest reported open-source results on GAIA and WebWalkerQA, suggesting that explicit task formalization can improve controllability, consistency, and difficulty of agent-training data.

## Problem & Motivation

Existing information-seeking data synthesis pipelines are largely information-driven: they retrieve web content first and then generate questions from the collected evidence. The paper argues this ordering creates two failure modes. First, the natural-language question can misalign with the latent reasoning structure implied by the retrieved information, producing inconsistent or weakly grounded QA pairs. Second, uncontrolled pre-retrieval tends to collect redundant, homogeneous structures, limiting structural diversity and coverage. WebShaper addresses this by specifying the target reasoning pattern first through a formal language over entity sets and relations, then letting retrieval and question generation serve that formal specification. The goal is better structural control, fewer answer inconsistencies, and harder yet verifiable training tasks for open-web agents.

## Method

- **Task formalization via set theory**: define a universal entity set `\mathcal{E}` and relation subspace `R \subseteq \mathcal{E} \times \mathcal{E}`. A Knowledge Projection is `R(V) = \{u \mid \exists v \in V, (u,v)\in R \text{ or } (v,u)\in R\}`.
- **Compositional target construction**: target entities are formed by intersections and unions of KPs, e.g. `T = \bigcap_i (R_i(S_{i,1}) \cup \dots \cup R_i(S_{i,t_i}))`, with recursive variants `T = R_1(T_1) \cap \dots \cap R_k(T_k)`.
- **KP representation**: each projection is linearized as a triplet `[X, r, S]`, where `X` is a variable and `S` is a variable or constant. The paper uses the distributive identity `R(S_1) \cup R(S_2) = R(S_1 \cup S_2)` to collapse `R`-Union into merged constants.
- **Seed-question construction**: build an offline Wikipedia graph, perform random walks over linked articles, and ask an LLM to generate grounded QA pairs. Filter the results by running the WebDancer framework with QwQ for `5` rollouts per question; keep only questions with at least one correct rollout, yielding about `18k` seed questions.
- **Agentic Expander**: recursively expand questions with `q^{n+1}(T) = \mathrm{Expand}(q^n(T))`. The Expander uses Search, Summarize, and Validate tools to retrieve evidence, merge constants, check formal consistency, and reject questions that an LLM can answer too directly.
- **Layer-wise expansion**: expand only leaf constants in the KP graph so the new graph preserves the answer while avoiding redundant constant-to-constant edges and shortcut paths directly into the target variable. The expansion depth `l` controls difficulty.
- **Trajectory construction**: instantiate a QwQ-based ReAct agent with Search and Visit tools, execute `5` rollouts per input question, and filter trajectories for answer correctness, tool-call validity, hallucinated observations, and severe repetition. The retained training set contains `5,000` trajectories.
- **Training objective**: apply supervised fine-tuning on trajectory tokens while masking observation loss, then optimize with GRPO. Appendix hyperparameters report SFT batch size `32`, learning rate `5e-6`, weight decay `0.1`, warmup plus cosine decay; RL uses group size `8`, temperature `1.0`, `top_p = 1.0`, batch size `128`, mini-batch size `32`, and learning rate `1e-6`.
- **Evaluation setup**: evaluate on GAIA and WebWalkerQA with LLM-as-Judge scoring and the `Pass@1` metric, and compare both against baseline datasets and open-/closed-source deep-research systems.

## Key Results

- On GAIA, WebShaper with `Qwen-2.5-72B` reaches `60.1` average (`69.2 / 63.4 / 16.6` across Levels 1/2/3), the best open-source result reported in the table.
- On WebWalkerQA, the same `Qwen-2.5-72B` setup achieves `52.2` average (`56.2 / 52.1 / 49.5` on Easy/Medium/Hard), again the top open-source score in Table 1.
- With `Qwen-2.5-32B`, WebShaper improves over WebDancer from `40.7` to `52.4` average on GAIA and from `38.4` to `51.4` average on WebWalkerQA.
- With `QwQ-32B`, WebShaper reaches `53.3` average on GAIA and `49.7` on WebWalkerQA, outperforming WebDancer on both averages (`51.5` and `47.9`).
- In dataset-only SFT comparisons on GAIA, training on WebShaper yields `43.6` average for `Qwen-2.5-32B`, `45.6` for `Qwen-2.5-72B`, and `53.3` for `QwQ-32B`, beating the sampled WebWalkerQA / E2HQA / MHQA alternatives in most settings.
- RL on top of WebShaper SFT adds `+7.8` GAIA points for the `32B` model and `+13.5` for the `72B` model, indicating that the synthesized tasks remain useful beyond imitation learning.

## Limitations

- The hardest GAIA setting remains weak: Level-3 accuracy is only `16.6` even for the strongest open-source configuration, far below the paper's cited `47.6` Level-3 score for OpenAI Deep Research.
- The pipeline depends on several external components, including Google search, page visiting/summarization infrastructure, QwQ-based validation, and judge models, so reproducibility and cost are tied to those tools.
- The formalization is tailored to entity-relation-style information-seeking tasks; richer web behaviors such as interactive forms, persistent tool state, or multimodal browsing are not modeled directly.
- Most validation is benchmark-based rather than human-annotated error analysis, so the paper does not fully quantify residual synthesis noise or judge bias.
