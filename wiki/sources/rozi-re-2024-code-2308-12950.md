## Summary

Code Llama is a family of open code-specialized large language models derived from Llama 2 and released in base, Python-specialized, and instruction-tuned variants at `7B`, `13B`, `34B`, and `70B` scales. The paper's central contribution is a staged specialization pipeline that combines large-scale code continued pretraining, fill-in-the-middle training, long-context fine-tuning, and instruction tuning tailored to programming tasks. This design gives the models competitive open-weight code generation, infilling, and repository-scale completion behavior while keeping strong natural-language ability from the base model. Across HumanEval, MBPP, MultiPL-E, and long-context completion benchmarks, the family establishes state-of-the-art results among openly available code models at the time, while also documenting tradeoffs such as mild short-context regression after long-context adaptation and modest pass@k loss from infilling training.

## Problem & Motivation

Open code LLMs lagged behind strong closed systems on synthesis, infilling, and practical coding-assistant use cases. The paper targets three gaps at once: code-domain specialization beyond generic language pretraining, bidirectional infilling for editor-style completion inside files, and much longer context windows for repository-level reasoning. It also argues that instruction tuning for coding should not rely only on generic chat data, because coding assistance benefits from execution-checked self-instruct data and safer alignment without completely sacrificing benchmark performance.

## Method

- **Initialization and model family**: start from Llama 2 style pretrained decoder-only transformers and specialize four scales: `7B`, `13B`, `34B`, and `70B`; release base Code Llama, Code Llama - Python, and Code Llama - Instruct variants.
- **Code continued pretraining**: train `7B/13B/34B` on `500B` additional tokens and `70B` on `1T` tokens using a near-deduplicated public code-heavy corpus; the initial Code Llama mix is `85%` code, `8%` natural language related to code, and `7%` general natural language.
- **Python specialization**: further train Code Llama - Python on `100B` extra tokens with a mix of `75%` Python, `10%` code, `10%` code-related natural language, and `5%` natural language.
- **Tokenizer**: retain the Llama/Llama 2 [[byte-pair-encoding]] tokenizer rather than introducing a code-specific tokenizer.
- **Infilling objective**: train `7B`, `13B`, and `70B` with a multitask [[fill-in-the-middle]] objective; split each document into prefix/middle/suffix spans at the character level, apply the transformation with probability `0.9`, and format half the examples as `PSM` and half as `SPM` with four special infilling markers.
- **Long-context fine-tuning**: extend context from `4,096` to `16,384` tokens during LCFT, then test extrapolation up to `100,000`; modify [[rotary-positional-embedding]] frequencies by increasing the base period from `θ = 10,000` to `θ = 1,000,000`.
- **LCFT optimization**: use learning rate `2e-5`, batch size `2M` tokens for `7B/13B` and `1M` for `34B`, default `10,000` gradient steps, with `11,000` for `34B` and `3,000` for `7B` after observing instability.
- **Instruction tuning**: train Code Llama - Instruct on roughly `5B` tokens from Llama 2 RLHF V5 data plus a coding-specific self-instruct set of about `14,000` question-test-solution triplets distilled from `62,000` generated programming questions.
- **Self-instruct construction**: prompt Llama 2 `70B` to create questions, use Code Llama `7B` to generate tests and `10` candidate Python solutions per question, then keep the first solution that passes the generated tests.
- **Rehearsal during instruct tuning**: mix in `6%` code data and `2%` natural language data to reduce regression on general coding and language understanding.

## Key Results

- On HumanEval pass@1 / MBPP pass@1, base Code Llama reaches `33.5% / 41.4%` at `7B`, `48.8% / 55.0%` at `34B`, and `53.0% / 62.4%` at `70B`; Code Llama - Instruct `70B` reaches `67.8%` on HumanEval and `62.2%` on MBPP.
- Code Llama - Python `7B` outperforms Llama 2 `70B` on both HumanEval and MBPP, scoring `38.4%` vs `30.5%` and `47.6%` vs `45.4%`.
- On MultiPL-E average pass@1, Code Llama `70B` reaches `45.3%`, Code Llama - Instruct `70B` reaches `45.9%`, and Code Llama - Python `70B` reaches `45.0%`, exceeding prior open models reported in the paper.
- On multilingual single-line infilling with MultiPL-E, Code Llama `13B` reaches `74.5%` exact match in Python `SPM`, `80.0%` in Java `SPM`, and `85.0%` in JavaScript `SPM`.
- LCFT improves LCC-balanced single-line completion: `7B` exact match rises from `36.86` to `39.23`, `13B` from `37.96` to `41.06`, and `34B` from `42.52` to `44.89`, with corresponding BLEU gains across all three sizes.
- Infilling training adds only a small downstream cost before LCFT: the paper reports average losses of about `0.6` percentage points for `7B` and `1.1` for `13B` across HumanEval/MBPP pass@k, while preserving autoregressive test loss.

## Limitations

- Long-context fine-tuning improves long-file completion but slightly hurts short-context coding benchmarks; the paper reports an average drop of `0.52` points on HumanEval pass@1 and `1.9` points on MBPP pass@1.
- Retrieval quality still degrades beyond the `16K` fine-tuning length, so extrapolation to very long prompts is not free.
- The `34B` base model was not trained with the infilling objective, so infilling support is inconsistent across released sizes.
- The instruct pipeline depends partly on proprietary RLHF and instruction-tuning data, reducing strict reproducibility.
- The paper notes that LCC examples are included in the training data, which weakens the cleanliness of the long-context completion evaluation.
- Safety-oriented instruction tuning can trade off against raw coding performance, especially for some pass@1 comparisons.
