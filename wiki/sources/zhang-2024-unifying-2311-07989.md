## Summary

⚠ Possibly outdated: published 2023/2024 in the fast-moving LLM/code domain; re-verify against recent literature.

This survey provides a panoramic review of language models applied to software engineering (SE), covering 70+ models, 40+ downstream tasks, 180+ datasets, and 900 related works. The key contribution is unifying the NLP and SE perspectives: SE applies language models to automate development tasks, while NLP borrows SE tasks (especially code generation) as evaluation benchmarks. The survey traces the historical evolution of code modeling from statistical models and RNNs through pretrained Transformers to LLMs and autonomous coding agents. Unlike prior surveys that focus on one community, this work covers the full software development life cycle including requirement engineering, testing, deployment, and operations, and is the first to provide a global view that goes beyond programming-related tasks.

## Problem & Motivation

Prior surveys on language models for code either took the NLP perspective (focusing on text-to-code generation) or the SE perspective (covering SE conference venues), but did not bridge the two communities. There was no comprehensive review covering both the models (NLP side) and the full set of SE downstream tasks (SE side), nor one that unified the taxonomy of pretrained Transformers with the development life cycle taxonomy used in SE. The authors also observed that evaluation of LLMs in NLP was increasingly relying on programming tasks, making a cross-community synthesis timely.

## Method

This is a survey paper; the methodology is literature analysis and taxonomy construction.

- **Taxonomy**: Downstream SE tasks classified by development life cycle stage — requirement engineering, development (code search, generation, editing, suggestion, explanation), testing and analysis, deployment and operations, and novel LLM evaluation tasks.
- **Model taxonomy**: general LLMs with coding ability (GPT family, LLaMA, Qwen, Mistral, Gemini, DeepSeek) vs. specialized code models, further split by architecture — encoder-only (CodeBERT, GraphCodeBERT, SynCoBERT), encoder-decoder (CodeT5/T5+, AlphaCode, PLBART), decoder-only (CodeGen, InCoder, StarCoder, DeepSeekCoder, Phi-1), UniLM (UniXcoder), and diffusion (CodeFusion).
- **Code features**: survey of code-specific pretraining signals — Abstract Syntax Tree (AST), Intermediate Representation (IR), control/data flow graph (CFG/DFG), type information, and program transformations (obfuscation, loop transform, dead code injection).
- **Advanced techniques**: instruction tuning (WizardCoder, OctoCoder, CodeFuse-CodeLLaMA), RL from compiler feedback (CodeRL, PPOCoder, RLTF, Pangu-Coder 2 with RRTF), multi-agent frameworks (ChatDev, MetaGPT), interactive coding with execution feedback.
- **Implementation design**: tracked architectural evolution — pre-norm, parallel attention (GPT-J → PaLM), MQA / GQA (Shazeer 2019, Ainslie 2023), RoPE (Su 2024), ALiBi (Press 2022), FlashAttention (Dao 2022), mixed-precision BF16.
- **Evaluation**: `pass@k` metric as the standard for functional correctness; HumanEval and MBPP benchmarks; repository-level benchmarks (SWE-bench, CrossCodeEval).

## Key Results

- GPT-4 achieves `pass@1 = 67.0` (early version 82) on HumanEval; among open models, Code LLaMA 34B reaches 53.7 and DeepSeekCoder-Instruct 33B reaches 79.3 with instruction tuning.
- Phi-1 (1.3B) trained on 7B high-quality tokens achieves 50.6 pass@1 on HumanEval, comparable to much larger models — demonstrating the primacy of data quality over scale.
- CodeFuse-CodeLLaMA (34B instruction-tuned) reaches 74.4 pass@1, surpassing GPT-4's published score.
- Encoder-decoder models generally outperform decoder-only models under equal compute budget (Wang et al. 2022c; Soltan et al. 2022).
- FIM (fill-in-the-middle) pretrained models (StarCoder, InCoder, DeepSeekCoder) outperform CLM-only counterparts on standard benchmarks.
- SWE-bench: SWE-agent resolves only 12.5% of real GitHub issues, revealing large gap between benchmark saturation (HumanEval) and real-world performance.
- The Stack v2: 32 TB of code in 619 programming languages — the largest public code pretraining corpus.

## Limitations

- Survey covers literature up to early 2024; the field evolves rapidly and some model rankings will be stale quickly.
- Performance comparisons across models are difficult due to differing evaluation settings (few-shot prompts, temperature, sampling budget).
- The survey's coverage of non-English programming contexts and low-resource languages (e.g., Verilog, Haskell, functional languages) is acknowledged as shallow.
- Data contamination of benchmarks (especially HumanEval) is flagged but not systematically analyzed.
- Safety, ethics, and license issues around code LLMs are discussed at high level only.
- The paper does not propose new methods; empirical validation of the survey taxonomy is absent.
