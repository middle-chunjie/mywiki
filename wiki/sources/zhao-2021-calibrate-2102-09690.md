## Summary

GPT-3's few-shot in-context learning accuracy is highly unstable: permuting the order of training examples, changing the prompt format, or selecting different examples can shift accuracy from near-chance to near state-of-the-art. This paper traces the instability to three systematic biases — majority label bias, recency bias, and common token bias — that cause the model's output distribution to be shifted toward certain answers regardless of input content. The proposed fix, **contextual calibration**, estimates this shift using a content-free input (e.g., "N/A") and applies a diagonal affine transformation to flatten the output distribution to uniform before decoding. The method requires no labeled data, adds trivial compute, and improves GPT-3 and GPT-2 accuracy by up to 30 absolute points across ten classification, fact retrieval, and information extraction benchmarks.

## Problem & Motivation

Few-shot in-context learning with large language models offers attractive properties — no parameter updates, natural language interface, low memory overhead — but practitioners report wildly variable results. This paper asks: *why* is GPT-3's accuracy so sensitive to prompt choices, and can the sensitivity be reduced without additional labeled data?

The key empirical finding is that varying only the permutation of four SST-2 training examples can swing GPT-3 2.7B accuracy from 54.3% (near chance) to 93.4% (near state-of-the-art), and this variance persists with more shots and larger models. The instability makes systematic comparison of prompting strategies unreliable.

## Method

**Three biases identified:**

- **Majority label bias**: the model overestimates whichever class appears most frequently in the prompt. Even a 3:1 class imbalance causes GPT-3 to predict the majority class for ~90% of test inputs.
- **Recency bias**: training examples near the *end* of the prompt receive higher weight; the class of the last example is repeated excessively. For LAMA 4-shot, overprediction of training answers increases monotonically: 8.5% / 8.3% / 14.3% / 16.1% for examples 1–4.
- **Common token bias**: the model prefers tokens that are frequent in its pre-training distribution. For DBPedia 0-shot, GPT-3 predicts the "book" class 11× more than the "artist" class; correlation between label-name frequency and prediction rate is r = 0.67.

**Contextual calibration procedure:**

1. Replace the test input with a content-free string (e.g., "N/A", "[MASK]", or empty string) to obtain probability vector `p_cf`.
2. Fit diagonal weight matrix `W = diag(p_cf)^{-1}` and bias `b = 0`.
3. For any test input with raw probabilities `p_hat`, compute calibrated scores `q_hat = softmax(W * p_hat + b)`, then take the argmax.

This is a special case of **vector scaling** (diagonal `W` in the Platt/Guo affine framework). The content-free input makes the parameters fit without any labeled data.

**Implementation details:**
- Ensemble over three content-free inputs: "N/A", "[MASK]", and the empty string; averaging reduces variance.
- For generation tasks (LAMA, slot filling), calibrate only the first output token's distribution; `p_hat` is the full vocabulary distribution (~50k tokens), so diagonal `W` is critical for tractability.
- For LAMA specifically, the content-free input replaces the subject entity (e.g., "N/A was born in").
- No gradient computation or training data required.

**Models evaluated:** GPT-3 (2.7B, 13B, 175B) and GPT-2 (1.5B), accessed via the OpenAI API.

## Key Results

- Contextual calibration improves mean accuracy on GPT-3 across all 10 datasets in Table 1; improvements reach up to **+30.0 absolute** (AGNews 0-shot 175B: 43.9% → 73.9%).
- On SST-2, variance across permutations is visibly reduced; best-case accuracy remains similar while worst-case rises sharply.
- GPT-3 2.7B + calibration outperforms GPT-3 175B baseline on several tasks (by up to **+19.3 absolute**), despite being >50× smaller in parameters.
- Reduces the accuracy *drop* from 0-shot to 1-shot (majority label bias causes this drop; calibration removes it in 3/4 cases studied).
- Calibration also works on GPT-2 XL (Table 4), indicating the bias problem is a general property of left-to-right language models.
- Ablation: contextual calibration approaches oracle performance (best possible diagonal `W` on a held-out validation set) on AGNews (Figure 8).
- Choice of content-free input matters but many options work; ensembling "N/A", "[MASK]", and empty string is the best single recipe.

## Limitations

- Calibration does not eliminate the need for prompt engineering — it narrows the variance but does not guarantee best performance across all prompt choices.
- The method applies a single global shift estimated from one content-free input; it cannot capture input-dependent or example-specific biases.
- Evaluated only on classification, fact retrieval, and slot filling; open-ended generation tasks (e.g., summarization, dialogue) are explicitly listed as future work.
- Relies on access to raw output probabilities; not applicable to APIs that return only the top-1 prediction.
- Common token bias analysis uses Google Ngrams as a proxy for pre-training frequency, which may not perfectly reflect GPT-3's actual training data distribution.
- Results limited to GPT-2 and GPT-3 family; generalization to instruction-tuned or RLHF models (which post-date the paper) is unclear.
