## Summary

This paper compares retrieval-augmented generation (RAG) and long-context (LC) prompting on nine long-context benchmarks using Gemini-1.5-Pro, GPT-4o, and GPT-3.5-Turbo, then proposes Self-Route to combine them. The main empirical finding is that sufficiently capable LC models now beat RAG on average, reversing an earlier narrative that retrieval is usually stronger. The practical counterpoint is cost: RAG remains much cheaper because it feeds only retrieved chunks rather than the full context. Self-Route exploits the fact that RAG and LC often make identical predictions by first asking the model whether retrieved evidence is sufficient, and escalating only unanswered cases to LC. The result is a simple routing scheme that preserves most of LC's accuracy while substantially reducing token usage and API cost.

## Problem & Motivation

Recent long-context models can directly read tens of thousands to millions of tokens, which raises a practical question for knowledge-intensive applications: should one still build retrieval pipelines, or simply rely on long-context prompting? The paper argues that this is now an empirical systems question rather than an article of faith. RAG offers a strong efficiency prior by restricting attention to a few retrieved chunks, but LC may win when modern models can reason over long inputs without retrieval errors. The authors therefore benchmark both approaches under matched settings, study where RAG fails, and design a hybrid that keeps RAG's efficiency while recovering LC's accuracy on harder cases.

## Method

- **Evaluation setup**: compare RAG, LC, and Self-Route on 9 datasets: 7 English query-based subsets from [[longbench]] and 2 very-long-context datasets from [[infinity-bench]]. Metrics are task-specific: F1 for open QA, accuracy for multiple-choice QA, and ROUGE for summarization.
- **Models**: evaluate [[gemini-1-5-pro]], [[gpt-4o]], and [[gpt-3-5-turbo]], with context windows of up to `1,000,000`, `128,000`, and `16,000` tokens respectively.
- **Retriever pipeline**: split each document into chunks of `300` words, embed query and chunks, and retrieve top-`k` chunks by cosine similarity using [[contriever]] or [[dragon]]; default is `k = 5`. Retrieved chunks are ordered by similarity score and prepended with chunk indices.
- **LC prompting**: pass the full context to the LLM and instruct it to answer based only on the provided passage, aiming to suppress parametric knowledge leakage.
- **RAG prompting**: pass only retrieved chunks to the LLM with the same answer-only constraint, so the comparison isolates retrieval versus full-context reading.
- **Self-Route step 1 (RAG-and-Route)**: run a RAG-style prompt but add the option `Write unanswerable if the query can not be answered based on the provided text`. If the model answers, accept that prediction directly.
- **Self-Route step 2 (LC fallback)**: only for queries predicted as unanswerable in step 1, send the full context to the long-context model and use the LC prediction as the final answer.
- **Cost analysis**: compare methods by relative input-token usage rather than only task score, since transformer compute scales roughly with context length and API pricing is token-based.
- **Ablations and diagnostics**: vary retrieval depth `k in {1, 5, 10, 50, 100}`, compare retrievers, inspect failure modes, and test synthetic PassKey variants to reveal evaluation artifacts.

## Key Results

- Across the 9 real datasets, LC beats RAG on average by `7.6%` for [[gemini-1-5-pro]], `13.1%` for [[gpt-4o]], and `3.6%` for [[gpt-3-5-turbo]].
- Self-Route improves over RAG by more than `5` average points for all three models; relative to LC, the average gap is only `-0.2%` for [[gpt-4o]] and `-2.2%` for [[gemini-1-5-pro]], while it is `+1.7%` for [[gpt-3-5-turbo]].
- Routing is aggressive: more than half of queries go through RAG first, and the answerable rate reaches `81.74%` for [[gemini-1-5-pro]].
- Token usage drops sharply: Self-Route with [[gpt-4o]] uses only `61%` of LC tokens while scoring `46.83` versus LC's `47.04`; for [[gemini-1-5-pro]], token usage falls to `38.6%`, corresponding to about `65%` cost reduction.
- RAG and LC are often behaviorally redundant: predictions are exactly identical on `63%` of queries, and score differences are under `10` points on `70%` of queries.
- The `k` ablation shows the best average Self-Route token trade-off around `k = 5`: appendix results report RAG `37.92`, Self-Route `43.33`, and Self-Route token ratio `38.63%`; at `k = 1`, RAG drops to `20.24` while Self-Route still reaches `41.35`.
- On synthetic PassKey, RAG can look deceptively strong (`80.34%` vs. LC `65.25%`), but tiny wording changes collapse it (`4.58%` on the "special token" variant, versus LC `69.32%`), exposing benchmark artifact sensitivity.

## Limitations

- The evaluation uses only three black-box proprietary LLMs and two dense retrievers, so the conclusions may not transfer directly to open-weight models or stronger task-specific retrieval stacks.
- The paper acknowledges unresolved dataset leakage and parametric-knowledge contamination; the prompt-based control ("based only on the provided passage") helps but does not guarantee clean isolation.
- Self-Route depends on the model's own calibration when deciding "answerable" versus "unanswerable", and the paper shows that alignment differences between Gemini and OpenAI models change the cost-performance trade-off.
- Most analysis is limited to English, query-based tasks and a selected subset of LongBench/InfinityBench, leaving multilingual, non-query, and real production pipelines underexplored.
- The synthetic-data section shows that some benchmark conclusions are fragile to prompt wording and task construction, so not all measured gains reflect robust long-context understanding.
