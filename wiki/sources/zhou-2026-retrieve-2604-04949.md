## Summary

This paper identifies a fundamental mismatch between human-centric retrieval training and the needs of agentic search: existing retrievers are trained on human click logs but are increasingly consumed by LLM-powered agents issuing intermediate sub-queries during multi-step reasoning. The authors propose LRAT (Learning to Retrieve from Agent Trajectories), which mines supervision directly from deep research agent interaction traces. Three key behavioral signals are extracted: browsed documents as positive candidates, unbrowsed documents as reliable negatives (free of position bias), and post-browse reasoning length as a relevance intensity proxy. A weighted InfoNCE objective incorporates this intensity signal via an exponential saturation function. Evaluated on InfoSeek-Eval (in-domain) and BrowseComp-Plus (out-of-domain) with six agent backbones spanning 4B–358B parameters, LRAT consistently raises success rate, evidence recall, and execution efficiency.

## Problem & Motivation

Traditional learning-to-retrieve methods are trained from large-scale human interaction logs (clicks, dwell time) and encode human-centric relevance assumptions. With LLM-powered agents increasingly dominating search system usage, retrieval is no longer a standalone endpoint: agents issue dynamically generated intermediate queries as reasoning actions, and retrieval quality directly gates what evidence agents can observe and reason over. Off-the-shelf retrievers trained under human assumptions misalign with agent usage patterns, creating a bottleneck that limits agent performance even as agent reasoning capacity scales up.

## Method

**Trajectory collection**: Trajectories generated by `Tongyi-DeepResearch-30B-A3B` on `10K` InfoSeekQA queries using four retrievers (BM25, Qwen3-Embedding-0.6B/4B/8B). Each trajectory follows ReAct-style alternation: `[Think] → [Search] / [Browse] → [Answer]`. [Search] returns top-`10` candidates with 64-token snippets; [Browse] returns full document content. 26,482 valid trajectories and 91,713 training pairs extracted.

**Relevance mining (Section 5.1)**:
- *Naive positives*: document `d_{t+1}` browsed after query `q_t` treated as positive; all other `K-1` candidates in retrieved set `D_t` treated as naive negatives — no skip-above heuristic needed because agent browsing shows weak position dependence.
- *Reasoning-aware positive filtering*: LLM judge (`Qwen3-30B-A3B-Thinking-2507`) reads post-browse reasoning trace `r_{t+2}` and labels `(q_t, d_{t+1})` as Relevant/Irrelevant. Retains `97.2%` of ground-truth evidence documents and `74.8%` of browsed non-evidence documents.

**Relevance intensity estimation (Section 5.2.1)**: Motivated by time-aware click models where dwell time proxies relevance. Post-browse reasoning length `l` is mapped to a bounded utility score via exponential saturation:

`u(l) = (β / ln 2) · (1 − exp(−ln2 · l / β))`

where `β` = median reasoning length across all trajectories. Normalized weight:

`w = (1 / μ_raw) · (1 − exp(−ln2 · l / β))`

ensures `E[w] ≈ 1` and `μ_raw` = global mean of unnormalized scores.

**Weighted contrastive training (Section 5.2.2)**: Bi-encoder dense retriever with dot-product or cosine similarity. Modified InfoNCE loss:

`L = −(1/N) Σ_i w_i · log [exp(s(q_i, d_i+) / τ) / (exp(s(q_i, d_i+) / τ) + Σ_{d− ∈ N_i} exp(s(q_i, d−) / τ))]`

Negatives `N_i` combine in-trajectory unbrowsed documents with in-batch negatives. Fine-tuning: 2 epochs, batch size `32`, lr `1e-6`, max input `512` tokens, group size `10`, temperature `τ = 0.02`, using [[flagembedding]] framework.

## Key Results

- **InfoSeek-Eval (in-domain)**: LRAT improves success rate across all six agent backbones. Representative gains — WebExplore-8B: `52.0 → 68.7` (+32.1%) with Qwen3-Emb; MiniMax-M2.1-229B: `58.7 → 78.3` (+33.4%); GLM-4.7-358B: `67.7 → 82.0` (+21.1%).
- **BrowseComp-Plus evidence recall (out-of-domain)**: Consistent improvements across agents; relative gains range from `+7%` to `+37.9%`. Example: AgentCPM-Explore-4B recall `23.2 → 32.0` (+37.9%); GLM-4.7 recall `66.6 → 77.8` (+16.8%).
- **Execution efficiency**: Average step counts reduced up to ~30% on InfoSeek-Eval (WebExplore-8B: `24.1 → 19.0` steps).
- **Ablation (BrowseComp-Plus, Qwen3-Emb)**: "+Naive" alone gives large gains; "+Filter" further improves by removing false positives; "+Reweight" adds additional gains from intensity-aware optimization.
- **Scalability**: Performance scales with training data volume up to `30K` trajectories tested; no early saturation observed.
- **Data flywheel**: Retrievers trained on even incorrect trajectories outperform the base (`+18.9–15.3%` across agents); iterative updates yield steady recall and success rate gains.

## Limitations

- Training trajectories are generated by a single agent (`Tongyi-DeepResearch-30B-A3B`) and a single corpus (Wiki-25-Dump), which may introduce agent-specific or corpus-specific biases in the learned retriever.
- The reasoning-length intensity heuristic is a noisy proxy; it depends on the agent's verbosity rather than a direct relevance signal and may degrade for agents with constrained generation budgets.
- Evaluation is bounded to 50 interaction turns per query during inference (vs. up to 100 in training), creating a train/test distribution mismatch.
- No human annotation is used for quality control of trajectories, so systematic errors in agent reasoning could propagate into training data.
- The approach is designed for bi-encoder dense retrievers; applicability to late-interaction models (e.g., ColBERT) or generative retrievers is untested.
