## Summary

⚠ Possibly outdated: published 2024 (submitted 2023); fast-moving LLM4IR area — verify against 2024–2025 literature.

This survey systematically maps how LLMs have been integrated into the four core components of IR pipelines: query rewriting, retrieval, reranking, and reading. For query rewriting, LLMs generate hypothetical documents (HyDE), pseudo-relevant expansions (Query2Doc), and conversational rewrites (LLMCS), addressing vocabulary mismatch and evolving user intent. For retrieval, LLMs both augment training data (pseudo-query generation, soft relevance labels) and serve as backbone encoders (RepLLaMA, GTR) or generative retrievers (DSI). For reranking, supervised encoder–decoder models (monoT5, RankT5) and unsupervised LLM prompting in pointwise, listwise, and pairwise modes are compared. The reader module encompasses passive once/periodic/aperiodic retrieval-augmented generation and active self-querying agents (Self-RAG, FLARE). Search agents (WebGPT, static/dynamic pipelines) are also reviewed. Future directions include latency reduction, multi-modal retrieval, trustworthiness, and generation-oriented evaluation metrics.

## Problem & Motivation

Classical IR systems rely on term-matching (BM25) or modest neural encoders (BERT-scale), which suffer from vocabulary mismatch, data scarcity in specialized domains, and limited semantic reasoning. LLMs' emergent abilities — in-context learning, chain-of-thought reasoning, and generalization from minimal examples — present an opportunity to radically improve each stage of the IR pipeline without the cost of full system redesign. This survey consolidates the rapidly fragmenting literature to provide a taxonomy and identify open challenges.

## Method

The survey organizes LLM-IR integration along five axes:

- **Query rewriting**
  - LLM-only (HyDE, Query2Doc): prompt LLM to generate a hypothetical document; expand query with it; addresses vocabulary mismatch.
  - Corpus-enhanced (QUILL, GRM, InteR): retrieval-augmented prompting or late fusion of LLM rewrites with PRF results.
  - Approach taxonomy: zero-shot / few-shot / chain-of-thought prompting; supervised fine-tuning (BEQUE); knowledge distillation to small student models.

- **Retrieval — data augmentation**
  - Pseudo-query generation: prompt LLM to produce `(doc, synthetic_query)` pairs; filter by round-trip or model-score (inPairs, PROMPTAGATOR, UDAPDR).
  - Relevance label generation: LLM log-likelihood of generating query given doc as soft label (ART).
  - Soft prompt tuning of LLM generator (SPTAR).

- **Retrieval — model architecture**
  - Dense bi-encoders initialized from LLMs: cpt-text (`300M–175B`), GTR (T5 family), RepLLaMA (LLaMA fine-tuned with `</s>` pooling).
  - Task-aware retriever: TART prepends task instruction ("Retrieve a Wikipedia text that answers this question"), distilled from cross-encoder TART-full to bi-encoder TART-dual.
  - Generative retrieval: DSI (T5 fine-tuned to decode DocIDs via semantically structured hierarchical identifiers + constrained beam search); LLM-URL (GPT-3 directly generates URLs in-context).

- **Reranking**
  - Supervised: monoBERT (encoder-only, `[CLS]` relevance head), monoT5/DuoT5/RankT5 (encoder-decoder generating "true"/"false" or ranking losses), RankLLaMA / RankingGPT (decoder-only, two-stage pretraining).
  - Unsupervised pointwise: relevance generation `f(q,d) = exp(S_Y)/(exp(S_Y)+exp(S_N))`; query generation via `score = (1/|q|) Σ log p(q_i | q<i, d, P)`.
  - Unsupervised listwise: RankGPT sliding-window strategy; TREC-DL19 NDCG@10 = 75.59 with GPT-4 vs. 50.58 BM25.
  - Unsupervised pairwise: PRP-AllPairs O(N²); heap/bubble sort variants O(N log N).
  - Data augmentation for reranking: distillation from RankGPT into Vicuna (RankVicuna), Zephyr (RankZephyr).

- **Reader**
  - Passive once-retrieval: RAG, REALM, REPLUG, Atlas — retrieve at start, feed docs as input.
  - Passive periodic: RETRO (cross-attention integration every `n` tokens), RALM, IRCoT (every sentence).
  - Passive aperiodic: FLARE (retrieves when token probability < threshold); Self-RAG (trains LLM to emit reflection tokens deciding when/whether to retrieve).
  - Active reader: Self-Ask, DSP — few-shot prompting triggering self-generated sub-queries.
  - Context compressor: LeanContext (RL-based sentence selector), RECOMP (extractive/abstractive), FILCO (hindsight alignment).

- **Search agent**
  - Static pipeline: LaMDA, SeeKeR, WebAgent, WebGLM — decompose web tasks into modular LLM components.
  - Dynamic: WebGPT — RL training of GPT-3 to use browser API (query, scroll, quote) with human feedback.

## Key Results

- RankGPT4 achieves NDCG@10 75.59 on TREC-DL19 (vs. BM25 50.58, monoBERT 70.50, monoT5 71.48).
- PRP-AllPairs (FLAN-UL2 20B) achieves 72.42 NDCG@10 TREC-DL19, competitive with supervised methods and much smaller than GPT-4.
- RepLLaMA confirms that larger LLM backbones consistently improve dense retrieval, especially in zero-shot transfer.
- DSI shows generative retrieval with semantically structured DocIDs scales with model size (following scaling law).
- QUILL's two-stage distillation surpasses direct model scaling (base→XXL) in query intent understanding.
- Listwise reranking collapses below BM25 with smaller models (FLAN-UL2); requires GPT-4-scale for competitive results.
- Self-RAG outperforms non-adaptive RAG on multiple open-domain QA and fact verification benchmarks.

## Limitations

- Coverage is mostly limited to text retrieval; multi-modal IR (image, audio, video) is surveyed only as a future direction.
- Most evaluated systems rely on proprietary LLM APIs (GPT-3.5/4), limiting reproducibility and cost control.
- Latency constraints of LLM-based retrievers and rerankers remain a major barrier to production deployment.
- Source bias problem: neural IR components exhibit a measurable preference for LLM-generated documents due to lower perplexity, creating evaluation contamination risks.
- Evaluation metrics for reader generation quality lag behind: BLEU/ROUGE are insensitive to factuality and semantic nuance.
- The survey was written in 2023; post-2023 developments (RAG-2.0, embedding APIs, long-context LLMs) are not covered.
