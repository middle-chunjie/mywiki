## Summary

ModernBERT is a modernized encoder-only transformer that delivers a comprehensive Pareto improvement over BERT, RoBERTa, and other existing encoders. Trained on 2 trillion tokens including code data with a native 8192-token context window, it integrates RoPE positional embeddings, GeGLU activations, alternating local/global attention, full-model unpadding, and hardware-aware model design. The resulting models (149M base, 395M large) set state-of-the-art results across GLUE classification, BEIR retrieval (DPR and ColBERT settings), long-context retrieval (MLDR), and code retrieval (CodeSearchNet, StackQA), while being the most memory- and speed-efficient encoder on common GPUs. ModernBERT-base is the first MLM-trained model to surpass DeBERTaV3-base on GLUE since its 2021 release.

## Problem & Motivation

Encoder-only models remain essential in production NLP pipelines for retrieval, classification, NER, and RAG. However, practitioners have been forced to rely on architecturally stale models (BERT, RoBERTa) due to limited improvements since their release: 512-token sequence limits, suboptimal activations and vocabulary sizes, no code data, and poor inference efficiency. Recent partial solutions (MosaicBERT, CrammingBERT, NomicBERT, GTE-en-MLM) addressed individual shortcomings but none achieved a holistic upgrade spanning architecture, data scale, context length, and GPU efficiency simultaneously.

## Method

### Architecture

- **RoPE instead of absolute positional encoding**: `theta = 10,000` for local layers, `theta = 160,000` for global layers after context extension.
- **Pre-normalization**: `LayerNorm` before each sublayer (not post); a single LayerNorm added after the embedding layer; the first attention layer's input LayerNorm removed to avoid redundancy.
- **GeGLU activation**: replaces BERT's GeLU with Gated Linear Unit variant: `GeGLU(x) = (xW) * sigmoid(xV * beta)`; consistent empirical gains over plain GeLU.
- **Alternating local/global attention**: every third layer uses global full-sequence attention; remaining layers use sliding-window local attention with `window = 128` tokens. This reduces quadratic complexity on long sequences while retaining long-range capability.
- **Full-model unpadding**: padding tokens removed before the token embedding layer (not just at the attention layer as in MosaicBERT), concatenated into a single jagged sequence for the entire forward pass; repads output optionally. Yields 10–20% efficiency gain over prior unpadding methods.
- **Flash Attention 3** for global layers; **Flash Attention 2** for local attention layers (FA3 lacked sliding-window support at writing time).
- **`torch.compile`**: ~10% throughput improvement with negligible overhead.
- **Hardware-aware deep-and-narrow design**: 22 layers (base) / 28 layers (large); hidden size `768 / 1024`; GLU expansion `2304 / 5248` chosen to maximize tensor core tiling efficiency across target GPUs (T4, A10, L4, RTX 3090/4090, A100, H100). Bias terms disabled except final decoder layer.

### Data

- 2 trillion tokens: web documents, code, scientific literature; code-aware BPE tokenizer (modified OLMo tokenizer, vocab size `50,368`; multiple of 64 for GPU efficiency).
- **Sequence packing** (greedy algorithm): >99% packing efficiency to avoid high batch-variance from unpadding.
- Next-Sentence Prediction objective removed; MLM masking rate raised to 30% (from BERT's 15%).

### Training Schedule

- **Optimizer**: StableAdamW — AdamW + Adafactor-style per-parameter update clipping.
- **LR schedule**: Warmup-Stable-Decay (trapezoidal): warmup → constant → `1-sqrt` decay. Base: constant LR `8e-4` for 1.7T tokens; large: `5e-4` for 900B then `5e-5` for 800B.
- **Batch size warmup**: from 768 → 4,608 (base) / 448 → 4,928 (large) over 50B / 10B tokens.
- **Weight tiling**: large model initialized from base weights (Phi-style), reducing warmup overhead.
- **Context extension**: after pretraining at `seq=1024`, global attention RoPE theta increased to 160,000 and trained for 300B tokens at `seq=8192` in two phases (constant LR then `1-sqrt` decay with quality upsampling).

## Key Results

- **GLUE (NLU)**: ModernBERT-base 88.4, surpassing all base encoders including DeBERTaV3-base (88.1). ModernBERT-large 90.4, matching DeBERTaV3-large (91.4) with 10% fewer parameters.
- **BEIR DPR (single-vector retrieval)**: base 41.6 nDCG@10 (vs GTE-en-MLM 41.4); large 44.0 (vs GTE-en-MLM 42.5).
- **BEIR ColBERT (multi-vector)**: base 51.3, large 52.4 — substantially above all competitors; NomicBERT base 49.9, GTE base 48.2.
- **MLDR (long-context) OOD ColBERT**: base 80.2, large 80.4 — at least +9 nDCG@10 over next-best long-context model (GTE-en-MLM 69.3 / 71.3).
- **CodeSearchNet**: base 56.4, large 59.5 (vs next-best GTE-en-MLM: 44.9 / 40.5).
- **StackQA (hybrid long-context code)**: base 73.6, large 83.9.
- **Inference speed (RTX 4090, fixed 512 tokens)**: base 148k tok/s, large 52k tok/s; for variable-length inputs ModernBERT is 14.5–31% faster than GTE-en-MLM at short context and ~99–119% faster at long context.
- **Memory (max batch size, 8192 tokens)**: base 98 (vs GTE 38), large 48 (vs GTE 28).
- Short-context fixed-length: slightly slower than vanilla BERT/RoBERTa (explained by higher parameter counts of modern encoders).

## Limitations

- **English-only**: trained exclusively on English data; direct applicability to multilingual or low-resource settings is limited.
- **Web data biases**: trained on large-scale web crawls; representations inherit social biases present in such corpora.
- **MLM-only objective**: DeBERTaV3's RTD (Replaced Token Detection) objective yields stronger classification in some settings; combining MLM+RTD is left as future work.
- **Long-context DPR gap without fine-tuning**: ModernBERT OOD single-vector on MLDR underperforms GTE-en-MLM; likely related to local attention distribution shift or less long-context pretraining compute.
- **Parameter scaling unexplored**: only base (149M) and large (395M) sizes studied; scaling behaviour beyond 395M not characterized.
