## Summary

Mixtral of Experts introduces Mixtral 8x7B, an open-weights sparse mixture-of-experts decoder model that keeps the Mistral 7B backbone but replaces each feed-forward block with `8` experts and a top-`2` router. Although the model contains `47B` total parameters, only `13B` are active per token, giving a much cheaper inference path than dense `70B` models while supporting a full `32k` context window. The paper reports broad benchmark parity or gains over Llama 2 70B and GPT-3.5, with especially strong results on math, code, multilingual tasks, and the instruction-tuned Mixtral Instruct variant trained with SFT plus DPO. It also analyzes expert routing, finding stronger syntactic and temporal locality than clean domain specialization, which matters for expert-parallel serving efficiency.

## Problem & Motivation

The paper targets a practical open-model scaling problem: dense decoder LLMs need very large active parameter counts to reach frontier quality, which raises inference latency, throughput cost, and deployment barriers. The authors aim to keep the strong Mistral 7B recipe while using sparse conditional computation so that each token activates only a small subset of parameters. They also want the resulting model to remain broadly useful rather than narrowly optimized, so Mixtral is pretrained with multilingual data, supports a dense `32k` context window, and is later instruction-tuned. The broader motivation is to show that an open-weights SMoE model can match or exceed dense `70B` baselines without paying dense per-token compute.

## Method

- **Backbone**: Mixtral keeps the Mistral-style decoder-only Transformer backbone and replaces each FFN block with an MoE block. The architecture table reports `dim = 4096`, `n_layers = 32`, `head_dim = 128`, `hidden_dim = 14336`, `n_heads = 32`, `n_kv_heads = 8`, `context_len = 32768`, and `vocab_size = 32000`.
- **Sparse expert layer**: each MoE block has `num_experts = 8`, and each token activates `top_k_experts = 2`. The gate is `G(x) = softmax(TopK(xW_g))`, so only the selected experts are computed.
- **Expert computation**: each expert is a SwiGLU feed-forward block. For an input token representation `x`, the layer output is `y = sum_i softmax(Top2(xW_g))_i * SwiGLU_i(x)`, so tokens can access different experts at different layers and time steps.
- **Capacity / cost split**: the model exposes about `47B` sparse parameters in total while using only `13B` active parameters per token at inference, which is the key compute-efficiency claim against dense baselines.
- **Systems angle**: the paper highlights Megablocks kernels for sparse FFN execution and expert parallelism for distributing experts across devices, while noting that routing imbalance can become a bottleneck for serving and training.
- **Training / post-training**: the base model is pretrained on multilingual data with a fully dense `32k` context. The instruction model is obtained with supervised fine-tuning followed by `Direct Preference Optimization` on paired preference data.

## Key Results

- On the paper's main benchmark suite, Mixtral 8x7B uses `13B` active parameters yet reaches `70.6` MMLU, `59.7` ARC-Challenge, `40.2` HumanEval, `60.7` MBPP, `28.4` MATH, and `74.4` GSM8K, beating or matching Llama 2 70B on almost all reported metrics.
- In the direct comparison table against Llama 2 70B and GPT-3.5, Mixtral scores `70.6` on MMLU vs `69.9` / `70.0`, `85.8` on ARC-Challenge vs `85.1` / `85.2`, `60.7` on MBPP vs `49.8` / `52.2`, and `58.4` on GSM-8K vs `53.6` / `57.1`.
- On multilingual benchmarks, Mixtral beats Llama 2 70B across French / German / Spanish / Italian MMLU with `70.9 / 71.5 / 72.5 / 70.9` versus `64.3 / 64.2 / 66.0 / 65.1`.
- For long-context behavior, Mixtral reaches `100%` passkey retrieval accuracy regardless of sequence length or passkey position, and its proof-pile perplexity decreases monotonically as context size grows.
- For instruction following, Mixtral-8x7B-Instruct-v0.1 reaches `8.30` on MT-Bench and an Arena Elo of `1121`, surpassing Claude-2.1 (`1117`), Gemini Pro (`1111`), and Llama-2-70b-chat (`1077`) on the reported LMSys snapshot.
- In bias evaluation, the base model improves BBQ accuracy from `51.5%` (Llama 2 70B) to `56.0%`; routing analysis also shows strong temporal locality, e.g. first-choice expert repetition at layer `15` reaches `27.9%` on ArXiv and `28.4%` on DM Mathematics versus a random baseline of `12.5%`.

## Limitations

- Sparse compute does not eliminate memory cost: the serving footprint still tracks the `47B` sparse parameter count, and MoE routing adds overhead that is best amortized in batched workloads.
- The paper reports that Mixtral is not uniformly better than Llama 2 70B; reading-comprehension benchmarks remain a relative weak spot, and WinoGrande is slightly lower than the denser baseline in one comparison table.
- The routing analysis does not find strong domain-specialized experts; expert usage appears more syntactic and temporally local than semantically modular, which weakens simple interpretations of MoE specialization.
- Long-context evaluation is narrow: passkey retrieval is synthetic, and the paper does not provide a broader battery of realistic long-context reasoning tasks.
- The post-training section gives high-level SFT/DPO stages but leaves many dataset and optimization details unspecified, limiting exact reproducibility.
