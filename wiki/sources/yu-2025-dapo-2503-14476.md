## Summary

The paper introduces DAPO, an open-source large-scale reinforcement learning system for reasoning-oriented LLMs. Starting from Qwen2.5-32B and evaluating on AIME 2024, the authors argue that naive GRPO underperforms in long chain-of-thought RL because training suffers from entropy collapse, zero-gradient batches, and noisy penalties on truncated samples. DAPO addresses these issues with asymmetric clipping, dynamic sampling, token-level policy gradient loss, and overlong reward shaping, while using rule-based correctness rewards on the 17K-example DAPO-Math-17K dataset. The full system reaches `50` avg@32 on AIME 2024, surpassing the reported `47` of DeepSeek-R1-Zero-Qwen-32B with only `50%` of the training steps, and is released with code and data to improve reproducibility in large-scale LLM RL.

## Problem & Motivation

Large-scale reinforcement learning has become central to reasoning-oriented LLMs, but prior high-performing systems such as OpenAI o1 and DeepSeek R1 do not disclose enough algorithmic detail for reliable reproduction. The authors show that a naive GRPO baseline on Qwen2.5-32B reaches only `30` on AIME 2024 and exhibits several failure modes in long-CoT training: entropy collapses too early, prompts with all-correct or all-wrong samples yield zero effective gradients, and hard penalties on truncated outputs inject reward noise. The paper is motivated by the need for an openly released, industry-scale RL recipe that preserves exploration, stabilizes gradients, and makes reasoning RL reproducible for the broader community.

## Method

- **Objective**: DAPO optimizes a token-level clipped objective `min(r_{i,t}(θ)Â_{i,t}, clip(r_{i,t}(θ), 1-ε_low, 1+ε_high)Â_{i,t})`, where `r_{i,t}(θ) = π_θ(o_{i,t}|q,o_{i,<t}) / π_{θ_old}(o_{i,t}|q,o_{i,<t})`.
- **Group-relative advantage**: like GRPO, it uses group-normalized rewards `Â_{i,t} = (R_i - mean({R_i})) / std({R_i})` over `G` sampled outputs for each prompt, and removes the explicit KL penalty used in standard GRPO variants.
- **Clip-Higher**: DAPO decouples clipping into `ε_low = 0.2` and `ε_high = 0.28`, instead of a symmetric clip, to give low-probability exploratory tokens more upward room while keeping the lower bound conservative.
- **Dynamic Sampling**: before an update, the system keeps sampling until each retained prompt satisfies `0 < |{o_i | is_equivalent(a, o_i)}| < G`, filtering out groups whose rewards would lead to zero gradients.
- **Token-level loss**: unlike sample-level GRPO reduction, DAPO normalizes by the total token count `Σ_i |o_i|`, so longer reasoning traces contribute proportionally to optimization and pathological long generations are penalized more directly.
- **Overlong Reward Shaping**: truncated responses receive a soft length penalty `R_length(y) = 0` when `|y| ≤ L_max - L_cache`, then decrease linearly within the cache interval, and become `-1` when `|y| > L_max`, reducing reward noise from hard truncation.
- **Reward design**: correctness is rule-based with `R(ŷ, y) = 1` if the predicted answer is equivalent to the ground truth and `-1` otherwise, avoiding a learned reward model and its reward-hacking risks.
- **Data and infrastructure**: training is built on [[verl]] and the transformed [[dapo-math-17k]] dataset, which converts math answers into integer-valued targets to simplify reliable rule-based verification.
- **Training hyperparameters**: AdamW with learning rate `1e-6`, linear warm-up over `20` rollout steps, prompt batch size `512`, `16` responses per prompt, mini-batch size `512`, expected max length `16,384`, soft-punish cache `4,096`, and max generation length `20,480`.
- **Evaluation setup**: AIME evaluation reports `avg@32` by repeating the set `32` times with `temperature = 1.0` and `top-p = 0.7`.

## Key Results

- On [[aime-2024]], naive GRPO reaches `30` avg@32, while full DAPO reaches `50`, a gain of `+20` points.
- The reported baseline [[deepseek-r1-zero-qwen-32b]] obtains `47`, so DAPO is `+3` absolute points higher on the same benchmark.
- DAPO reaches its main result with only `50%` of the training steps used by DeepSeek-R1-Zero-Qwen-32B.
- Progressive ablation on AIME 2024 improves as follows: `30` (naive GRPO) -> `36` (+ overlong filtering) -> `38` (+ Clip-Higher) -> `41` (+ soft overlong punishment) -> `42` (+ token-level loss) -> `50` (+ dynamic sampling).
- Training uses `512` prompts per rollout, `16` sampled responses per prompt, and `16` gradient updates per rollout step.
- The paper reports that dynamic sampling increases sampled data volume but still reduces convergence time because fewer training steps are needed.

## Limitations

- The experiments are concentrated on mathematical reasoning, with [[aime-2024]] as the headline benchmark, so cross-domain generalization to coding, planning, or broader open-ended tasks is not established here.
- The reward function is restricted to verifiable tasks with rule-based checking; the method does not address domains where correctness cannot be robustly parsed.
- DAPO depends on long generations up to `20,480` tokens, which implies substantial systems cost and may limit accessibility despite the open-source release.
- The paper provides training-dynamics observations and case studies of emergent reflection, but does not offer a mechanistic explanation for why these behaviors arise.
