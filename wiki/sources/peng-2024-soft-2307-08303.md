## Summary

This paper proposes SPTAR, a data augmentation pipeline for [[dense-retrieval]] under scarce domain-specific labels. Instead of relying on brittle hand-written prompts, the method freezes an open-source LLM and learns task-specific soft prompt embeddings from a small labeled subset, then uses the tuned prompt plus filtered in-context examples to generate weak queries for unlabeled documents. The generated document-query pairs are cleaned with a BM25-based filter and used to train downstream retrievers. On MS MARCO and FiQA-2018, SPTAR consistently beats no-augmentation training and the hard-prompt baseline InPars, with the best NDCG@10 reaching `0.2647` and `0.2785` under BM25CE. The core claim is that prompt optimization and example selection materially improve LLM-based retrieval augmentation.

## Problem & Motivation

Dense retrievers need substantial in-domain supervision, but many retrieval settings provide only a small number of labeled query-document pairs. Prior LLM-based augmentation methods such as hard-prompt query generation help, yet they depend heavily on manual prompt wording and on the particular in-context examples inserted into the prompt. The paper targets this instability: it asks whether task-specific soft prompts, trained on limited labeled data while keeping the LLM frozen, can produce higher-quality weak queries and therefore yield better retrievers than both unsupervised BM25 and hard-prompt augmentation.

## Method

- **Data setup**: split the labeled dataset into `D_train`, `D_eval`, and `D_test`, then sample `S_train^X` and `S_eval^Y` from `D_train`; in the main setting, `X = 50` distinct training queries and `Y = 100` evaluation queries.
- **Soft prompt tuning**: initialize a soft prompt `f_θ(s)` from a repeated hard prompt `s`, freeze LLM parameters `Φ`, and optimize only prompt parameters `θ` by maximizing `log p_{θ,Φ}(q_j | t_j)` for training instance `t_j = [s; c_j]`.
- **Loss**: use negative log-likelihood `L = -log p_{θ,Φ}(q_j | t_j)`, where `c_j` concatenates `M` example document-query pairs with the current target document.
- **Prompt filtering**: after learning `θ*`, sample groups of example pairs from `S_train^X` and pick the group with the lowest evaluation loss on `S_eval^Y`; this explicitly optimizes example selection instead of trusting arbitrary in-context demonstrations.
- **Weak query generation**: use the tuned prompt and filtered examples to generate weak queries for unlabeled documents, producing `W_large = 100K` weak pairs and `W_small = 5000` sampled pairs.
- **Weak data filter**: apply BM25 top-`k` filtering `F_k(W_large)`; if the paired document is not retrieved in the top `k`, discard the weak pair. The paper tunes `k ∈ {10, 30, 50, 70}`.
- **Downstream retrievers**: train DPR, ColBERT, and BM25CE on `S_train^50 + S_eval^100 + F_k(W_large)`.
- **LLMs and hyperparameters**: soft prompt tuning uses LLaMA-7B or Vicuna-7B with `max_length = 1024`, `lr = 3e-2`, AdamW, `max_epochs = 100`, early stop `= 5`, batch size `= 4` for LLaMA-7B and `= 2` for Vicuna-7B on `1 × A100 80G`.
- **Retriever training**: DPR and ColBERT use `batch_size = 32`, `max_length = 350`, `lr = 2e-5`, AdamW, and up to `20` epochs on `4 × A100 40G`; BM25CE re-ranks BM25 outputs using the best DPR BERT checkpoint.

## Key Results

- **Main comparison**: SPTAR achieves the best NDCG among compared methods for all three retrievers. On MS MARCO, NDCG@10 reaches `0.2580` (DPR), `0.2445` (ColBERT), and `0.2647` (BM25CE), beating BM25 (`0.2284`) and InPars (`0.1901`, `0.2418`, `0.2311`).
- **FiQA-2018**: SPTAR reaches NDCG@10 `0.2404` (DPR), `0.2749` (ColBERT), and `0.2785` (BM25CE), again above BM25 (`0.2361`) and InPars (`0.2226`, `0.2370`, `0.2673`).
- **Average gains over BM25**: the reported improvements are `11.82%`, `16.94%`, and `18.04%` on average across NDCG, MAP, and Recall depending on the retriever/metric slice summarized in Table 6.
- **Soft prompt filter**: choosing the best examples instead of the worst reduces perplexity from `4.1934` to `3.6649` on MS MARCO (`+12.60%`) and improves DPR NDCG@10 from `0.2132` to `0.2376` (`+11.44%`); on FiQA-2018, perplexity drops from `410.9207` to `5.7898` and NDCG@10 rises from `0.1855` to `0.1923`.
- **Weak data filter**: on MS MARCO, DPR NDCG@10 improves from `0.2319` without filtering to `0.2580` with top-`k = 30`; on FiQA-2018, it improves from `0.2242` to `0.2404` with top-`k = 70`.
- **Efficiency**: only `0.0030%` of parameters are trainable for LLaMA-7B and Vicuna-7B prompt tuning, with best convergence at epoch `5` and `4`, respectively.

## Limitations

- The evaluation only covers two retrieval datasets, MS MARCO and FiQA-2018, so cross-domain generality is still limited.
- Performance depends on several tuned hyperparameters, including `X`, `M`, LLM choice, and weak-data filter top-`k`, which reduces plug-and-play usability.
- The soft prompt filter does not exhaustively score all example-pair combinations when the search space is large; it samples candidate groups for tractability.
- The method still depends on the quality of LLM-generated weak queries, so bad generations can propagate into retriever training before filtering.
- The paper uses a pointwise generation loss and explicitly notes that negative instances are underused; pairwise or listwise objectives are left to future work.
