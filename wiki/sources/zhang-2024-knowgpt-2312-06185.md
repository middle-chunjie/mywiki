## Summary

⚠ Possibly outdated: published 2024 (arXiv 2023); KG-augmented LLM prompting is a rapidly evolving area — re-verify against recent literature.

KnowGPT is a black-box knowledge injection framework that augments closed-source LLMs (ChatGPT, GPT-4) with structured knowledge from knowledge graphs via API-only access. The system first uses deep reinforcement learning to extract concise, context-relevant reasoning paths from a KG, then applies a Multi-Armed Bandit (MAB) algorithm to adaptively select the best combination of path extraction strategy and prompt format for each question. Evaluated on CommonsenseQA, OpenBookQA, and MedQA-USMLE, KnowGPT achieves 23.7% average improvement over ChatGPT and 2.9% over GPT-4, with 91.6% accuracy on the OpenBookQA leaderboard, approaching human-level performance.

## Problem & Motivation

State-of-the-art LLMs like ChatGPT and GPT-4 are restricted to black-box API access, preventing white-box knowledge injection techniques (which require model architectures and weights). These models suffer from hallucination and limited domain-specific knowledge (e.g., medical, commonsense) because their training corpus does not cover all specialized knowledge. Two key challenges arise: (1) selecting the most relevant knowledge triples from million-triple KGs given LLMs' limited context windows (2048–4096 tokens), and (2) encoding that knowledge in the most effective prompt format — minor prompt variations can drastically change LLM outputs.

## Method

**Overall framework**: Given a question Q with source entities Q_s and target entities Q_t, KnowGPT learns a prompting function `f_prompt(Q, G)` that incorporates KG knowledge into a prompt for black-box LLM `f`.

**Path Extraction with Deep RL (P_RL)**:
- The reasoning subgraph G_sub is modeled as a set of paths `{P_1, ..., P_m}`, one per source entity.
- Each path is sampled as an MDP with state `s_t = (e_t, e_target − e_t)` (current + target entity embeddings), actions = neighboring entities, deterministic transitions.
- Reward is composite: `r_total = λ_1 * r_reach + λ_2 * r_cr + λ_3 * r_cs`
  - `r_reach ∈ {+1, −1}`: whether the path reaches the target within `K` steps
  - `r_cr = (1/|i|) Σ cos(W × P_i, c)`: cosine similarity of mapped path embedding to question context embedding (step-wise)
  - `r_cs = 1/|P_i|`: conciseness reward; shorter paths score higher
- Policy network `π_θ(s, a)` is trained via policy gradient on training questions; node embeddings are initialized from KG triples converted to sentences and passed through BERT-Base.
- Fallback: `P_sub` extracts a 2-hop subgraph around source and target entities heuristically.

**Prompt Construction with MAB**:
- Two path extraction strategies (P_RL, P_sub) × three prompt formats (F_t triples, F_s sentences, F_g graph description) = 6 combinations PF_i.
- MAB selects the best combination per question by maximizing `E(Q|PF_i) = c · α_i + β_i`:
  - `α_i` learned via OLS: `α_i = ((C_i^T C_i + λ_i I)^{-1}) C_i^T r_pf_i`
  - `β_i = γ * sqrt(c_i ((C_i^T C_i + λ_i I)^{-1}) c_i^T)` where `γ = 1 + sqrt(ln(2/δ)/2)` (UCB exploration term)
- Reward `r_pf ∈ {0, 1}` is LLM answer correctness; feedback is used to update α online.
- Prompt formats: **Triples** (raw KG triples), **Sentences** (fluent natural language paraphrasing), **Graph Description** (LLM-generated structured narrative around center entity).

**KGs used**: ConceptNet (>8M entities, 34 relations) for commonsense tasks; USMLE biomedical KG (9,958 nodes, 44,561 edges, combining UMLS + DrugBank) for medical tasks.

## Key Results

- **CommonsenseQA** (IH split): KnowGPT 82.7% dev / 81.8% test vs. ChatGPT 73.5% / 71.0% (+9.2% / +10.8%)
- **OpenBookQA**: KnowGPT 90.0% dev / 92.4% test vs. ChatGPT 59.8% / 60.0% (+31.2% / +32.4%); vs. GPT-4 87.8% / 91.0% (+2.2% / +1.4%)
- **MedQA-USMLE**: KnowGPT 77.6% dev / 78.1% test vs. ChatGPT 48.4% / 48.7% (+29.2% / +29.4%); vs. GPT-4 73.9% / 76.3% (+3.7% / +1.8%)
- **Average over ChatGPT**: +23.7%; **average over GPT-4**: +2.9%
- **OpenBookQA official leaderboard**: 91.6% (comparable to human 91.7%; 0.4% below GenMC Ensemble)
- Ablation: P_RL alone (ChatGPT, F_s) gives 80.0% test on CSQA vs. P_sub 73.9%; full MAB (P_RL + P_sub + 3 formats) adds another +1.8%
- Prompt format ablation: Sentence format typically outperforms triples by 1–3%; graph description occasionally introduces noise

## Limitations

- **API cost**: each MAB update requires calling the black-box LLM, making training expensive; calling GPT-4 for graph description generation adds further cost.
- **Entity linking dependence**: source entities Q_s are assumed given via concept recognition; errors in linking propagate to path extraction.
- **KG coverage**: only ConceptNet and USMLE KGs tested; domain applicability to KGs with different structural properties (e.g., Freebase, temporal KGs) is unverified.
- **RL instability**: deep RL for path extraction is noted as notoriously unstable, motivating the MAB fallback; hyperparameter sensitivity is not fully analyzed.
- **Context window**: evaluated on 2048-token (ChatGPT) and 4096-token (GPT-4) limits; longer-context models may reduce the need for aggressive path pruning.
- **Comparison gap**: white-box KG-enhanced LMs (e.g., GreaseLM, QA-GNN) cannot use SOTA LLMs as backbone; the comparison is not fully apple-to-apple.
