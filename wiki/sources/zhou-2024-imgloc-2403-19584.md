## Summary

Img2Loc reframes image geolocation — predicting geographic coordinates from a ground-view photograph — as a text generation task handled by large multimodal models (LMMs) such as GPT-4V or LLaVA. The system first encodes a reference gallery of over 4.7 million geotagged images (MP-16) into CLIP embeddings stored in a FAISS vector database. At inference time, the query image's embedding is used to retrieve the most similar (positive neighbors) and most dissimilar (negative neighbors) gallery entries; their GPS coordinates are concatenated with the query image into a structured prompt fed to the LMM. No model training is required. On Im2GPS3k and YFCC4k, Img2Loc (GPT-4V) surpasses prior SOTA by up to +10.67% at 25 km and +9.27% at 750 km thresholds, demonstrating that retrieval-augmented prompting transfers effectively to spatial reasoning.

## Problem & Motivation

Image geolocation is difficult because traditional methods fall into two families, each with fundamental drawbacks. Classification-based approaches partition the Earth into discrete cells and predict cell labels; errors are bounded below by cell size and suffer from edge artifacts when the true location is far from a cell center. Retrieval-based approaches find the nearest neighbor in a geo-tagged gallery; they depend on gallery coverage and feature quality, and a single nearest-match is brittle. Both families require task-specific training. The authors seek a training-free approach that leverages the spatial reasoning capability of LMMs, using retrieval only as a context-injection mechanism rather than as a final predictor.

## Method

- **Image-Location Database**: all images from the MediaEval Placing Tasks 2016 (MP-16) dataset (`4.72M` geotagged Flickr images) are encoded by the CLIP vision encoder into dense embeddings. Embeddings and GPS labels are stored in a FAISS flat index with GPU-accelerated inner-product search.
- **Positive Neighbor Retrieval**: given a query image, compute its CLIP embedding and retrieve the top-`k` nearest neighbors by inner product (higher = more similar). Their GPS coordinates serve as anchor hints.
- **Negative Neighbor Retrieval**: invert the query embedding (negate it) and retrieve the farthest gallery points as "negative neighbors" — geographically distant from the query. These negative coordinates are also appended to the prompt, so the LMM can rule out implausible regions.
- **Augmented Prompt Construction**: a multi-part prompt is assembled from (a) the query image itself, (b) the coordinates of positive neighbors, and (c) the coordinates of negative neighbors. The LMM is instructed to output a precise latitude/longitude.
- **LMM Backbone**: experiments use GPT-4V and LLaVA; the framework is backbone-agnostic and can swap in newer releases.
- **Output**: the predicted coordinate is displayed on an interactive web map.
- No gradient updates are performed at any stage — the system is fully training-free.

## Key Results

On Im2GPS3k (street = 1 km, city = 25 km, region = 200 km, country = 750 km, continent = 2500 km):

- Img2Loc (GPT-4V): `17.10 / 45.14 / 57.87 / 72.91 / 84.68` (%)
- Prior SOTA (GeoCLIP): `14.11 / 34.47 / 50.65 / 69.67 / 83.82`
- Improvements over GeoCLIP: `+2.99 / +10.67 / +7.22 / +3.24 / +0.86` pp

On YFCC4k:

- Img2Loc (GPT-4V): `14.11 / 29.57 / 41.40 / 59.27 / 76.88` (%)
- Prior SOTA (GeoGuessNet): `10.3 / 24.4 / 33.9 / 50.0 / 68.7`
- Improvements over GeoGuessNet: `+3.81 / +5.17 / +7.50 / +9.27 / +8.18` pp

LLaVA-based variant underperforms prior SOTA on both benchmarks, indicating the approach is gated by LMM quality.

## Limitations

- Performance is strongly dependent on the choice of LMM backbone; LLaVA yields worse results than several supervised baselines.
- No analysis of how retrieval database coverage affects performance in under-represented geographic regions (potential global coverage bias).
- Negative neighbor construction is heuristic (farthest embedding from negated query); its interaction with LMM spatial priors is not ablated.
- The number of retrieved positive and negative neighbors `k` is not systematically ablated in the paper.
- At street-level resolution (1 km), improvements over GeoCLIP are modest (~3 pp), suggesting text-generation approaches may still lag at fine granularity.
- Computational cost of calling GPT-4V per image at inference may be prohibitive for large-scale deployment.
