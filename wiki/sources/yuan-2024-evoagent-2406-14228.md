## Summary

EvoAgent proposes a generic way to turn a manually designed single-agent or fixed multi-agent framework into a more capable multi-agent system by treating agent settings as evolvable variables. Starting from an initial agent, the method uses LLM-mediated crossover, mutation, selection, and result-update operators to generate diverse specialized agents and aggregate their outputs. The paper evaluates this evolutionary generation pipeline on reasoning, writing, coding, multimodal QA, interactive science tasks, and real-world travel planning. Across these settings, EvoAgent consistently improves over direct prompting, Self-Refine, and prior human-designed multi-agent baselines such as AgentVerse and AutoAgents, while remaining architecture-agnostic and relatively simple to apply on top of existing LLM-based agent frameworks.

## Problem & Motivation

Existing LLM-based agent systems often depend on handcrafted roles, prompts, task scopes, and collaboration protocols. That manual design is useful for bootstrapping capable agents, but it also constrains scalability: extending a strong specialized agent into a broader multi-agent system requires additional human labor and hard-codes assumptions about which roles should exist. The paper targets this bottleneck by asking whether agent generation itself can be automated, so that an initial agent can be expanded into a diverse team of specialists that better handles complex reasoning, multimodal understanding, interactive environments, and real-world planning.

## Method

- **Initialization**: start from an initial agent `A_(0,0)` and its initial output `R_0`; EvoAgent treats agent settings such as role, skills, and system prompts as evolvable variables rather than fixed human-authored configuration.
- **Crossover**: given parent outputs `R_(0,t-1), ..., R_(N-1,t-1)` and the initial agent, `EvoCrossover(...)` asks an LLM to identify which agent skills should be improved and to update parent settings into revised agents `A'_(i,t-1)`.
- **Mutation**: `EvoMutation(...)` expands the revised parents into a larger child pool `N' > N`, explicitly encouraging diversity so child agents remain distinct from their parents while preserving task-solving capability.
- **Selection**: an LLM-based quality-check module `LLMQuality(...)` filters the candidate pool and keeps `N` evolved agents that both inherit useful parent characteristics and remain meaningfully different from previously generated agents.
- **Result update**: each selected child agent produces a candidate output `R_(i,t)`, and the system integrates them with the prior iteration result via `R_t = EvoUpdate({R_(0,t), ..., R_(N-1,t)}, R_(t-1))`.
- **Search regime**: for most NLP, multimodal, and planning experiments the paper uses `N = 1, T = 3`, so each sample gets three newly evolved specialized agents; for ScienceWorld it uses `N = 1, T = 1` for efficiency because evolution happens at each environment step.
- **Implementation details**: the update/selection LLM uses the same backbone family as the initial agent; OpenAI models are run as `gpt-35-turbo` and `gpt-4-32k` (Azure preview `2024-02-15`), Gemini uses Gemini-Pro APIs, and all models use `temperature = 0`.
- **Generality claim**: the method is designed to sit on top of arbitrary agent frameworks, and the appendix demonstrates adaptation to frameworks such as MetaGPT, CAMEL, and AutoGen without changing their core infrastructure.

## Key Results

- On Logic Grid Puzzle / Trivia Creative Writing / Codenames Collaborative, GPT-4 with `EVOAGENT(1,3)` improves from `60.50 / 75.40 / 79.38` to `77.00 / 84.40 / 84.53`, outperforming Direct, CoT, Self-Refine, AgentVerse, and AutoAgents on all three tasks.
- On the same three tasks, GPT-3.5 improves from `48.00 / 56.20 / 76.29` to `71.50 / 60.80 / 79.38`, showing especially large gains on logic (`+23.5`) and writing (`+4.6`).
- On ScienceWorld, GPT-4 rises from `27.97` to `30.42` overall average score, with the short-trajectory subset improving from `42.41` to `48.67`; GPT-3.5 rises from `17.12` to `19.02`.
- On TravelPlanner, GPT-4 improves commonsense micro from `79.4` to `81.5`, commonsense macro from `15.8` to `21.1`, hard-constraint macro from `16.1` to `18.9`, and final pass rate from `2.2` to `7.2`.
- In the TravelPlanner ablation average, `EVOAGENT(1,3)` with quality check reaches `68.9` commonsense and `14.0` hard-constraint scores versus Direct at `59.5` and `13.7`, supporting the value of evolutionary generation over prompt-only refinement.

## Limitations

- EvoAgent increases token and runtime cost because it must generate, filter, and aggregate multiple specialized agents rather than query a single agent once.
- The paper reports only limited direct comparisons with existing multi-agent frameworks beyond AgentVerse and AutoAgents, so ecosystem-level coverage remains incomplete.
- Gains are weaker on longer-horizon interactive tasks, and the authors explicitly suspect context-length pressure reduces the benefit of larger multi-agent collaboration in those settings.
- The generated agents are not manually evaluated for quality or interpretability; the paper mainly validates them indirectly through downstream task metrics.
- Larger population sizes can create redundant or biased agents, which is why the paper needs an additional quality-check stage to control similarity and failure modes.
