## Summary

⚠ Possibly outdated: published 2023; re-verify against recent literature.

This paper introduces FORB, a flat-object image retrieval benchmark designed to test whether image embeddings trained on common retrieval domains generalize to more diverse 2D objects such as logos, paintings, book covers, cards, currency, packaged goods, and movie posters. The benchmark contains `13,901` query images, `4,585` index images, and `49,850` distractors, with per-query difficulty labels and a new thresholded metric, `t-mAP`, that evaluates not just ranking quality but also the margin between true matches and false positives. Across `10` representative baselines, the study finds that mid-level or handcrafted features often generalize better than many high-level global descriptors on flat-object retrieval, while high-level semantic models remain stronger on feature-scarce domains such as logos.

## Problem & Motivation

Existing image retrieval benchmarks are concentrated on 3D landmarks, clothes, vehicles, or other relatively narrow domains, so they do not cleanly answer whether a learned image embedding is truly universal across object types. Flat 2D objects are common in practical visual search but underrepresented in retrieval benchmarks, and existing flat-object datasets are often small, logo-only, or too visually clean. The paper therefore proposes FORB to stress-test generalization on in-the-wild flat-object queries, with explicit distractors and difficulty labels, and to evaluate both ranking accuracy and score-margin robustness on out-of-distribution retrieval behavior.

## Method

- **Benchmark construction**: FORB covers `8` flat-object domains: animated trading cards, photorealistic trading cards, book covers, paintings, currency, logos, packaged goods, and movie posters. The full benchmark contains `13,901` query images, `4,585` index images, and `49,850` distractors.
- **Difficulty annotation**: each query is labeled as `easy`, `medium`, or `hard` based on occlusion, blur, truncation, color distortion, perspective distortion, texture complexity, and object area in the image. Final counts are `1,175` easy, `11,580` medium, and `1,146` hard queries, with majority voting used to reduce annotator subjectivity.
- **OOD-oriented setup**: instead of adding a separate OOD query split, the benchmark queries images against distractor images so that false positives are explicitly measured; this makes the benchmark useful for analyzing [[out-of-distribution-generalization]] of [[image-embedding]] methods.
- **Ranking metric**: retrieval quality uses `mAP@k = (1 / Q) Σ_q AP@k(q)`, with `AP@k(q) = (1 / min(m_q, k)) Σ_{i=1}^{min(n_q, k)} P_q(i) rel_q(i)`, where `Q` is the number of queries, `m_q` the number of ground-truth matches, and `rel_q(i)` the relevance indicator at rank `i`.
- **Margin-sensitive metric**: the paper introduces [[thresholded-mean-average-precision]], defined as `t-mAP = (1 / τ(1)) ∫_0^{τ(1)} mAP(t) dt`, where `τ(x) = min{t | FPR(t) = 1 - x}` is the threshold achieving a target false positive rate. In practice, the integral is approximated with `11` thresholds: `t ∈ {0, τ(0.1), ..., τ(1.0)}`.
- **Baselines**: `10` retrieval methods are grouped into bottom-up (`BoW`, `FIRe`), top-down (`DELG`, plus reranking), and top-only (`CLIP`, `SLIP`, `BLIP`, `BLIP2`, `DINO`, `DINOv2`, `DiHT`) strategies, enabling comparison across low-, mid-, and high-level feature regimes.
- **Implementation details**: all images are resized so the longest side is `≤ 480` while preserving aspect ratio. `BoW` and `FIRe` build their codebooks from `10k` database images. For top-only methods, query scales are `{1/√2, 1, √2}` and database images use `7` scales; per-scale global features receive `L2` normalization, are average-pooled, and then normalized again in a [[multi-scale-feature-aggregation]] pipeline.
- **Compute setup**: inference speed is measured on a machine with `120 GB` RAM, an `NVIDIA T4` GPU, and `32` Intel Xeon CPUs at `2.30 GHz`.

## Key Results

- FORB substantially expands flat-object retrieval evaluation relative to prior flat benchmarks: `13,901` queries and `54,435` total database-side images (`4,585` index + `49,850` distractors), with explicit distractors and difficulty labels.
- On overall retrieval quality, `CLIP` reaches the best `mAP@5 = 89.36`, while `FIRe` is nearly tied on `mAP@5 = 88.08` and achieves the best `t-mAP@5 = 77.50`, indicating the strongest score margin against false positives.
- `BoW` remains surprisingly competitive with `mAP@5 = 78.44` and `t-mAP@5 = 62.49`, outperforming `DELG` (`48.81` / `34.92`) and several top-only neural baselines on this OOD-style flat-object benchmark.
- Difficulty annotations align with empirical hardness: for `FIRe`, `mAP@5` drops from `98.48` on easy queries to `90.14` on medium and `56.58` on hard; for `CLIP`, the corresponding numbers are `98.23`, `90.00`, and `73.84`.
- On logo retrieval, feature-scarce images strongly favor high-level semantic models: `CLIP` reaches `85.29 / 53.90` (`mAP@5 / t-mAP@5`) while `FIRe` gets `42.50 / 33.32` and `DINOv2` only `6.62 / 1.44`.
- Inference-speed comparisons show a practical tradeoff: `FIRe` runs at `0.124 s/query`, faster than `CLIP` (`0.513 s/query`) and much faster than `DELG (+ rerank)` (`6.015 s/query`), while still matching or exceeding them on several quality metrics.

## Limitations

- Baseline comparisons are not fully apples-to-apples because methods differ in architecture scale, supervision, and training data volume; the authors explicitly note the fairness issue.
- Some top-only baselines may have train-test overlap with FORB because web-scale data such as `LAION-5B` subsets or CLIP's proprietary web crawl can contain duplicate images; the paper treats these results as approximate upper bounds.
- Distractors currently come mainly from the same broad domains as the index images, so cross-domain negative diversity is still limited.
- The benchmark does not yet include a separate set of OOD query domains beyond the `8` indexed domains, which constrains how far the margin analysis can be pushed.
- The dataset is sourced from Internet images with manual filtering, so data provenance, consent, and copyright considerations are only partially addressed in the paper.
