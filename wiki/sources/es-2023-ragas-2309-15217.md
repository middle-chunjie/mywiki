## Summary

⚠ Possibly outdated: published 2023; re-verify against recent literature.

RAGAS proposes a reference-free evaluation framework for [[retrieval-augmented-generation]] pipelines that decomposes quality into [[faithfulness]], [[answer-relevance]], and [[context-relevance]]. Instead of requiring gold answers or human labels at evaluation time, it uses LLM prompting plus embedding similarity to score whether an answer is supported by retrieved evidence, whether it addresses the user question, and whether the retrieved context is focused. The paper also introduces the [[wikieval]] benchmark, a 50-example human-annotated dataset for pairwise comparison across these three dimensions. On WikiEval, RAGAS reaches 0.95 accuracy for faithfulness, 0.78 for answer relevance, and 0.70 for context relevance, outperforming direct GPT scoring and ranking baselines.

## Problem & Motivation

RAG systems depend on several interacting components, including the retriever, source corpus, prompt, and generator, so practitioners need fast feedback when tuning pipelines. Existing automated evaluation strategies are often mismatched to that need: perplexity may not predict downstream QA quality, closed API models may not expose token probabilities, and benchmark-style evaluation with reference answers is expensive and often unavailable in real deployments. The paper therefore targets reference-free, self-contained evaluation that can estimate whether a RAG answer is grounded, on-topic, and supported by focused retrieved context.

## Method

- **Task setup**: for a question `q`, a RAG system retrieves context `c(q)` and generates answer `a_s(q)`. RAGAS evaluates this triple without gold references.
- **Quality dimensions**: the framework scores three aspects: `faithfulness`, `answer relevance`, and `context relevance`.
- **Faithfulness pipeline**: an LLM first decomposes `a_s(q)` into atomic statements `S(a_s(q))`, then checks each statement `s_i` against `c(q)` with a verification function `v(s_i, c(q))`.
- **Faithfulness formula**: the score is `F = |V| / |S|`, where `|V|` is the number of statements judged supported by the context and `|S|` is the total number of extracted statements.
- **Answer relevance pipeline**: given `a_s(q)`, the LLM generates `n` reverse questions `q_i`; embeddings are computed for `q` and each `q_i`, and semantic similarity is measured by cosine similarity.
- **Answer relevance formula**: `AR = (1 / n) * sum_i sim(q, q_i)`, where `sim` is cosine similarity between question embeddings.
- **Context relevance pipeline**: given `q` and `c(q)`, the LLM extracts the subset of sentences `S_ext` from the context that are sufficient to answer the question, or returns "Insufficient Information".
- **Context relevance formula**: `CR = (# extracted sentences) / (# total sentences in c(q))`, so longer but unfocused contexts are penalized.
- **LLM backends in experiments**: prompt evaluations use `gpt-3.5-turbo-16k`; question embedding similarity uses `text-embedding-ada-002`.
- **Benchmark construction**: the paper introduces [[wikieval]], built from `50` recent Wikipedia pages. Each instance is annotated by `2` fluent English annotators, with disagreements resolved by discussion.
- **Integrations**: the framework is implemented as [[ragas]] and exposed for common RAG stacks such as llama-index and LangChain.

## Key Results

- On [[wikieval]], RAGAS reaches `0.95` accuracy for faithfulness, `0.78` for answer relevance, and `0.70` for context relevance in pairwise agreement with human judgments.
- The `GPT Score` baseline reaches `0.72 / 0.52 / 0.63` on faithfulness, answer relevance, and context relevance respectively.
- The `GPT Ranking` baseline reaches `0.54 / 0.40 / 0.52`, substantially below RAGAS on all three dimensions.
- Human annotator agreement on WikiEval is about `95%` for faithfulness, `95%` for context relevance, and `90%` for answer relevance.
- WikiEval is built from `50` Wikipedia pages about events after the start of `2022`, explicitly targeting knowledge that may fall beyond model training cutoffs.

## Limitations

- The framework still depends on LLM-as-judge prompting, so scores can be sensitive to prompt design and to the capabilities of the specific evaluator model.
- Answer relevance and especially context relevance remain harder to estimate than faithfulness; the reported accuracies drop to `0.78` and `0.70`.
- WikiEval is relatively small (`50` source pages) and centered on recent Wikipedia events, so external validity to enterprise or domain-specific RAG settings is only partially established.
- The implementation relies on proprietary APIs (`gpt-3.5-turbo-16k`, `text-embedding-ada-002`), which complicates reproducibility and may age quickly as model APIs change.
- The paper evaluates reference-free proxies rather than end-task utility directly, so high RAGAS scores do not guarantee better user satisfaction or downstream task performance.
