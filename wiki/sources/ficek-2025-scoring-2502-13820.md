## Summary

The paper introduces a recipe for converting execution-based coding benchmarks into scoring and ranking benchmarks for synthetic verifiers, i.e. methods that must estimate solution quality without relying only on the benchmark's released oracle tests. The authors generate diverse correct and incorrect solutions, score them with predefined tests, deduplicate by score and runtime, and select evenly spaced solutions to form ranked candidate sets. Using this procedure, they release HE-R, HE-R+, MBPP-R, and MBPP-R+ and evaluate standard LLMs, reasoning models, and code reward models on synthetic test generation and reward scoring. The main empirical finding is that reasoning models generate markedly stronger synthetic test suites than standard models, and that scaling the number of generated test cases improves verifier quality further.

## Problem & Motivation

Execution-based reinforcement learning and inference-time selection for code increasingly depend on verification signals, but most coding datasets expose only a small number of predefined tests. Prior work on generated unit tests or reward models shows these synthetic verifiers can help, yet existing benchmarks mostly measure raw test generation quality or pairwise preference accuracy rather than whether a verifier can recover the true ranking of candidate solutions. This paper addresses that gap by building benchmarks where multiple candidate programs for each problem have oracle-derived fractional correctness scores, enabling direct evaluation of whether synthetic verification methods can score solutions accurately and identify the best one.

## Method

- **Benchmark transformation pipeline**: starting from a coding dataset with predefined tests, the authors generate many candidate solutions per problem using multiple prompts, seeds, and sampling settings, including prompts aimed at both correct and intentionally incorrect code.
- **Score computation and filtering**: each solution is executed against the benchmark's predefined tests and assigned a fractional pass score `s_i in [0, 1]`. Solutions with identical scores are deduplicated, with lower average execution time used as the tie-breaker, and non-assertion zero-score failures are removed.
- **Even-coverage selection**: for a desired solution count `k`, the effective count is `k' = min(n, k)`. Let `m` be either the smallest positive score below `0.1` or the minimum observed score otherwise; targets are `T_i = 1 - (i / k') (1 - m)` for `i = 1, ..., k' - 1`, and the selected solution set minimizes `|s - T_i|` to approximate evenly spaced correctness levels.
- **Released benchmarks**: the paper instantiates the recipe on HumanEval / HumanEval+ and MBPP / MBPP+, producing HE-R, HE-R+, MBPP-R, and MBPP-R+. For the `+` variants the target is `k = 5`; for the base variants `k` ranges from `2` to `5` because fewer oracle tests limit score resolution.
- **Solution generation details**: GPT-4o (`2024-11-20`) is used to generate candidate solutions. When the automatic pipeline fails to find enough uniquely scored solutions, the authors manually annotate `10` HE-R+ solutions and `15` MBPP-R+ solutions.
- **Synthetic test generation setup**: verifier models generate `10` tests per problem for the main comparison using nucleus sampling with temperature `1.0`. Each generated test is executed with the candidate solution under a timeout of `3` seconds.
- **Reward-model evaluation**: reward models score each candidate solution directly; scores are normalized per problem using the observed maximum and minimum model scores. Nemotron4-340B-Reward is evaluated using only its correctness attribute.
- **Metrics**: synthetic verifiers are measured by Top-1 Accuracy, Spearman's `rho`, Bottom-1 Accuracy, and Mean Absolute Error (MAE), comparing predicted scores or rankings against the oracle ranking induced by predefined tests.

## Key Results

- The transformation yields four datasets with the following statistics: HE-R `164` problems / `9.6` average tests / `742` synthetic solutions, HE-R+ `164` / `764.1` / `820`, MBPP-R `974` / `3.0` / `3249`, and MBPP-R+ `378` / `108.5` / `820`.
- On the main `10`-test setting, `o3-mini` is the strongest synthetic test generator with HE-R+ Top-1 `88.2`, Spearman `0.85`, Bottom-1 `84.0`, MAE `0.18`; on MBPP-R+ it reaches Top-1 `79.9`, Spearman `0.78`, Bottom-1 `80.1`, MAE `0.20`.
- Reasoning models outperform comparable standard models on synthetic test generation: DeepSeek-R1 reaches HE-R+ Top-1 `83.8` and MBPP-R+ Top-1 `77.5`, beating Qwen2.5-32B-Instruct at `74.9` and `68.8` respectively.
- Scaling the number of generated tests improves verification quality: DeepSeek-R1 rises from HE-R+ Top-1 `83.8` at `10` tests to `91.6` at `25` tests, while MBPP-R+ reaches `80.3` at `25` tests.
- Reward models are competitive but generally weaker than the best reasoning generators. For example, AceCodeRM-32B scores HE-R+ Top-1 `77.4`, and Nemotron4-340B-Reward scores MBPP-R+ Top-1 `75.1`, both below the strongest reasoning systems.
- Saturation analysis shows benchmark suitability depends on test-count granularity: the lower bound of the `95%` confidence interval for Spearman exceeds `0.90` at `k = 6` on HE+ and `k = 5` on MBPP+.

## Limitations

- The transformed verifier benchmarks inherit the reliability of the original benchmark tests; weak or incomplete oracle test suites directly weaken the benchmark's gold scores.
- The base HE-R and MBPP-R variants have relatively few predefined tests, so they provide coarser score resolution and fewer uniquely ranked solutions than the `+` variants.
- Candidate solution generation uses GPT-4o alone, so the released benchmark may still contain self-evaluation bias from the same model family appearing in downstream verifier comparisons.
- The paper's main synthetic test-generation comparison uses `10` tests partly because reasoning-model API costs and context limits made broader scaling expensive.
- Reward-model analysis is limited to a small set of available code reward models and does not fully resolve whether their weaker rankings reflect model capacity, training objective mismatch, or benchmark bias.
