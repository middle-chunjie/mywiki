## Summary

RAG-Star proposes a deliberative retrieval-augmented reasoning framework for multi-hop question answering that separates internal reasoning from external evidence use. Instead of feeding retrieved passages directly into every generation step, the method lets a policy model plan sub-queries and answers with Monte Carlo Tree Search, then uses retrieval-augmented verification to score whether each planned query is plausible and whether each intermediate answer aligns with retrieved evidence. This design aims to reduce knowledge conflicts between parametric memory and retrieved documents while still using retrieval to steer search. Across HotpotQA, 2WikiMultihopQA, MusiQue, and StrategyQA, RAG-Star consistently beats standard RAG, iterative RAG, and other reasoning baselines for both GPT-4o and Llama-3.1-8B-Instruct.

## Problem & Motivation

The paper targets complex multi-hop reasoning, where large language models often fail when a problem requires several intermediate inference steps across documents. Prior chain-of-thought and tree-search methods mostly rely on internal model knowledge, while standard retrieval-augmented generation injects retrieved documents directly into generation and can create conflicts between external evidence and the model's parametric beliefs. The authors therefore ask whether retrieval can be repurposed as guidance for deliberative search rather than as direct context stuffing, so that LLMs can explore reasoning paths with internal knowledge but still receive external factual feedback.

## Method

- **Search state and selection**: RAG-Star represents each node as `s_t = [q_t, a_t, N(s_t), V(s_t)]` and uses UCT to choose branches, with `UCT(s_t) = V(s_t) + w * sqrt(ln N(p) / N(s_t))`. The exploration weight is set to `w = 0.2`.
- **Tree-based planning**: Starting from the input question `q_0`, the policy model builds a reasoning history `H = {q_0; <q_1, a_1>; ...; <q_t, a_t>}` and samples the next sub-query `q_(t+1)` from that path instead of retrieving documents first.
- **Answer deduction from internal knowledge**: For each sampled sub-query, the policy model generates an answer with `a_(t+1) = pi_theta(H, q_(t+1))`. External documents are intentionally excluded at this stage to avoid interference with the model's inherent reasoning trajectory.
- **Repeated expansion**: Each selected node expands by sampling `m_q = 3` candidate sub-queries at a time. Search runs up to `50` simulations with a maximum depth of `6` layers.
- **Retrieval-augmented verification**: For each sub-query, the system retrieves top-`K = 5` documents and assigns an answer-aware score `r_a` plus a query-aware score `r_q`. The final node reward is `r = r_a * r_q`.
- **Answer-aware reward and refinement**: The answer-aware reward takes values `r_a in {1, 2, 3}` depending on whether the generated answer is unverifiable, conflicting, or aligned with retrieved evidence. Conflict cases can trigger answer refinement using retrieved documents before subsequent reasoning continues.
- **Query-aware reward**: The query-aware reward is binary, `r_q in {0, 1}`, and checks whether the planned sub-query is logically consistent with the existing reasoning path `H`.
- **Backpropagation and reward-model training**: Node statistics are updated with `N_new(s_t) = N_old(s_t) + 1` and `V_new(s_t) = (V_old(s_t) * N_old(s_t) + r) / N_new(s_t)`. The reward model is either GPT-4o-mini or a distilled Llama-3.1-8B-Instruct model trained on synthetic step-level supervision.
- **Implementation details**: Retrieval uses BGE-large-en-v1.5 embeddings with FAISS indexing; answer generation uses temperature `0.9` and top-`p = 1.0`, while sub-query generation uses temperature `1.0` and top-`p = 1.0`.

## Key Results

- On **Llama-3.1-8B-Instruct**, RAG-Star with GPT-based reward modeling reaches `46.0/49.0/60.0` (EM/CEM/F1) on HotpotQA, `38.0/43.0/46.8` on 2WikiMultihopQA, `22.2/27.0/30.7` on MusiQue, and `67.6/69.0/67.6` on StrategyQA.
- On **GPT-4o**, RAG-Star with GPT-based reward modeling reaches `48.0/57.0/68.6` on HotpotQA, `48.0/63.0/61.7` on 2WikiMultihopQA, `29.0/40.0/43.5` on MusiQue, and `60.0/81.0/60.0` on StrategyQA.
- Relative to the authors' stated baselines, RAG-Star improves average performance by up to `18.98%` for Llama-3.1-8B-Instruct and `16.19%` for GPT-4o.
- In the StrategyQA ablation, full RAG-Star scores `84.0` Cover EM / `68.3` F1 with GPT-4o and `75.0` / `73.3` with Llama-3.1-8B, outperforming variants without retrieval, answer scoring, query scoring, or refinement.
- Scaling maximum simulations from `10` to `60` improves performance initially but eventually saturates, indicating diminishing returns from extra test-time search once the policy model's inherent knowledge becomes the bottleneck.

## Limitations

- The framework is computationally expensive because tree search requires many simulations and repeated reward evaluations.
- Experiments cover only four multi-hop QA datasets with `100` validation examples each, so generalization to other reasoning domains is not established.
- The search algorithm is restricted to Monte Carlo Tree Search; the paper does not test whether other search strategies would work better.
- Performance depends heavily on reward-model quality, and the paper acknowledges that stronger supervision or better fine-tuning strategies may be needed.
- Retrieved evidence is used only for verification and refinement, so the method may still struggle when the policy model cannot propose useful sub-queries from internal knowledge alone.
