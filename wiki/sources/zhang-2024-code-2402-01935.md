## Summary

⚠ Possibly outdated: published 2024; re-verify against recent literature in code embedding.

The paper introduces CodeSage, a family of encoder-only code representation models (130M, 356M, 1.3B parameters) trained on 237M source files from The Stack across nine programming languages. Training proceeds in two stages: Stage I jointly optimizes identifier deobfuscation (DOBF) and full-mask MLM without the 80-10-10 rule; Stage II applies bimodal contrastive learning with hard positives (function body stripped of signature and return) and distance-weighted hard negatives. CodeSage-large achieves 38.51% average MAP on zero-shot Code2Code search (vs. 27.33% for OpenAI Ada-002) and strong gains on NL2Code retrieval and code classification tasks, showing that large-scale pretraining with carefully designed objectives, not just scale alone, drives performance.

## Problem & Motivation

Existing code embedding models (CodeBERT, GraphCodeBERT, UnixCoder) cap at ~125M parameters and train on only a few million examples from CodeSearchNet, leaving the vast corpora now available for code LLMs largely untapped for representation learning. The standard 80-10-10 MLM masking from BERT degrades code quality because replacing tokens with random vocabulary entries disrupts both the syntax and the semantics of source code. Dropout-based unimodal contrastive learning collapses on large datasets (training loss saturates), motivating a bimodal alternative. The gap between code embedding quality and code generation quality motivates building off-the-shelf, zero-shot-capable encoders that benefit from scale.

## Method

**Stage I — Token-level Denoising**

- **DOBF (Identifier Deobfuscation)**: Code is obfuscated by replacing class names (`c_i`), function names (`f_i`), variable names (`v_i`) with special tokens. The model learns to recover originals, requiring understanding of semantics, data flow, and structure. NL tokens (docstrings, comments) are left unmasked, encouraging NL–PL alignment.
- **Random Masking (Full Mask)**: Standard BERT MLM at `15%` rate, but with 100% mask-token replacement (no random token substitution). The 80-10-10 rule is dropped because random token substitution corrupts code structure and semantics far more severely than text.
- **Mixing**: Each training example independently draws either DOBF or random masking with equal `p = 0.5` probability. Ablations show parallel mixing beats sequential (first MLM then DOBF) across all task categories.
- **Data**: `237M` code files from The Stack; max sequence length `1024` with block attention and concatenation.
- **Models**: 12-layer / `d=768` (small), 24-layer / `d=1024` (base), 24-layer / `d=2048` (large); all initialized from RoBERTa-style pretraining setup with SentencePiece tokenizer.

**Stage II — Bimodal Contrastive Learning**

- **Positives**: (text, function) pairs where `text` is the first sentence of the docstring and `function` is the body with signature and return statement removed. Removing signature mitigates shortcut learning from lexical overlap between function name and docstring.
- **Hard Negatives**: Distance-weighted in-batch negatives. For anchor `h_i` and in-batch negative `h_k`, weight `γ_i^k = exp(h_i ⊙ h_k / τ) / Σ_{j≠(i,i+,k)} exp(h_i ⊙ h_j / τ)`. This assigns larger gradient weight to negatives closer to the anchor.
- **Loss**: Symmetric InfoNCE: `L_CL(h_i, h_i+) = −(log [exp(sim/τ) / Σ γ·exp] + symmetric term)`, temperature `τ = 0.05`, cosine similarity.
- **Batch size**: `N = 8192` pairs; performance monotonically increases with batch size in experiments.
- **Data**: `75M` bimodal (text, code) pairs derived from The Stack docstrings across nine languages.

## Key Results

- **Code2Code search (zero-shot MAP)**: CodeSage-small 26.08%, CodeSage-base 32.95%, CodeSage-large **38.51%**. Beats OpenAI Ada-002 (27.33%) at base size; large model +40.91% relative improvement.
- **NL2Code (zero-shot MRR)**: CodeSage-small CoSQA 49.92% (+12.86% rel. over Ada-002), AdvTest 41.28% (+8.4% rel.); CodeSage-large AdvTest 52.67% (best overall).
- **Code classification (frozen head F1 macro)**: CodeSage-large Complexity 90.32% vs. UnixCoder 76.45%, RunTime error 24.42% vs. OpenAI 20.84%.
- **Masking ablation**: Full-mask MLM consistently outperforms 80-10-10 across all tasks and sizes; degradation from 80-10-10 is more severe for code than for text.
- **DOBF vs. MLM ablation**: DOBF stronger on classification and NL2Code; MLM stronger on Code2Code search; parallel mixture (P) consistently best.
- **Scaling**: Stage-I-then-Stage-II framework enables performance to scale with model size; contrastive learning alone from scratch does not scale.

## Limitations

- Evaluated only on nine languages from The Stack; very low-resource or domain-specific languages (e.g., SQL, Solidity) are not covered.
- Zero-shot evaluation paradigm; fully finetuned classification results (Appendix B.3) show some reversals (e.g., code defect detection where CodeSage underperforms Ada-002 in the frozen setting).
- DOBF requires tree-sitter parsing plus alignment between tree-sitter and SentencePiece tokenizations, adding engineering complexity not well-documented in the main paper.
- Hard negatives are approximated by in-batch distance; true semantic-equivalence annotation is unavailable, so false negatives may exist despite large batch sizes.
- Bimodal pretraining depends on docstring quality and availability; languages with sparse or low-quality docstrings receive fewer Stage II training pairs.
