## Summary

This paper proposes ASG-SI, a self-improvement framework for agentic large language models that treats capability growth as audited promotion of reusable skills rather than opaque parameter drift. The system logs trajectories with tool calls and memory operations, compiles successful subsequences into canonical skills with explicit interfaces, and promotes them only after replay-based verification produces evidence bundles sufficient to reconstruct shaped rewards. The design combines verifier-backed reward decomposition, synthetic experience for stress testing and curriculum expansion, and bounded continual memory to support long-horizon interaction without uncontrolled context growth. The contribution is primarily architectural: the paper offers a reference implementation and evaluation plan, but does not yet report benchmark gains or ablation results.

## Problem & Motivation

The paper targets a deployment setting where an LLM-based agent keeps improving while interacting with tools, external state, and memory under partial observability. In that setting, optimizing only final task outcomes creates three coupled risks: sparse and delayed rewards make learning unstable, intermediate violations such as malformed tool use or broken contracts are not directly measured, and the resulting behavioral drift is hard to audit or reproduce. The authors therefore argue that the operational unit of improvement should be a reusable skill artifact with explicit contracts and replayable evidence, not an uninspected checkpoint update.

## Method

- **System decomposition**: ASG-SI factors the runtime into `4` subsystems: a policy runtime, a bounded memory subsystem, a skill compiler, and a verifier-auditor connected by logs and registries.
- **Trajectory logging**: online interaction is recorded as trajectories `τ = (o_1, a_1, ..., o_T, a_T)` augmented with tool transcripts, intermediate artifacts, and memory updates so later compilation and replay do not depend on hidden model state.
- **Skill compilation**: reusable subsequences are normalized into candidate skills `s = (I, P, C)` with explicit interface `I`, canonical program or template `P`, and checkable contracts `C` over preconditions and postconditions.
- **Reward decomposition**: the paper organizes shaped reward schematically as `R = R_tool + R_outcome + R_reuse + R_comp + R_mem`, where the terms score tool validity, outcome verification, verified skill reuse, composition integrity, and memory discipline. Progressive scheduling shifts weight from structural validity toward correctness and efficiency; exact coefficients are left unspecified.
- **Verifier-backed promotion**: a candidate is promoted only if replay under a controlled harness passes held-out tasks, contract checks, and perturbation tests, yielding an evidence bundle `E = {schema logs, test outcomes, hashes, verifier version}` that also supports reward reconstruction.
- **Skill graph runtime**: promoted skills become nodes in a directed multigraph, while edges encode composition constraints, dependency ordering, and guarded fallbacks so failures can be localized at the node or edge level.
- **Evidence integrity**: each bundle is treated as a tamper-evident manifest with canonical field ordering, artifact or transcript hashes, and explicit versioning for the verifier, tool schemas, and environment harness.
- **Experience synthesis**: synthetic trajectories are used to stress-test interfaces, expand curriculum coverage, and probe rare edge cases, but promotion is periodically re-grounded with real-environment rollouts to control simulation drift.
- **Continual memory control**: memory operations are part of the auditable action space, and the design explicitly enforces bounded context growth rather than unconstrained transcript accumulation.
- **Reference implementation**: the prototype is described as a single runnable file, `asg_si_demo.py`, with deterministic exact-match verification, schema-checked pure-function tools, append-only JSON audit traces, and a bounded-memory note store.

## Key Results

- The paper specifies a `4`-subsystem architecture and a `5`-component reward decomposition, giving a concrete operational breakdown of how audited self-improvement is supposed to work.
- The reference implementation is described as `1` runnable prototype file plus append-only JSON audit traces, making replay and evidence inspection explicit rather than implicit.
- The proposed evaluation covers `3` axes: capability growth, retention under continual task streams, and constraint adherence under optimization pressure.
- The authors define `1` composite metric, audited improvement rate, combining newly promoted verified skills with their held-out reuse rate.
- Empirical evidence remains limited: the manuscript reports `0` benchmark tables, `0` head-to-head baseline comparisons, and `0` ablation studies, so the contribution is currently architectural and procedural rather than quantitatively validated.

## Limitations

- The paper does not provide controlled experiments, so the claimed advantages over prior agent-training pipelines remain hypotheses rather than demonstrated gains.
- Reward decomposition is conceptually clear, but weighting schedules, verifier thresholds, and promotion criteria are underspecified for real deployments.
- Replay-based verification assumes deterministic or tightly controlled harnesses; noisy tools and nondeterministic environments make evidence construction harder.
- The security story depends on isolation of the verifier and evidence store. If either boundary is compromised, auditability becomes superficial.
- Evidence bundles may contain sensitive tool outputs or user data, but the paper does not work through storage minimization, retention policy, or access-control design in depth.
