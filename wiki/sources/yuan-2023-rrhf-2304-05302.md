## Summary

⚠ Possibly outdated: published 2023; re-verify against recent literature.

The paper proposes RRHF, a preference-alignment method for large language models that replaces PPO-style online reinforcement learning with offline ranking over multiple sampled responses. For each prompt, the model scores candidate responses by length-normalized conditional log-probability and is trained to rank them in the same order as human or reward-model preferences, while also maximizing likelihood on the best response. This keeps the pipeline close to supervised fine-tuning, requires only 1 to 2 models during tuning instead of PPO's 4-model stack, and can use responses from the base model, stronger external models such as ChatGPT or GPT-4, or human-written data. On Anthropic's Helpful and Harmless benchmark, RRHF matches or slightly exceeds PPO on reward-model and human evaluations, with performance strongly governed by candidate-sampling quality.

## Problem & Motivation

RLHF pipelines based on SFT, reward modeling, and PPO can align language models effectively, but PPO is brittle in practice: it requires several simultaneously loaded models, extensive hyperparameter tuning, online rollout infrastructure, and careful KL control. The authors target a simpler alignment recipe that preserves PPO-like preference optimization while reducing systems complexity and memory cost. Their key intuition is that if multiple candidate responses for the same prompt are already available, the language model can be trained to assign higher probability to better-ranked responses directly, without fitting a value model or running token-level online policy optimization.

## Method

- **Training setting**: for each query `x`, collect `k` candidate responses `y_i` from arbitrary sources `rho_i`, including the base model, later checkpoints, external LLMs such as ChatGPT or GPT-4, and human-written chosen/rejected responses.
- **Model score**: RRHF scores each response by length-normalized conditional log-probability under the trainable model `pi`, `p_i = (sum_t log P_pi(y_{i,t} | x, y_{i,<t})) / ||y_i||`.
- **Ranking objective**: align model scores with preference scores `r_i = R(x, y_i)` using `L_rank = sum_{r_i < r_j} max(0, p_i - p_j)`, so better responses should receive larger model probability than worse ones.
- **SFT-style anchor**: let `i' = argmax_i r_i` denote the highest-reward response, then add `L_ft = -sum_t log P_pi(y_{i',t} | x, y_{i',<t})` to preserve direct imitation of the best candidate.
- **Total loss**: optimize the unweighted sum `L = L_rank + L_ft`; the authors report that larger weights on `L_rank` such as `10` or `100` degraded performance in preliminary trials.
- **Sampling policy**: the main experiments use `4` model-generated rollouts plus `2` provided HH responses per query. Diverse beam search uses beam size `4`, group count `4`, diversity penalty `1.0`, temperature `0.8`; top-p sampling uses beam size `4`, `top-p = 1.0`, temperature `0.8`; outputs are capped at `128` tokens during sampling.
- **Fine-tuning hyperparameters**: RRHF is trained for `3` epochs, warms the learning rate to `2e-5` then linearly decays to `0`, uses per-GPU query batch size `1` with gradient accumulation `8` for total batch size `64`, truncates query+response length to `192` tokens, and typically runs on `8 x 80GB` Nvidia A100 GPUs for `4-6` hours without online sampling.
- **Comparison to PPO**: unlike PPO, RRHF does not estimate an advantage baseline with a value model and, in its offline form, avoids a reference-model KL term because sampling occurs before optimization rather than from the continually changing policy during training.

## Key Results

- On HH automatic evaluation, `Alpaca-RRHFSP` achieves the best average reward score `-0.96`, outperforming `Alpaca-PPO` at `-1.03`, `Alpaca` at `-1.18`, and human-written good responses at `-1.24`.
- `Alpaca-RRHFDP` reaches reward `-1.03`, matching PPO on the proxy reward metric while retaining much simpler training mechanics; iterative sampling improves further to `IP-2 = -0.96` and `IP-3 = -0.94`.
- In human evaluation on HH, `RRHFDP` beats dataset good responses `59 / 30 / 11` (win / tie / lose) and edges out PPO `27 / 48 / 25`.
- As a reward model surrogate, `Alpaca-RRHFDP` reaches `61.75%` pairwise accuracy on chosen-vs-rejected comparisons, clearly above `Alpaca-PPO` at `46.03%` and vanilla `Alpaca` at `45.13%`, though still below the proxy reward model's `68.49%`.
- Online RRHF can push proxy reward to `0.34`, but perplexity collapses to `63.78`, indicating reward hacking; adding KL regularization recovers a more reasonable trade-off with reward `-0.86` and perplexity `19.76`.
- In the paper's Wombat study, RRHF training on ChatGPT-scored response sets improves Vicuna-style pairwise scores over `Alpaca` (`616` vs `567`) and `Alpaca (ChatGPT)` (`612` vs `574`), though it still trails `ChatGPT` (`548` vs `669`).

## Limitations

- The method is highly sensitive to candidate-sampling quality; the paper explicitly concludes RRHF behaves like a best-of-`n` learner, so weak sampled responses limit final performance.
- Main experiments rely on `Dahoas/gptj-rm-static` as a proxy reward model, which may not faithfully capture human preference and can bias both optimization and evaluation.
- RRHF requires multiple responses per query, increasing memory and data-preparation cost relative to plain SFT and some PPO setups.
- Online and iterated variants can over-optimize proxy rewards and learn degenerate, reward-hacking response patterns unless additional KL control is introduced.
- The Wombat analysis still lags ChatGPT, especially on logical reasoning, so the method does not fully solve high-quality alignment or reasoning transfer.
