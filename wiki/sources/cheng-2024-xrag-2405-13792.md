## Summary

xRAG proposes a retrieval-augmented generation architecture that replaces the retrieved document text with a single projected retrieval embedding token. Instead of concatenating a roughly 175-token passage to the query, the model feeds `W(E) ⊕ Emb(q)` into a frozen LLM, where `E` is a dense document embedding and `W` is a lightweight trainable projector. The method is trained in two stages: paraphrase pretraining to align retrieval embeddings with document semantics, then context-aware instruction tuning with self-distillation from full-text RAG. Across six knowledge-intensive tasks, xRAG substantially outperforms other compression baselines, nearly matches or exceeds uncompressed RAG on average, and cuts inference cost to `1` added token with `x3.53` lower FLOPs.

## Problem & Motivation

Standard RAG improves factual coverage by appending retrieved documents to the prompt, but this makes inference substantially more expensive and can exceed context limits. Prior context-compression methods either keep large activation memories for soft prompts or preserve only a partial surface-form summary, which limits compression quality or deployability. xRAG starts from a different premise: dense retrievers already compress each document into a high-dimensional embedding, so the key problem is how to fuse that embedding into a text-only LLM without retraining the retriever or the language model.

## Method

- **Formulation**: standard RAG uses `F_phi(D ⊕ q)` where the retrieved document `D` adds `|D|` tokens to the prompt, while xRAG replaces text with a projected embedding and feeds `W(E) ⊕ Emb(q)`, reducing the added retrieval context from `|D|` tokens to `1`.
- **Projector**: xRAG uses a two-layer MLP `W` as a modality bridge from retriever embedding space to the LLM representation space. It is the only trainable module, accounting for `0.46%` of Mistral-7B parameters and `0.07%` of Mixtral-8x7B parameters.
- **Paraphrase pretraining**: for retrieval pairs `(E, D)`, the model learns to reconstruct the document with `L_nll = -Σ_i log p_phi(d_i | W(E), X_instruct, d_<i)`, using varied paraphrase prompts so the LLM aligns projected retrieval features with document semantics.
- **Context-aware instruction tuning**: the model is then tuned on about `955,338` context-question-answer examples spanning reading comprehension, open-domain QA, and summarization, replacing textual context with `E_context = SE_theta(X_context)`.
- **Self-distillation**: xRAG optimizes `L = L_nll + α L_kl` with `α = 2.0`, where `L_kl = D_KL(p_phi(X_answer | X_context, ·) || p_phi(X_answer | W(E_context), ·))`. The teacher is full-text RAG and the student is xRAG.
- **Retrieval setup**: experiments use the December 2021 Wikipedia dump, split into about `37M` passages with average length `180` tokens. Retrieval uses top-1 passages from ColBERT-v2, then re-encodes them with SFR sentence embeddings for xRAG.
- **Backbones and data**: the paper evaluates instruction-tuned `[[mistral-7b-instruct-v0.2]]` and Mixtral-8x7B backbones. The instruction-tuning mix contains `488,344` reading-comprehension examples, `385,173` open-domain QA examples, and `81,821` summarization examples.
- **Training hyperparameters**: paraphrase pretraining uses AdamW, `lr = 6e-3`, warmup ratio `0.03`, `1` epoch, batch size `12`, gradient accumulation `4`, max sequence length `336`, flash attention enabled, and up to `2,000,000` samples on `8` A100 GPUs. Instruction tuning uses AdamW, `lr = 2e-5`, warmup ratio `0.03`, `1` epoch, batch size `4`, gradient accumulation `2`, max sequence length `1024`, KL temperature `1.0`, and `8` GPUs.

## Key Results

- On Mistral-7B, xRAG reaches average score `42.46`, up from `36.54` without retrieval (`+16.2%`) and close to uncompressed RAG at `43.63`, while using `1` retrieved token instead of average document length `175.1`.
- On Mixtral-8x7B, xRAG reaches `46.91`, exceeding uncompressed RAG at `46.22` and improving over the no-retrieval baseline `42.76` by `+9.7%`, again with a single retrieval token.
- xRAG clearly beats other compression baselines: on Mistral-7B it scores `42.46` versus LLMLingua `38.01` / `37.48` and TF-IDF `37.49`; on Mixtral-8x7B it scores `46.91` versus LLMLingua `42.32` / `44.17` and TF-IDF `43.41`.
- Efficiency improves substantially: averaged over four datasets, xRAG reduces CUDA time by `x1.64` and GFLOPs by `x3.53`; on FactKG alone, it achieves `x2.01` lower CUDA time and `x3.63` lower GFLOPs than RAG.
- Ablations show both training stages matter. For Mistral-7B, average performance drops from `44.58` to `37.87` without finetuning and `38.95` without pretraining; removing self-distillation lowers average performance to `41.49`.
- Robustness improves under noisy retrieval. The reported resilience rate is `82.3%` for Mistral-7B xRAG and `84.9%` for Mixtral-8x7B xRAG, higher than standard RAG in the paper's fine-grained analysis.

## Limitations

- xRAG still trails full-text RAG on reasoning-heavy tasks that require explicit document composition: for Mistral-7B, HotpotQA is `34.05` versus `38.79` and FactKG is `63.08` versus `67.76`; similar gaps remain on Mixtral-8x7B.
- The method uses only the top-1 retrieved document and compresses it into a single token, which limits information capacity for multi-hop reasoning and fine-grained evidence tracking.
- Performance depends on retrieval and embedding quality; the appendix shows stronger sentence embedding models consistently improve downstream results.
- The evaluation uses a fixed December 2021 Wikipedia corpus and frozen LLM backbones, so the paper does not address document freshness, retriever updating, or multi-document fusion at deployment time.
- The paper explicitly lists future work on reasoning over embeddings, handling multiple documents, and combining xRAG with multi-vector retrieval, indicating these remain open limitations.
