## Summary

This paper proposes RAGraph, a retrieval-augmented graph learning framework that plugs external graph memory into pre-trained graph neural networks for unseen node, graph, and link prediction tasks. The core idea is to build a toy-graph vector base from resource graphs, store time, structure, environment, and semantic keys for each master node, and retrieve the most relevant toy graphs at inference time. Retrieved hidden embeddings and task-specific output vectors are then injected into the query graph through message passing and fused with the backbone decoder. Across static and dynamic benchmarks, RAGraph consistently improves over standard GNN baselines and usually over graph prompting and graph in-context learning baselines, while also remaining effective in no-fine-tuning settings on unseen datasets.

## Problem & Motivation

The paper targets a gap between strong pre-trained GNN backbones and their weak transfer to unseen graphs, tasks, and domains. Prior graph prompting and graph in-context learning methods such as PRODIGY mainly learn an `X -> Y` mapping from fixed examples, but they do not explicitly retrieve and inject external graph evidence at inference time. The authors argue that graph generalization needs a graph-native analogue of RAG: retrieve structurally and semantically similar subgraphs, expose both features `X` and task outputs `Y`, and let the downstream query graph absorb that information without requiring task-specific retraining in every new scenario.

## Method

- **Toy graph construction**: sample a master node `v_m` from each resource-graph snapshot using inverse importance sampling with `I(v) = alpha * PR(v) + (1 - alpha) * DC(v)`, `I'(v) = 1 / (I(v) + epsilon)`, and `p_i = I'(v_i) / sum_j I'(v_j)`, so low-importance long-tail nodes are sampled more often.
- **Toy graph definition**: build a `k`-hop ego net `G^e_t(v_m)` around each sampled node, then create augmented toy graphs with count `n_aug(G^e_t(v_m)) = floor(K * I_bar_prime(G^e_t(v_m)))`, where `I_bar_prime` is average reversed importance and `K` is a scaling constant.
- **Augmentation operators**: use node dropout with probability `1 - p_i`, Gaussian feature noise `X'(v_i) = X(v_i) + N(0, sigma^2)`, node interpolation `X'(v_new) = lambda * X(v_i) + (1 - lambda) * X(v_j)`, and edge rewiring with probability `(p_i + p_j) / 2`.
- **Key-value memory**: for each toy graph, store timestamp `t`, environment key `N(v_m^t)`, structure key `s_m^t` from position-aware distance-to-centroid encoding, and semantic key `h_m^t`; store node hidden embeddings `{h_i^t}` and task-specific outputs `{o_i^t}` as values. The implementation uses [[faiss]] for storage and retrieval.
- **Retrieval scoring**: rank toy graphs by `S(v_c, v_m) = w_1 S_time + w_2 S_structure + w_3 S_environment + w_4 S_semantic`, then retrieve `TopK` toy graphs. Appendix details define `S_time` as exponential decay over timestamps, `S_environment` as Jaccard overlap of neighborhoods, and the structure/semantic terms as cosine similarities.
- **Knowledge injection propagation**: first run toy-graph intra propagation `z_m = GNN({z_i | v_i in N(v_m)})`, then query-toy inter propagation with `h_c = GNN({h_i | v_i in N(v_c) union {v_m}})` and `o_c = GNN({o_i | v_i in {v_m}})`. The propagation can be parameter-free or learnable.
- **Fusion layer**: combine retrieved task outputs and decoder outputs as `o_hat_c = gamma * o_c + (1 - gamma) * DECODER(h_c)`, then use the fused vector for node classification, graph classification, or link prediction within the paper's unified task formulation.
- **Task setup and hyperparameters**: node/graph tasks follow `5`-shot evaluation; dynamic recommendation uses `Recall@20` and `nDCG@20`. Reported core settings are `k = 2`, `TopK = 5`, `alpha = 0.5`, `lambda = 0.5`, `K = 3`, `w_1 = w_2 = w_3 = 0.05`, `w_4 = 0.85`, with task-specific `gamma` values such as `0.8` for ENZYMES node classification and `0.5-0.8` across graph-classification datasets.
- **Robustness tuning**: noise-based graph prompt tuning adds inner-toy-graph noise by injecting irrelevant nodes during construction and toy-graph noise by appending bottom-`K` retrieved toy graphs during training, but not during inference.

## Key Results

- **Node classification**: on PROTEINS (5-shot), `RAGRAPH/NFT` reaches `59.83 +/- 0.40`, above the best non-RAG baseline `53.26 +/- 6.42` (`PRODIGY/FT`) by `6.57` points; on ENZYMES, `RAGRAPH/NFT` reaches `76.23 +/- 1.63`, beating `57.98 +/- 12.37` by `18.25` points.
- **Graph classification**: best RAGraph variants achieve `62.33 +/- 2.52` on PROTEINS, `76.60 +/- 2.30` on COX2, `49.17 +/- 4.64` on ENZYMES, and `77.53 +/- 5.26` on BZR; the COX2 score exceeds the best listed baseline (`67.87 +/- 3.39`, GCN) by `8.73` points.
- **Link prediction**: `RAGRAPH/NFT` gets `24.89 +/- 2.01` Recall and `24.51 +/- 1.44` nDCG on TAOBAO; `RAGRAPH/FT` gets `34.27 +/- 3.93` Recall and `24.82 +/- 2.69` nDCG on KOUBEI; `RAGRAPH/FT` gets `18.69 +/- 7.45` Recall and `9.09 +/- 3.89` nDCG on AMAZON.
- **Generalization without fine-tuning**: `RAGRAPH/NF` already outperforms `Vanilla/NF` and usually `PRODIGY/NF`, e.g. ENZYMES node classification improves from `48.81 +/- 1.88` to `75.92 +/- 1.72`, showing strong transfer to unseen datasets.
- **Qualitative case study**: on an ENZYMES node from class 3, GraphPrompt predicts `[0.28, 0.34, 0.38]`, while the retrieved task-output path yields normalized `[0, 0.08, 0.92]`; after fusion with `gamma = 0.5`, the final normalized vector becomes `[0.157, 0.170, 0.673]`.

## Limitations

- The method currently retrieves only subgraphs; the paper explicitly leaves richer graph-structured memory such as nodes, edges, and trees to future work.
- Performance depends on hand-designed retrieval signals and hyperparameters such as `k`, `TopK`, and similarity weights; the hyperparameter study shows both overly large hops and overly large retrieval sets can hurt accuracy.
- The framework assumes access to a pre-trained GNN backbone, a maintained toy-graph database, and enough resource graphs to support retrieval, which adds engineering and storage overhead beyond standard prompting baselines.
- Experiments cover seven benchmark datasets, but the paper does not evaluate text-rich graphs, larger graph foundation models, or stronger post-2024 retrieval-augmented graph baselines.
