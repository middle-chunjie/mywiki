## Summary

⚠ Possibly outdated: published 2023; re-verify against recent literature.

The paper argues that vanilla [[infonce-loss]] is suboptimal for [[code-search]] fine-tuning because it treats all in-batch negatives equally, even when some are duplicates or partially relevant. It proposes [[soft-infonce]], which inserts a learned weight `w_ij` into the denominator so that less negative samples are pushed away less aggressively while preserving the total negative mass. The weights are estimated with BM25, SimCSE, or a trained retrieval model, then normalized with softmax. The authors provide a theoretical link to [[kl-divergence]] control over negative-similarity distributions and to reduced bias in [[mutual-information]] estimation via [[importance-sampling]]. On CodeSearchNet, the method consistently improves CodeBERT, GraphCodeBERT, and UniXCoder by roughly 1-2 MRR points with limited time overhead.

## Problem & Motivation

Contrastive fine-tuning is central to neural code retrieval, but standard InfoNCE assumes that every non-matching in-batch code snippet is equally negative. The paper argues this is wrong for large code corpora because duplicates create false negatives and semantically related implementations vary in how negative they really are. In retrieval, a query about quick sort should rank bubble sort above unrelated file I/O, but vanilla InfoNCE cannot encode that graded relevance. The authors therefore target a fine-tuning objective that respects continuous negative relevance instead of binary exclusion.

## Method

- The model uses a shared [[siamese-network]] encoder `g` to map code `x_i` and query `y_i` into a joint space, with standard query-code similarity computed by dot product `q_i · c_j`.
- Vanilla InfoNCE is rewritten as `L = -E log( exp(q_i · c_i) / (exp(q_i · c_i) + Σ_{j≠i} exp(q_i · c_j)) )`, exposing the assumption that all negatives contribute equally.
- [[soft-infonce]] inserts weights into the denominator: `L = -(1/N) Σ_i log( exp(q_i · c_i) / (exp(q_i · c_i) + Σ_{j≠i} w_ij exp(q_i · c_j)) )`.
- Negative-pair weights are defined as `w_ij = (β - α · sim_ij) / (β - (α/(N-1)) Σ_{j≠i} sim_ij)`, where `sim_ij ∈ [0,1]` and `Σ_{j≠i} sim_ij = 1`; this keeps `Σ_{j≠i} w_ij = N - 1`.
- The gradient on each negative logit is proportional to `w_ij`, so less negative samples get smaller repulsive updates while harder negatives retain larger force.
- The paper studies three similarity estimators for `sim_ij`: [[bm25]] over query-code lexical overlap, [[simcse]] over natural-language query similarity, and a trained retrieval model initialized from each backbone.
- For neural estimators, pre-trained parameters are frozen during Soft-InfoNCE training; estimated similarities are normalized with a softmax temperature `t`.
- Experimental settings include `30` epochs, seeds `1234/12345/123456`, and a single GeForce RTX A6000 GPU. The best reported hyperparameters are `α = 1.3, β = 0.7` for SimCSE and trained-model estimation, while BM25 uses `α = 1.5, β = 0.5`; all weights are clamped to be at least `0.1`.
- Theoretical analysis shows the uniformity term upper-bounds a combination of negative log-probability and [[kl-divergence]] toward the target similarity distribution; the mutual-information argument interprets weighting as [[importance-sampling]] over the negative pool.

## Key Results

- On CodeBERT, average MRR rises from `0.648/0.636/0.663/0.594/0.615/0.878` with InfoNCE to `0.682/0.675/0.682/0.615/0.631/0.898` on Ruby/Python/Java/JavaScript/PHP/Go using trained-model Soft-InfoNCE.
- On GraphCodeBERT, InfoNCE scores `0.705/0.690/0.691/0.647/0.648/0.896`; Soft-InfoNCE with SimCSE reaches `0.721/0.700/0.702/0.656/0.657/0.894`, with the strongest gain on Python from `0.690` to `0.700`.
- On UniXCoder, InfoNCE gives `0.740/0.720/0.726/0.684/0.676/0.915`; Soft-InfoNCE improves this to `0.753/0.728/0.733/0.693/0.684/0.916` with BM25 and to `0.753/0.726/0.731/0.699/0.684/0.913` with SimCSE.
- Against alternative objectives, Soft-InfoNCE achieves overall MRR `0.687/0.722/0.751` on CodeBERT/GraphCodeBERT/UniXCoder, outperforming BCE (`0.641/0.679/0.686`), weighted InfoNCE (`0.639/0.674/0.714`), and KL regularization (`0.648/0.688/0.716`).
- On GraphCodeBERT over CSN-Python, false-negative cancellation baselines top out at MRR `0.689`, while Soft-InfoNCE reaches `0.700`.
- Time overhead is limited: per-batch time rises from `0.75s` for InfoNCE to `0.77s` with BM25 Soft-InfoNCE, `0.81s` with SimCSE, and `0.98s` with trained-model estimation.

## Limitations

- The paper evaluates only three weight estimators and explicitly leaves open whether better or cheaper `sim_ij` estimation methods exist.
- Soft-InfoNCE adds an extra similarity-estimation stage, so it is slower than vanilla InfoNCE even if the overhead is moderate.
- The benchmark labels are still binary, so the claim about modeling graded negative relevance is only validated indirectly through downstream retrieval gains rather than direct graded-relevance supervision.
