## Summary

ReTool studies how to train large language models to use a code interpreter strategically rather than merely imitating tool-augmented traces. Starting from a cold-start supervised fine-tuning stage on automatically converted code-integrated reasoning data, the method then performs PPO-based reinforcement learning with interleaved real-time code execution inside the rollout loop. The model emits natural language, code blocks, and interpreter feedback in a single hybrid trajectory, while learning only from an outcome reward on the final answer. On AIME, this design produces large gains over text-only RL: the Qwen2.5-32B-Instruct variant reaches 67.0% on AIME2024 and 49.3% on AIME2025 after only 400 RL steps, and the stronger DeepSeek-R1-Distill-Qwen-32B variant reaches 72.5% and 54.3%, respectively.

## Problem & Motivation

Recent reasoning LLMs trained with reinforcement learning are strong at long-form textual chain-of-thought, but they remain weak on tasks that require exact symbolic computation, geometric verification, or concise equation solving. Code interpreters can supply precise execution, but prompting or supervised imitation alone does not teach a model when to call the tool, how to react to failures, or how to integrate execution results into subsequent reasoning. ReTool addresses this gap by treating tool use itself as a policy-learning problem: the model must discover effective invocation timing, code complexity, and recovery strategies from outcome feedback rather than handcrafted tool heuristics.

## Method

- **Two-stage training pipeline**: construct a code-integrated cold-start dataset `D_CI` from filtered mathematical reasoning data `D_init`, then run RL on top of the SFT-initialized model.
- **Automatic data conversion**: replace calculation-heavy textual spans with executable code snippets plus interpreter outputs, then apply format verification and final-answer verification before SFT.
- **Policy optimization**: optimize a PPO objective over hybrid trajectories with interpreter-conditioned rollout probabilities `pi_theta(o_t | q, o_<t; CI)`, using clipped importance ratios around `1 ± epsilon`.
- **Outcome-only reward**: final-answer reward is binary, `R(a, a_hat) = 1` if `is_equivalent(a, a_hat)` else `-1`, intentionally avoiding extra executability rewards to reduce reward hacking.
- **Interleaved execution rollout**: the model writes reasoning text and code delimited by `<code>...</code>`; when `</code>` appears, execution pauses, the sandbox runs the snippet, and feedback is injected via `<interpreter>...</interpreter>` before generation resumes.
- **Hybrid trajectory format**: each rollout is a sequence like `t_1 ⊕ c_1 ⊕ f_1 ⊕ ... ⊕ o`, enabling the policy to condition later reasoning on execution success or error messages.
- **Training details**: SFT runs for `2` epochs; RL uses PPO in VeRL with AdamW, learning rate `1e-6`, maximum sequence length `16384`, mini-batch size `512`, and KL coefficient `0.0`, using Qwen2.5-32B-Instruct as the main backbone.
- **Loss masking**: tokens inside `<interpreter>...</interpreter>` are masked out of the loss, which isolates external sandbox outputs from gradient computation.
- **Efficiency engineering**: KV-cache reuse stores the prefix cache before each execution boundary and appends only the interpreter-feedback segment, lowering rollout memory cost.
- **Sandbox system**: an asynchronous worker-pool sandbox executes code in parallel, improving throughput and preventing slow threads from dominating RL wall-clock time.

## Key Results

- ReTool on `Qwen2.5-32B-Instruct` reaches `67.0%` pass@1 on AIME2024 and `49.3%` on AIME2025 after only `400` RL steps.
- The text-based RL baseline on the same backbone reaches `40.0%` on AIME2024 and `36.7%` on AIME2025, and the paper states it uses `1080` training steps.
- The cold-start-only model scores `40.9%` on AIME2024 and `34.5%` on AIME2025, versus `26.7%` for the untuned base model on AIME2024.
- ReTool with `DeepSeek-R1-Distill-Qwen-32B` reaches `72.5%` on AIME2024 and `54.3%` on AIME2025.
- Against named baselines, ReTool exceeds `s1-32B` by `10.3` points on AIME2024 (`67.0` vs `56.7`) and exceeds `OpenAI o1-preview` by `11.4` points on AIME2025 (`49.3` vs `37.9`).
- Behavioral analysis reports average response length dropping from roughly `10k` tokens to `6k` tokens, a reduction of about `40%`.
- The proportion of responses containing code rises to nearly `98%` by the end of RL training.
- Total correct code counts over the test set rise from about `1k` to `5k` during RL, and average code length grows to nearly `5x` its pre-RL level.

## Limitations

- The evaluation is narrow: the paper centers on AIME2024 and AIME2025, so generalization to broader tool-using tasks, other domains, or non-math environments is not established.
- ReTool uses only a code interpreter; the results do not show whether the same training recipe transfers cleanly to search, databases, or more heterogeneous tool APIs.
- The reward is outcome-only and binary, which keeps the setup simple but provides little direct supervision for tool efficiency, safe execution, or intermediate-step quality.
- The main reported models are `32B`-scale systems with long-context rollout and sandbox infrastructure, so the training recipe is operationally expensive even if it is sample-efficient in RL steps.
