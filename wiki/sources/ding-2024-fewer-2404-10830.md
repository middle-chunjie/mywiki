## Summary

This paper argues that the standard concatenate-then-split pipeline for decoder-only pretraining damages data integrity by fragmenting documents that would otherwise fit inside the model context window. It introduces Best-fit Packing, a two-stage procedure that first chunks only overlength documents and then packs resulting chunks into fixed-length training sequences without further breaking them. The packing stage is cast as bin packing and implemented with an optimized Best-Fit-Decreasing algorithm that runs in `O(N log L)` time by indexing remaining capacities with a segment tree over `L` possible lengths. Across 7B and 13B LLaMA-style models trained on RefinedWeb and the Stack, the method preserves near-identical packing efficiency while improving reading comprehension, context following, summarization, and code synthesis, and substantially reducing closed-domain hallucination.

## Problem & Motivation

LLM pretraining usually concatenates documents and slices the token stream into fixed-length sequences to avoid padding. That choice is token-efficient, but it unnecessarily truncates many documents that would fit intact within the context window if they were packed more carefully. The authors argue that this fragmentation removes grounding context, making next-token prediction less faithful to the local document state and encouraging hallucinations such as unsupported summary details, incorrect code variable usage, or weaker acquisition of tail knowledge. The paper therefore targets a data-pipeline question rather than a model-architecture change: can pretraining keep whole documents whenever possible while remaining as efficient as concatenation at billion-document scale?

## Method

- **Two-stage formatting**: given max sequence length `L`, segment each document into chunks of length `<= L`; only documents longer than `L` are truncated, so all shorter documents remain intact.
- **Packing objective**: pack chunk set `C = {c_1, ..., c_N}` into sequence groups `S = {s_1, ..., s_M}` with constraint ``sum_{c in s_i} l(c) <= L`` while minimizing the number of groups `M`.
- **Best-Fit-Decreasing heuristic**: sort chunks by descending length, then place chunk `c_i` into bin ``j* = argmin_{j in J} r(b_j)`` where `J = {j | r(b_j) >= l(c_i)}` and `r(b_j)` is remaining capacity.
- **Optimized search structure**: because capacities are integers in `[1, L]`, the authors replace an `O(N)`-scale bin tree with an `O(L)` segment tree whose `i`th leaf stores value `i` iff some bin has remaining capacity `i`; this yields `O(N)` counting-sort preprocessing plus `O(N log L)` packing time.
- **Scalability details**: a capacity-to-bin map retrieves an actual best-fit bin after the tree returns the smallest feasible remaining capacity; the method is implemented offline over tokenized chunk lengths and scales linearly in data size apart from the `log L` factor.
- **Training setup**: models use the LLaMA 7B/13B architecture, RefinedWeb for natural language, and the Stack subset of 7 languages for code; configurations are `13B/2048/500B tokens`, `13B/8192/500B tokens`, and `7B/2048/300B tokens`.
- **Optimization**: training uses AdamW, learning rate `3e-4`, cosine schedule, `3000` warmup steps, global batch size `2M` tokens, FlashAttention2, and `256` A100 GPUs.
- **Attention masking choice**: concatenation allows cross-document attention inside a sequence, while Best-fit Packing masks cross-document attention so each packed document chunk behaves as a standalone sample despite sharing a batch sequence.
- **Evaluation protocol**: the paper compares concatenation against packing on reading comprehension, NLI, context following, summarization, commonsense/closed-book QA, and program synthesis, with additional code hallucination measurement via static analysis of undefined names.

## Key Results

- **Runtime / scalability**: at `1B` documents and `L = 2048`, standard FFD takes `26,354s` while optimized BFD takes `10,816s` (`2.44x` faster); at `2B`, runtime is `55,074s` vs `22,244s` (`2.48x`).
- **Compactness**: compared with concatenation, packing adds only `+6,253` sequences on RefinedWeb-2k (`0.0024%`), `+411` on RefinedWeb-8k (`0.00063%`), and `+1,786` on Stack-2k (`0.0028%`), supporting the claim of near-equal training efficiency.
- **Reading comprehension**: average gains are strongest on open-ended tasks; for example Natural Questions at `2k` improves from `53.86%` to `61.61%` EM, and SQuAD at `8k` improves from `23.52%` to `25.35%` EM.
- **Natural language inference / context following**: RTE at `2k` rises from `55.74%` to `60.28%`, and MemoTrap at `2k` rises from `35.58%` to `41.56%` (`+16.8%` relative), with no statistically significant degradations reported across tasks.
- **Summarization faithfulness**: on CNN/DailyMail at `2k`, ROUGE-1/2/L improves from `29.46/11.04/19.47` to `33.79/13.14/22.88`, SummaC from `32.77%` to `39.55%`, and QAFactEval from `3.348` to `3.772`.
- **Program synthesis / hallucination**: on HumanEval, `Pass@100` improves from `35.28%` to `40.57%` (`+15.0%` relative) while undefined-name errors drop from `5.10%` to `2.41%`; on MBPP, undefined-name errors fall from `9.52%` to `3.97%` (`-58.3%`).

## Limitations

- The paper studies only decoder-only LLaMA-style models at `7B` and `13B`, with context lengths `2k` and `8k`; it does not establish whether the same gains persist at substantially larger scales or under different tokenizer/data-mix regimes.
- Best-fit Packing is an offline data-pipeline method that still requires preprocessing, chunk-index bookkeeping, and block attention support during training, so the implementation burden is not literally zero even if throughput remains close to concatenation.
- The hallucination analysis is narrow by design: it focuses on closed-domain, context-grounded failures and code undefined-name errors, not open-domain factuality or long-horizon agentic reliability.
- Faithfulness conclusions for summarization rely on automatic metrics such as SummaC, QAFactEval, and FAVA rather than direct human annotation.
- The theoretical section uses a simplified stochastic process to motivate truncation harm; it is suggestive, not a full theory of transformer learning under truncation.
