## Summary

LONGPROC is a benchmark for evaluating long-context language models on long procedural generation rather than short-answer retrieval. It defines six deterministic tasks, including HTML-to-TSV extraction, theory-of-mind tracking, countdown search, and travel planning, with outputs ranging from roughly `0.5K`, `2K`, to `8K` tokens. The core idea is to force models to repeatedly integrate dispersed evidence, maintain state over long traces, and emit structured long-form outputs that can be scored with rule-based metrics. Across 23 models, frontier proprietary systems dominate short settings, but performance collapses at `8K` outputs: GPT-4o averages `38.1`, Gemini-1.5-Pro `54.0`, and many open-weight models approach failure. The paper argues that long-context recall alone is an insufficient proxy for robust long-form reasoning and execution.

## Problem & Motivation

Existing long-context benchmarks mostly test retrieval from long inputs while requiring only short responses, so they under-measure abilities needed for realistic tasks such as multi-page web interaction, state tracking, or constrained planning. LONGPROC targets cases where relevant evidence is highly dispersed and each next output segment depends on both the original context and prior generated state. The authors want a benchmark that is harder than low-dispersion recall suites, supports long outputs up to `8K` tokens, and remains objectively scorable without subjective LLM judging.

## Method

- The benchmark formalizes procedural generation as a deterministic process `y_i^* = π(X, y_1^*, ..., y_{i-1}^*)`, where the gold output `Y^* = {y_1^*, ..., y_n^*}` is a sequence of structured entries.
- Models are prompted to execute the procedure with detailed instructions and few-shot traces, producing `Y = LCLM(I, X)` rather than a short final answer only.
- LONGPROC contains 6 tasks: HTML to TSV, Pseudocode to Code, Path Traversal, Theory-of-Mind Tracking, Countdown, and Travel Planning.
- Difficulty is controlled by target output length, with subsets around `0.5K`, `2K`, and `8K` tokens (measured with the Llama-3 tokenizer); Pseudocode to Code has no `8K` split and Travel Planning has no `0.5K` split.
- Search-heavy tasks use explicit depth-first procedures. Countdown follows a DFS over number states with operations in `[+, -, *, /]`; Travel Planning applies DFS over partial itineraries with checks such as `EXISTSDIRECTCONNECTION(...)` and `COMPATIBLEWITHFIXEDSCHEDULE(...)`.
- Evaluation is rule-based by task: HTML uses row-level F1, Pseudocode to Code uses unit-test pass rate, Path Traversal and ToM Tracking use exact match, and Countdown / Travel Planning verify final-solution constraints.
- The study evaluates `23` LCLMs with greedy decoding. Models receive an extra `0.5K-1K` output buffer for tokenizer differences, and reasoning models are allowed up to `16K` output tokens to absorb additional thinking traces.
- Analysis further slices outputs into four equal segments to measure whether correctness degrades with generation depth, isolating long-range coherence failures.

## Key Results

- Average scores degrade sharply with output length: GPT-4o goes from `94.8` (`0.5K`) to `83.4` (`2K`) to `38.1` (`8K`); Gemini-1.5-Pro goes from `89.2` to `79.4` to `54.0`.
- Open-weight models below `15B` parameters largely fail on `2K` tasks; the best sub-15B model reported for 2K is R1-Distill-Llama-3-8B at `24.6` average.
- The strongest `8K` average is Gemini-1.5-Pro at `54.0`; GPT-4o reaches `38.1`, and Claude-3.5-Sonnet reaches `22.0`, despite all advertising context windows of at least `128K`.
- Reasoning variants often beat instruct variants overall: R1-Distill-Qwen2.5-32B scores `80.6 / 60.9 / 22.4` across `0.5K / 2K / 8K`, versus `68.4 / 50.3 / 17.1` for Qwen2.5-32B-Inst.
- Human evaluators still far exceed frontier models on difficult search tasks: `100.0` vs `68.6` (GPT-4o) / `32.3` (Gemini-Pro) on Countdown `8K`, and `91.4` vs `14.2` / `40.0` on Travel Planning `8K`.
- Error analysis shows concrete long-range failures: for GPT-4o at `8K`, `8/20` HTML errors come from skipped rows, `19/20` ToM errors from incorrect long-range inferences, and `10/20` Travel Planning errors from hallucinated direct flights.

## Limitations

- LONGPROC is still a synthetic benchmark suite with tightly specified procedures, so it does not fully capture open-ended agent behavior or subjective writing quality.
- The benchmark emphasizes deterministic structured outputs; this improves evaluation reliability but excludes many realistic tasks whose correctness is partly semantic or underspecified.
- Task coverage is broad but still limited to six mostly text-based tasks, with no multimodal setting and no direct measurement of tool use or interactive execution.
- Some findings about reasoning models are confounded by token-budget effects: lower accuracy can reflect non-termination under a `16K` generation budget rather than worse conditional reasoning.
