## Summary

⚠ Possibly outdated: published 2023; re-verify against recent literature.

MEGABYTE proposes a multiscale decoder-only Transformer for end-to-end autoregressive modeling of sequences longer than one million bytes without tokenization. The model groups bytes into fixed-size patches, runs a large global Transformer over patch representations, and uses a smaller local Transformer to predict bytes within each patch, so most parameters are spent at patch rather than byte resolution. This changes the dominant long-context cost from per-byte feed-forward work and `O(T^2)` attention toward `O(T^2/P^2 + TP)`, with `P = T^{1/3}` yielding `O(T^{4/3})` attention complexity. Under compute- and data-matched comparisons, MEGABYTE beats baseline Transformers and PerceiverAR on long-context text, matches ImageNet64 state of the art with less than half the GPU hours, scales to `1,228,800`-byte ImageNet640 sequences, and improves raw-audio modeling and generation speed.

## Problem & Motivation

Standard autoregressive Transformers scale poorly to very long sequences such as books, source code, raw audio, and high-resolution images. The paper argues that the bottleneck is not only quadratic self-attention but also the cost of large per-position feed-forward networks, which makes byte-level modeling of million-byte contexts impractical. The authors want a single tokenization-free architecture that works directly on raw bytes across modalities, preserves autoregressive likelihood modeling, and remains competitive with subword systems on text while also handling images and audio where lossless tokenization is unavailable. They further motivate a local autoregressive model because directly predicting a whole patch would require a `256^P` output space.

## Method

- **Input decomposition**: a byte sequence of length `T` is segmented into `K = T / P` fixed-size patches of size `P`; the patch embedder losslessly concatenates `P` byte embeddings to produce `h^{global-in} in R^{K x (P·D_G)}`.
- **Global module**: a causal decoder-only Transformer operates over the patch sequence and outputs `h^{global-out}` for each patch; the input is shifted by `P` bytes so the model never leaks future bytes.
- **Local module**: for byte position `t = k·P + p`, the local input is `h_{k,p}^{local-in} = W^{GL} h_{k,p}^{global-out} + E_{x_{t-1}}`, with a learned local pad token used when `p = 0`; a smaller causal Transformer then models bytes within patch `k`.
- **Output distribution**: the byte-level likelihood is `p(x_t | x_{<t}) = softmax(E^{local-embed} h_{k,p}^{local-out})_{x_t}` over a fixed vocabulary `V = 256`.
- **Complexity reduction**: the attention cost becomes `O(T^2 / P^2 + T·P)` instead of `O(T^2)`; choosing `P = T^{1/3}` gives `O(T^{4/3})`, and allocating feed-forward compute per patch allows larger models at the same budget.
- **Architectural extensions**: the paper studies a causal CNN patch encoder with filter sizes `3/5/7`, cross-patch local attention that reuses the previous patch's last `r` keys/values with rotary embeddings, and strided inference with two passes offset by `P / 2`.
- **Training defaults**: Adam with `beta_1 = 0.9`, `beta_2 = 0.98`, gradient clipping `1.0`, polynomial decay with `500` warmup steps and final LR `0`, dropout `0.1`, weight decay `0.1`, pre-norm blocks, and ReLU activations.
- **Representative settings**: compute-matched text experiments use MEGABYTE global/local sizes `758M / 262M` with context `8,192` and patch size `8`; ImageNet640 uses `125M / 83M` with patch size `192`; raw audio uses `350M / 125M`, context `524,288`, and patch size `32`.

## Key Results

- **Compute-matched text modeling**: on PG-19 / Stories / Books / arXiv / Code, MEGABYTE reaches `1.000 / 0.978 / 1.007 / 0.678 / 0.411` bpb versus Transformer `1.057 / 1.064 / 1.097 / 0.816 / 0.575` and PerceiverAR `1.104 / 1.070 / 1.104 / 0.791 / 0.546`.
- **Large-scale PG-19**: MEGABYTE achieves test perplexity `36.4`, far better than the compute-matched byte Transformer `69.4` and byte PerceiverAR `88.8`, while approaching strong subword systems such as PerceiverAR `28.9` and BlockRecurrent `26.5`.
- **ImageNet64 SOTA at lower cost**: on `64 x 64` images, MEGABYTE matches PerceiverAR at `3.40` bpb while using less than half the GPU hours reported for the comparator.
- **Image scaling**: in compute-controlled experiments, MEGABYTE beats Transformer / PerceiverAR on Image64 (`3.52` vs `3.62` / `3.55`), Image256 (`3.158` vs `3.801` / `3.373`), and Image640 (`2.282` vs `2.847` / `2.345`), and handles the full `1,228,800`-byte Image640 sequence in one forward pass.
- **Raw audio**: on `16 kHz`, 16-bit audio modeled as bytes, MEGABYTE reaches `3.477` bpb versus PerceiverAR `3.543` and Transformer `3.567`.
- **Generation efficiency**: on PG-19, an `8,192`-byte sample takes `93s` with MEGABYTE versus `132s` for a `350M` Transformer, about `40%` faster despite the MEGABYTE system using a `1.3B` global model plus `218M` local model.
- **Ablations**: removing the local model hurts strongly on arXiv / audio / ImageNet256 (`1.263 / 5.955 / 4.768` bpb), and removing the global model also degrades performance (`1.373 / 3.659 / 3.181`), confirming that both scales matter.

## Limitations

- The strongest claims rely mainly on compute- and data-matched experiments using the authors' own implementations, so comparisons to external state of the art can still be affected by tuning and evaluation protocol differences.
- The method introduces several modality-specific hyperparameters, especially patch size and local/global capacity ratio, which must be tuned for good performance.
- Better inference quality can require extra passes: strided inference and sliding windows improve PG-19 bpb, but increase inference cost from `1X` to `2X` or `4X`.
- Longer context alone is not sufficient: on PG-19, extending from `8,192` to `16,384` bytes slightly worsens the best reported bpb (`0.8751` to `0.8787`), so the gains are not purely from larger windows.
- Some extensions provide only small or inconsistent gains; for example, cross-patch removal remains competitive on arXiv (`0.6781` vs `0.6871`) and the CNN encoder changes results only marginally.
- The evaluation is focused on likelihood-based modeling for text, images, and audio, not downstream instruction following or interactive multimodal generation.
