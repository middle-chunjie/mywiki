## Summary

⚠ Possibly outdated: published 2023; re-verify against recent literature.

The paper proposes SKCoder, a retrieval-augmented code generation pipeline that explicitly models code reuse as `retrieve -> sketch -> edit` rather than direct copying from similar programs. Given a natural-language requirement, it retrieves a similar code snippet with BM25/Lucene, predicts a code sketch by keeping only requirement-relevant tokens from the retrieved code, and then edits that sketch into full target code with an encoder-decoder model. The sketcher is initialized from GraphCodeBERT and the editor from CodeT5-base. Across HearthStone, Magic, and the new AixBench-L benchmark, SKCoder consistently beats strong pretrained and copy-based baselines, including CodeT5-base and REDCODER, with especially large gains on exact correctness metrics such as EM and Pass@1.

## Problem & Motivation

Prior retrieval- or copy-based code generation systems can benefit from similar code, but they often copy irrelevant details and fail to adapt the retrieved snippet to the new requirement. The paper argues that real developers do not reuse code by literal copying: they first identify a reusable high-level pattern, treat it as a sketch, and then fill in requirement-specific details. SKCoder is motivated by this observation and aims to separate "how to write" knowledge from the retrieved code sketch and "what to write" knowledge from the natural-language description.

## Method

- **Pipeline**: decompose code generation into three modules, `retrieve`, `sketch`, and `edit`, where the model predicts target code `Y` from an NL description `X` via retrieved code `Y'` and sketch `S`.
- **Retriever**: use Lucene with the BM25 scoring function over natural-language descriptions to retrieve a similar code snippet `Y'`; the retrieval corpus is the training split for HearthStone/Magic and mined GitHub projects for AixBench-L.
- **Sketcher input/output**: concatenate `X` and `Y'`, encode them as `[H; H'] = Encoder([X; Y'])`, then classify each retrieved-code token with `p_i = softmax(W_s h'_i + b_s)`.
- **Sketch extraction rule**: construct sketch tokens with `s_i = y'_i if p_i > t else <pad>`, then merge consecutive placeholders; this keeps reusable structure while masking irrelevant details.
- **Sketch supervision**: build silver sketches from the longest common subsequence between retrieved code and gold code, treating `LCS(Y', Y)` as the target sketch used to train the sketcher.
- **Sketcher objective**: optimize token-level binary extraction with `L_s = -Σ[I_i log(p_i) + (1 - I_i) log(1 - p_i)]`.
- **Editor**: use an encoder-decoder Transformer that takes `[X; S]` as input and autoregressively predicts code `Y`, trained with `L_e = -Σ log P(y_i | X, S, y_<i)`.
- **Architecture / initialization**: the sketcher is a `12`-layer Transformer encoder initialized from [[graphcodebert]]; the editor uses a `12`-layer encoder and `12`-layer decoder initialized from [[codet5-base]].
- **Training / inference details**: the deterministic retriever has no trainable parameters; training uses `Top-5` retrieved snippets to build sketcher/editor data, batch size `32`, and `2 x NVIDIA A100` GPUs; inference uses `Top-1` retrieval and beam search with beam size `10`.
- **Datasets**: evaluate on HearthStone (`533/66/66` train/dev/test), Magic (`11,969/664/664`), and AixBench-L (`190,000/10,000/175`), where AixBench-L also provides unit tests for functional evaluation.

## Key Results

- On HearthStone, SKCoder reaches `30.30` EM, `83.12` BLEU, and `70.97` CodeBLEU, improving over [[codet5-base]] by `17.26%` in EM.
- On Magic, SKCoder reaches `35.39` EM, `85.39` BLEU, and `82.42` CodeBLEU, outperforming [[codet5-base]] by `22.41%` in EM.
- On AixBench-L, SKCoder reaches `20.00` Pass@1 and `38.70` AvgPassRatio, versus `15.43` and `24.53` for [[codet5-base]], a relative `29.62%` gain in Pass@1.
- Compared with the strongest copy-based baseline REDCODER, SKCoder improves EM by `42.86%`, Pass@1 by `25%`, and AvgPassRatio by `16.78%`.
- Ablation shows the full retrieve-plus-sketch setup improves Pass@1 by up to `114.3%` over an RNN editor, `45.51%` over CodeT5-small, and `29.62%` over CodeT5-base.
- Human evaluation reports `1.3705` correctness, `1.5639` code quality, and `1.7764` maintainability for SKCoder, all significantly above CodeT5-base with `p < 0.005`.

## Limitations

- The retriever is purely lexical BM25 over descriptions, so retrieval quality may degrade when semantically similar tasks have weak token overlap; the paper leaves stronger neural retrieval as future work.
- The overall pipeline is not trained end-to-end because sketch extraction uses hard decisions and silver sketches derived from LCS.
- Evaluation is concentrated on three datasets and two programming languages, with the functional benchmark using only `175` test examples in AixBench-L.
- The strongest comparisons are 2021-2023 pretrained code models rather than newer large-scale code LLMs, so the absolute frontier position is likely outdated.
