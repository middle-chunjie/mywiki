## Summary

SKILL0 proposes an in-context reinforcement learning framework for internalizing reusable agent skills into model parameters instead of retrieving skill files at inference time. During training, the agent conditions on compact visual renderings of interaction history plus selected skills, then a dynamic curriculum progressively removes skills according to on-policy helpfulness so the policy must transition from scaffolded execution to zero-shot autonomy. The policy jointly predicts action `a_t` and compression ratio `c_t`, optimizing task reward plus a compression bonus `lambda * log(c_t)` on successful trajectories. On ALFWorld and Search-QA, Qwen2.5-VL-3B with Skill0 reaches `87.9` and `40.8` average performance, beating AgentOCR by `+9.7` and `+6.6` while using only `0.38k` and `0.18k` tokens per step.

## Problem & Motivation

Inference-time skill augmentation helps LLM agents by injecting retrieved procedural guidance, but the paper argues that this setup has three structural problems: retrieval noise can add misleading instructions, repeated skill injection inflates token cost over long horizons, and the model never truly learns the skill because competence remains in prompt context rather than in parameters. A naive RL alternative is also unsatisfactory: removing skills from the start makes hard multi-step tasks difficult to learn, while keeping full skill context throughout training creates prompt dependence. Skill0 is motivated by the need for a training regime that begins with explicit skill scaffolding, then gradually withdraws it so policy optimization can convert external procedural knowledge into intrinsic behavior.

## Method

- **Agent formulation**: the task is modeled as sequential decision making with history `h_t = {o_1, ..., o_t}` and policy `pi_theta(a_t | I, h_t)`, where the environment returns the next observation after each action.
- **SkillBank structure**: reusable knowledge is organized as grouped markdown skill files `S_k` in a two-level library containing general skills and task-specific skills; training uses skill files rather than individual retrieved snippets.
- **Context rendering**: interaction history `h_t` and selected skills `S` are rendered into a compact RGB image and encoded as `V_t = Enc(h_t, S; c_t)`, turning long textual state into visual context to reduce token overhead.
- **Joint action and compression policy**: the model predicts both task action and compression ratio via `(a_t, c_t) ~ pi_theta(a_t, c_t | I, V_t)`, so compression becomes part of the learned policy rather than a fixed preprocessing choice.
- **ICRL reward**: successful trajectories receive an additional compression reward `r_t^comp = log(c_t)` and total reward `r_t_tilde = r_t + lambda * r_t^comp`, encouraging compact context only when task success is preserved.
- **Policy optimization**: Skill0 uses a clipped policy objective over grouped rollouts with importance ratio `r_i,t(theta) = pi_theta(...) / pi_theta_old(...)` plus a KL penalty to a reference policy; advantages are normalized within each rollout group.
- **Dynamic curriculum**: at stage `s`, the skill budget is bounded by `M^(s) = ceil(N * (N_S - s) / (N_S - 1))`; every `d` steps the method evaluates each skill file's helpfulness `Delta_k = Acc_k^(w/ skill) - Acc_k^(w/o skill)`, filters `Delta_k <= 0`, ranks the rest, and keeps the top files under the current budget.
- **Training setup**: both ALFWorld and Search-QA use `N_S = 3`, validation subset size `1000`, and `d = 10`; the authors train Qwen2.5-VL for at most `180` steps on `4` H800 GPUs, with ALFWorld using batch size `16`, `8` rollouts, prompt limit `3072`, and Search-QA using batch size `128`, prompt limit `4096`, and E5 retrieval.

## Key Results

- On Qwen2.5-VL-3B, Skill0 reaches `87.9` average success on ALFWorld versus AgentOCR `78.2`, GRPO `79.9`, and SkillRL `82.4`, while keeping context cost at `0.38k` tokens per step versus SkillRL `2.21k`.
- On Qwen2.5-VL-3B Search-QA, Skill0 scores `40.8`, outperforming AgentOCR `34.2` by `+6.6` and EvolveR `38.2`, with only `0.18k` tokens per step versus SkillRL `0.87k`.
- On Qwen2.5-VL-7B, Skill0 obtains `89.8` on ALFWorld and `44.4` on Search-QA, remaining competitive with or better than SkillRL (`89.9`, `47.1`) while not requiring any inference-time skill retrieval.
- Skill0 is especially strong on hard out-of-domain Search-QA subsets, reaching `63.7` Bamboogle on 3B and `66.9` on 7B, far above Search-R1 (`32.5` avg on 3B, `38.5` avg on 7B overall) and ZeroSearch (`31.7` / `39.1` avg).
- The dynamic curriculum ablation is decisive on ALFWorld: full Filter-and-Rank-and-Select yields `87.9` without skills at inference, while removing filtering drops to `78.9` and removing ranking collapses to `62.9`.
- Skill budget scheduling matters: the proposed budget yields `+1.6` transfer when skills are removed at inference, whereas fixed-full settings such as `[6,6,6]` suffer `-13.3` and static no-skill training `[0,0,0]` reaches only `78.9`.

## Limitations

- The paper explicitly notes that Skill0 depends on the quality of the initial SkillBank; weak or poorly structured skill files would directly limit what can be internalized.
- Relevance-driven offline grouping requires repartitioning validation tasks for each new domain, so the curriculum setup is not plug-and-play across arbitrary environments.
- Empirical evaluation is concentrated on two agent settings, ALFWorld and Search-QA, so the paper does not yet establish whether the same training recipe transfers to broader software, GUI, or embodied agent workloads.
- The method adds nontrivial infrastructure, including skill-file grouping, periodic helpfulness validation, and visual context rendering, which may increase implementation complexity relative to standard text-only RL baselines.
