## Summary

SkillsBench studies whether runtime agent skills materially improve LLM agents instead of assuming that extra task context is automatically useful. The paper builds a benchmark of 84 evaluated containerized tasks across 11 domains, each paired with curated skill packages, oracle solutions, and deterministic verifiers, and evaluates 7 agent-model configurations over 7,308 valid trajectories under no-skills, curated-skills, and self-generated-skills conditions. Curated skills raise mean pass rate from `24.3%` to `40.6%` (`+16.2` percentage points), while self-generated skills slightly underperform the vanilla baseline. The analysis further shows that `2-3` focused skill modules outperform larger bundles, moderate-length skill documents beat comprehensive ones, and smaller models with curated skills can outperform larger models without skills.

## Problem & Motivation

Agent skills are increasingly used as a practical way to inject procedural know-how into terminal agents, but prior benchmarks mainly evaluate raw model or harness capability rather than the marginal value of skills themselves. That leaves two gaps: practitioners cannot estimate whether a given skill ecosystem is worth the complexity, and researchers lack a controlled setting for studying what kinds of skills help, hurt, or fail. SkillsBench reframes skills as first-class evaluation artifacts by pairing each task with curated skill packages, deterministic verifiers, and matched no-skill baselines.

## Method

- **Benchmark construction**: curate `84` evaluated tasks across `11` domains from `322` candidate submissions by `105` contributors; each task includes an instruction, Docker environment, oracle solution, and deterministic verifier.
- **Skill definition**: formalize a skill as a reusable package with `1` required `SKILL.md` file plus `0+` optional resources; skills must encode procedural guidance for a task class rather than declarative answers for a single instance.
- **Quality control**: require structural validation, oracle execution with `100%` verifier pass rate, human-written instructions, leakage audits, and human review over realism, oracle quality, skill quality, and anti-cheating properties.
- **Evaluation matrix**: run `7` model-harness configurations across `3` conditions (`no-skills`, `with-skills`, `self-generated-skills`) with `5` trials per task, yielding `7,308` valid trajectories.
- **Metrics**: compute task pass rate by averaging binary rewards over `5` trials and report normalized gain `g = (pass_skill - pass_vanilla) / (1 - pass_vanilla)` alongside absolute deltas.
- **Runtime protocol**: use `temperature = 0`, per-task round limits of `10/30/50` for `core/extended/extreme`, a sliding-window context budget of `8K` tokens, and container budgets of `1-4` CPU cores, `2-10 GB` RAM, `10 GB` storage, and `0` GPUs.
- **Skill injection**: copy `environment/skills/` into harness-specific discovery paths such as `~/.claude/skills`, `~/.codex/skills`, `~/.gemini/skills`, and `~/.agents/skills` so each harness can discover the same packaged skills.

## Key Results

- Curated skills improve average pass rate from `24.3%` to `40.6%`, a mean gain of `+16.2` percentage points and `21.5%` normalized gain.
- Gemini CLI with Gemini 3 Flash achieves the best with-skills performance at `48.7%`, while Claude Code with Opus 4.5 shows the largest absolute gain at `+23.3` points (`22.0% -> 45.3%`).
- Self-generated skills do not replicate curated benefits: the supported configurations average `-1.3` points versus no-skills, and Codex with GPT-5.2 drops from `30.6%` to `25.0%`.
- Domain effects are highly uneven: Healthcare gains `+51.9` points and Manufacturing `+41.9`, while Mathematics gains only `+6.0` and Software Engineering `+4.5`.
- Skill design matters: `2-3` skills per task yield the best improvement (`+18.6`), `4+` skills fall to `+5.9`, and comprehensive skills are the only complexity tier with a negative delta (`-2.9`).
- Skills can partially substitute for model scale: Claude Haiku 4.5 with skills reaches `27.7%`, exceeding Claude Opus 4.5 without skills at `22.0%`.

## Limitations

- The benchmark is restricted to terminal-based, containerized tasks, so transfer to GUI agents, multi-agent systems, and longer-horizon workflows remains untested.
- The paper does not fully isolate procedural structure from context-length effects; stronger length-matched controls such as irrelevant text or retrieval-only baselines are left to future work.
- Containerization and repeated trials reduce but do not eliminate nondeterminism, benchmark contamination, or skill-solution leakage.
- The paper has a reporting inconsistency: the abstract mentions `86` tasks, while the main benchmark description and reported metrics use `84` evaluated tasks.
