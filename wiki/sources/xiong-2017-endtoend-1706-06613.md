## Summary

⚠ Possibly outdated: published 2017; re-verify against recent literature.

The paper introduces K-NRM, an end-to-end neural ranking model for ad-hoc search that learns relevance-oriented word embeddings directly from ranking supervision. Given query and document titles, it builds a cosine-similarity translation matrix over token embeddings, applies multiple RBF kernels to count soft matches at different similarity levels, and feeds the resulting soft-TF features into a shallow learning-to-rank layer. This design explicitly separates exact match, useful semantic match, and noise, rather than collapsing them into a single average or max similarity. On a large commercial Sogou.com query log, K-NRM substantially outperforms feature-based and earlier neural baselines across in-domain, cross-click-model, and raw-click evaluation settings, with especially large gains at top-ranked positions.

## Problem & Motivation

Ad-hoc retrieval depends heavily on exact lexical overlap, but pure exact matching fails under vocabulary mismatch. Earlier neural IR models tried to use distributed representations for soft matching, yet often underperformed because they either compressed query/document text into coarse representations or used pre-trained embeddings whose similarity geometry was not optimized for ranking relevance. The paper asks how to preserve strong exact-match behavior while still exploiting graded semantic similarity, and how to learn that similarity metric from ranking labels rather than from generic distributional context. K-NRM is proposed as a differentiable architecture that turns query-document word interactions into trainable multi-level soft-match features.

## Method

- **Input and embedding layer**: use only query words and document-title words; each token `t` is mapped to a trainable embedding `v_t` with dimension `L = 300`, initialized from word2vec over the training corpus.
- **Translation matrix**: construct `M_{ij} = cos(v_{t_i^q}, v_{t_j^d})`, yielding a query-document interaction matrix over word-level similarities instead of manually engineered retrieval features.
- **Kernel pooling**: for each query term row `M_i`, apply `K = 11` RBF kernels `K_k(M_i) = sum_j exp(-(M_{ij} - mu_k)^2 / (2 sigma_k^2))` and aggregate with `phi(M) = sum_i log K(M_i)`, producing multi-level soft-TF features.
- **Kernel design**: one exact-match kernel uses `mu_0 = 1.0`, `sigma = 10^-3`; the other 10 kernels use means `mu in {0.9, 0.7, ..., -0.9}` with `sigma = 0.1`, so different similarity bands capture exact match, useful soft match, and noise.
- **Ranking layer**: score a query-document pair by `f(q, d) = tanh(w^T phi(M) + b)`, where `w` and `b` are learned jointly with the embeddings.
- **Training objective**: optimize pairwise hinge loss `l = sum_q sum_(d+, d-) max(0, 1 - f(q, d+) + f(q, d-))`, letting ranking supervision backpropagate through kernels into token similarities and embeddings.
- **Optimization details**: Adam with batch size `16`, learning rate `0.001`, `epsilon = 1e-5`, early stopping patience `5`; implementation uses [[tensorflow]] and full training converges in about `12` hours on an AWS GPU machine.
- **Model scale and data**: vocabulary size is `165,877`, K-NRM has `49,763,110` parameters, and experiments use `95,229` training queries sampled from roughly `35` million search sessions.

## Key Results

- On Testing-SAME, K-NRM reaches `NDCG@1 = 0.2642`, `NDCG@3 = 0.3210`, `NDCG@10 = 0.4277`, improving over Coor-Ascent by `+65.75%`, `+43.25%`, and `+20.58%`.
- On Testing-DIFF, K-NRM reaches `NDCG@1 = 0.2984`, `NDCG@3 = 0.3092`, `NDCG@10 = 0.4201`, outperforming Coor-Ascent by `+42.84%`, `+28.26%`, and `+11.28%`.
- On Testing-RAW, K-NRM improves `MRR` from `0.2415` to `0.3379` over Coor-Ascent, a relative gain of `+39.92%`.
- Ablations show the full model is materially better than exact-match-only, word2vec-fixed, click2vec-fixed, max-pooling, and mean-pooling variants; for example, full-model Testing-SAME `NDCG@1 = 0.2642` vs exact-match `0.1351` and mean-pool `0.2297`.
- Performance declines on rarer queries but remains positive: on tail queries (`< 50` appearances), K-NRM still improves Testing-RAW `MRR` from `0.2977` to `0.3230` (`+8.49%`).

## Limitations

The experiments rely on proprietary Sogou.com click logs rather than public manual relevance judgments, so reproducibility is limited and some conclusions depend on click-model assumptions. Training and evaluation for neural models use only document titles, not full documents, which may understate or distort behavior on richer ranking settings. The method is data hungry: with only `2K` training sessions it underperforms Coor-Ascent, and its gains grow with vocabulary coverage. K-NRM also uses a large embedding table (`49.8M` parameters), which can overfit or become expensive without large-scale logs. Finally, the paper is confined to Chinese web search reranking from 2017, so its claims should be re-checked against later neural IR architectures and datasets.
