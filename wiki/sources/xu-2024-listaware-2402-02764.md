## Summary

The paper argues that list-aware retrieval should not treat reranking and truncation as two disconnected stages, because the split loses shared list-level context and amplifies error propagation from reranking into cut-off prediction. It proposes GenRT, a joint encoder-decoder model that generates the final ranked list autoregressively while making a truncation decision at each step. The encoder captures global dependencies within the input list, while the decoder uses sequential history plus a local backward window to approximate bidirectional evidence for truncation. Across web-search learning-to-rank benchmarks and retrieval-augmented generation settings, GenRT reports state-of-the-art results for both reranking and truncation, and shows that jointly optimizing the two tasks improves not only ranking quality but also downstream LLM efficiency-accuracy trade-offs.

## Problem & Motivation

Prior list-aware retrieval work usually reranks first and truncates later, but the paper argues this pipeline is structurally suboptimal. Reranking and truncation depend on the same list-level evidence, yet separate models cannot fully share that information, and small reranking mistakes can cause large truncation errors by pushing irrelevant documents into the top positions where they trigger premature cut-offs. This is especially problematic for retrieval-augmented generation, where the number and order of retrieved passages jointly affect LLM accuracy, latency, and noise exposure. The paper therefore targets a single model that can optimize both tasks concurrently.

## Method

- **Input representation**: given query `q`, documents `D = [d_1, ..., d_N]`, and initial ranking scores `L = [l_1, ..., l_N]`, each item is embedded as ``u_{d_i} = f(q, d_i, l_i)``. For text ranking, the model uses interaction-based `bert-base` representations plus a learnable rank-position embedding.
- **Global dependency encoder**: document embeddings are stacked into ``U in R^{N x Z}``, projected by ``X = Swish(MLP(U))``, then passed through residual self-attention as ``O = X + MHSA(LN(X))``. Implementation uses `2` encoder MHSA blocks, each with `8` heads and hidden size `256`.
- **Sequential dependency decoder**: GenRT generates the final list step by step. At step `T`, previous outputs provide sequential context through decoder self-attention, and cross-ranking features are computed with ``I = (1 + MLP(O)) ⊙ FFN-Swish(U)`` and ``S = rFFN(Concat(I, M))`` where `M` repeats the current sequential state. Used documents are masked so no duplicate can be selected.
- **Joint truncation during generation**: after choosing the current top document `r_1^T`, truncation is formulated as binary classification using forward history `F`, the current item embedding, and a local backward window `B` of size ``beta = 4``. Relative-position MHSA aggregates this evidence and outputs ``P = Softmax(MLP(j_{r_1^T}))`` for cut vs. no-cut.
- **Loss design**: reranking uses step-adaptive attention loss with ``alpha_t = 1 / log(1 + t)`` plus step-by-step lambda loss, combined as ``L_R = L_sa-att + eta * L_sbs`` with ``eta = 0.1``. Truncation uses RAML-style soft labels derived from `TDCG@T` and `TDCG@(T + beta)`.
- **Training and inference**: the first epoch trains reranking only; later epochs alternate reranking and truncation batches while freezing the other head. Optimization uses Adam with learning rate ``1e-5`` and batch size `16`. `GenRT fast` reuses first-step scores for reranking-only inference.

## Key Results

- On web-search reranking, GenRT achieves the best reported metrics on all three benchmarks, including `NDCG@10 = 0.4919` / `MAP = 0.5200` on MSLR30K, `0.7505` / `0.7172` on Yahoo!, and `0.7032` / `0.7329` on Istella.
- On web-search truncation, end-to-end GenRT reaches `TDCG = 0.84` on MSLR, `3.11` on Yahoo!, and `5.03` on Istella, outperforming both prior truncation baselines and its truncation-only ablation `GenRT-w/o R` (`0.61`, `3.01`, `4.48`).
- On retrieval-augmented QA reranking, GenRT improves downstream LLM accuracy to `58.79` on Natural Questions and `64.37` on TriviaQA, with higher recall than Seq2Slate and SetRank (`R@5 = 60.78` on NQ, `70.01` on TriviaQA).
- On retrieval-augmented QA truncation, GenRT keeps Natural Questions accuracy at `58.15` with average list length reduced to `17.25`, compared with `58.20` at fixed length `40`; on TriviaQA it reaches `63.25` accuracy with average length `22.19`.
- The paper reports that `GenRT fast` is `0.6x` shorter than SetRank for reranking inference, while full GenRT is `1.6x` longer than a SetRank-AttnCut pipeline but can reduce downstream LLM running time and GPU usage in the end-to-end pipeline (`4.25s`, `35.25 GB`, `57.49` accuracy in one appendix experiment).

## Limitations

- The autoregressive generation paradigm increases inference latency relative to one-shot rerankers, even with a lighter decoder and the `GenRT fast` shortcut.
- The method relies on multiple specialized components and alternating optimization schedules, making training and implementation more complex than standard reranking or truncation baselines.
- Truncation quality depends on the handcrafted local backward window; the paper shows performance degrades when ``beta > 4``, indicating sensitivity to this approximation.
- Evaluation is bounded to top-`40` candidate lists, two open-domain QA datasets, and `gpt-3.5-turbo-16k`, so robustness to stronger LLMs or broader retrieval regimes is not fully established.
- The paper references code release in the future tense and includes a placeholder ACM DOI, so immediate reproducibility details appear incomplete in the manuscript snapshot.
