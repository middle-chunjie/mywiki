## Summary

WebWalker introduces WebWalkerQA, a bilingual benchmark for web traversal that tests whether large language models can dig through a root website and its subpages rather than relying on shallow search-engine retrieval alone. The dataset contains `680` QA pairs spanning `1373` webpages across conference, organization, education, and game domains, with both single-source and multi-source questions. To tackle the task, the paper proposes WebWalker, a two-agent system with an explorer that performs ReAct-style page traversal and a critic that accumulates useful evidence in memory before answering. Results show current LLM agents remain weak on this setting, while combining standard RAG with WebWalker-style vertical exploration improves information-seeking performance.

## Problem & Motivation

Standard retrieval-augmented generation systems usually perform horizontal search over search-engine results, which often surfaces shallow pages and misses information buried several clicks deep inside official websites. Prior web-agent benchmarks also emphasize action execution in noisy HTML or multimodal environments, rather than text-centric reasoning over structured site hierarchies for information seeking. This paper therefore targets web traversal as a distinct setting: given a root URL and a query, the agent must decide which subpages to click, accumulate evidence across depth or multiple sources, and answer from authoritative site content. The motivation is both evaluative and practical, since many real information needs require deep traversal rather than one-shot retrieval.

## Method

- **Benchmark construction**: build [[webwalkerqa]] from official websites in four domains by recursively traversing sublinks, then use a two-stage annotation funnel with GPT-4o draft generation plus human verification; final size is `680` QA pairs over `1373` webpages.
- **Task formalization**: given root URL `U_root` and query `Q`, the agent must navigate the site to collect enough evidence to answer; evaluation uses QA accuracy and successful-run action count `A.C.`.
- **Data taxonomy**: single-source questions are labeled `single_source_i` with `i in [2,4]`, while multi-source questions are labeled `multi_source_i` with `i in [2,8]`, where `i` tracks page depth or summed depth across two pages.
- **Explorer agent**: WebWalker uses a ReAct-style explorer with observation `O_t = (p_t, l_t)`, where page content `p_t` is paired with clickable buttons `l_t = {button_i}_{i=1}^K`; the explorer selects an action `A_t` under policy `pi(A_t | H_t)` from the running history `H_t`.
- **Critic agent**: after each explorer step, the critic reads `(Q, O_t, A_t)`, updates memory `M`, decides whether evidence is sufficient, and emits an answer only when the accumulated information is judged complete.
- **Interaction representation**: current-page markdown and HTML buttons are extracted with Beautiful Soup, and the implementation is built on [[qwen-agent]] with generation hyperparameter `top_p = 0.8`.
- **Experimental setup**: all methods run in zero-shot mode with maximum exploration budget `K = 15`; evaluated backbones have at least `128K` context and at least `7B` parameters.
- **RAG integration**: WebWalker serves as vertical exploration layered on top of horizontal RAG, appending critic memory `M` to retrieved documents before generation.

## Key Results

- **Dataset statistics**: WebWalkerQA contains `680` QA pairs and `1373` webpages; language split is `60.5%` Chinese and `39.5%` English, while domain split is `46.3%` education, `24.0%` conference, `24.0%` game, and `7.9%` organization.
- **Best WebWalker result**: with GPT-4o backbone, WebWalker reaches `37.50` overall accuracy at `4.67` action count, versus Reflexion `35.29 / 4.27` and ReAct `33.82 / 3.83`.
- **Open-weight scaling trend**: with Qwen-2.5-72B, WebWalker reaches `33.26 / 4.32`, outperforming Reflexion `32.50 / 4.09` and ReAct `30.73 / 2.86`; smaller Qwen-2.5-7B only reaches `19.85 / 3.94`.
- **Difficulty grows with depth and multi-source needs**: under GPT-4o WebWalker, single-source accuracy drops from `55.00` on easy to `30.00` on hard, and multi-source accuracy drops from `47.50` on easy to `15.83` on hard.
- **Standard RAG remains weak**: among searched systems, Tongyi is best at `40.73`, Kimi reaches `37.35`, naive RAG reaches `20.73`, MindSearch reaches `11.32`, and close-book o1-preview reaches only `9.85`.

## Limitations

- The released high-quality benchmark is still modest at `680` QA pairs; the paper mentions about `14k` additional silver QA pairs that are not yet human-verified.
- The environment is text-only and click-only, relying on HTML-DOM parsing rather than screenshots or richer multimodal observations.
- WebWalker is prompt-driven and zero-shot, without agent tuning on gold trajectories or dedicated policy learning.
- Better RAG integration is still unfinished: the system assumes a root URL is already known rather than learning end-to-end routing from raw user queries.
- Even the strongest reported WebWalker configuration stays below `40%` overall accuracy, so the task remains far from solved.
