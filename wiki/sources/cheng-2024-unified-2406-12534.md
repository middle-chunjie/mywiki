## Summary

The paper argues that active retrieval in [[retrieval-augmented-generation]] cannot be handled well by a single trigger signal because real user inputs differ along intent, factuality, temporal volatility, and what the base [[large-language-model]] already knows. It proposes Unified Active Retrieval (UAR), which decomposes retrieval timing into four orthogonal binary decisions: intent-aware, knowledge-aware, time-aware, and self-aware. Each decision is made by a lightweight classifier over the frozen LLM's last-token hidden state, and the final retrieval choice is composed by a priority-ordered decision tree called UAR-Criteria. This design keeps generation unchanged, adds minimal latency, and yields stronger retrieval-timing accuracy and downstream QA performance than FLARE, Self-RAG, and SKR across AR-Bench and six evaluation datasets.

## Problem & Motivation

The paper studies when a RAG system should retrieve rather than assuming retrieval is always helpful. The authors argue that always retrieving can hurt creativity, reasoning, and latency on instructions that do not need external knowledge, while failing to retrieve harms factuality on time-sensitive or model-unknown questions. Prior active retrieval methods each rely on one criterion, such as task factuality or model uncertainty, which makes them brittle under mixed real-world instructions. UAR is motivated by the need for a unified decision procedure that handles explicit retrieval intent, knowledge intensity, temporal sensitivity, and model self-knowledge within one plug-and-play framework.

## Method

- **Core formulation**: UAR freezes the base LLM and trains four criterion-specific classifiers on the last-token hidden state `h_T`, where each classifier predicts a binary decision `c_i(h_T) in {0, 1}` for whether retrieval is needed under one criterion.
- **Four orthogonal criteria**: the framework evaluates `intent-aware`, `knowledge-aware`, `time-aware`, and `self-aware` signals rather than relying on a single heuristic. This turns heterogeneous retrieval decisions into a uniform set of binary classification tasks.
- **Classifier architecture**: each criterion uses a single-layer MLP with input size equal to the model hidden dimension and output dimension `2`. The design is explicitly plug-and-play and does not modify the LLM parameters.
- **Training data construction**: self-aware labels come from model-specific IDK data built from TriviaQA with `10` sampled answers per question; time-aware positives come from TAQA and negatives from TriviaQA; knowledge-aware data combines Self-RAG non-retrieval instructions with time-aware data; intent-aware data mixes `3,000` generated retrieval intents with `52,949` sampled user queries or factual questions.
- **Training hyperparameters**: UAR classifiers are trained with batch size `32`, learning rate `5e-5`, and `10` epochs, saving checkpoints after each epoch and selecting the best validation checkpoint.
- **Unified decision policy**: UAR-Criteria applies the classifiers in a priority order. It first checks retrieval intent, then whether the instruction is knowledge-intensive, then whether the knowledge is time-sensitive, and finally whether the model already knows the answer.
- **Generation path**: when retrieval is triggered, external documents are appended to the original input through a RAG prompt template, so the original instruction encoding can be reused instead of recomputed from scratch.
- **Retrieval setup in experiments**: for TAQA and FreshQA the system uses Google Search; for other tasks it uses Contriever-MS MARCO with up to `10` retrieved documents and feeds the top `5` passages into generation.

## Key Results

- On AR-Bench, UAR reaches `85.32` overall accuracy with Llama2-7B-chat and `86.33` with Llama2-13B-chat, beating the best baseline SKR by `23.18` and `23.63` points respectively.
- On 7B AR-Bench, UAR scores `91.88` intent-aware, `90.38` knowledge-aware, `86.69` time-aware, and `72.32` self-aware, substantially outperforming FLARE, Self-RAG, and SKR in every scenario.
- On downstream tasks, UAR achieves the best overall average: `50.49` for 7B and `53.26` for 13B, surpassing always-retrieve baselines (`49.23`, `52.09`) and active baselines such as FLARE (`48.50`, `48.90`) and SKR (`47.17`, `50.16`).
- For time-sensitive datasets, UAR retrieves almost always and performs best: on TAQA it reaches `34.46` with `99.7%` retrieval on 7B and `34.14` with `99.6%` retrieval on 13B; on FreshQA it reaches `59.74` and `55.45`.
- For non-knowledge-intensive tasks, UAR suppresses unnecessary retrieval: on GSM8K it uses `0.1%` retrieval for 7B and `0.0%` for 13B while matching the best non-retrieval result (`26.91` and `40.64`).
- The main bottleneck remains self-awareness: a standalone self-aware classifier gets `72.56` on 7B, while full UAR gets `72.32`; fine-tuning the whole LLM only raises self-aware accuracy to `75.65`/`76.28` at much higher cost.

## Limitations

- The experiments focus on short-form generation and a single retrieval call; the paper does not solve multi-step active retrieval inside longer responses.
- The four retrieval criteria are designed from practical experience and may miss additional scenarios that matter in deployment.
- Self-awareness remains the hardest criterion, and deeper or more expensive classifiers may improve it, but the paper does not explore that space beyond a simple whole-LLM comparison.
- The classifier family is limited to a single-layer MLP, so the trade-off between richer decision heads and efficiency is still open.
