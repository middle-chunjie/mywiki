## Summary

⚠ Possibly outdated: published 2023; re-verify against recent literature. CodeAttack is a black-box adversarial attack framework for sequence-to-sequence programming-language models that perturbs code in the natural channel rather than changing executable semantics directly. It ranks vulnerable tokens by the drop in victim-model logits after masking, then uses a masked CodeBERT generator plus code-specific filtering constraints to propose minimally edited substitutes. Across code translation, code repair, and code summarization, the method is more query-efficient and usually more successful than TextFooler and BERT-Attack while keeping adversarial code relatively fluent and consistent with the original snippet. The paper argues that current PL models over-rely on human-facing lexical cues such as identifiers, comments, and keywords, exposing a robustness gap in code generation systems.

## Problem & Motivation

The paper studies whether pre-trained programming-language models are robust to small, human-plausible perturbations in code's natural channel, such as identifier or operator edits that preserve much of the original readability. Existing adversarial NLP methods are poorly matched to code because they ignore syntax, code token classes, and coding style, while prior code attacks focus mostly on classification tasks rather than sequence-to-sequence generation. The authors therefore target a realistic black-box threat model for code translation, repair, and summarization, aiming to expose robustness failures with perturbations that remain minimal, consistent, and fluent to a human reader.

## Method

- **Threat model**: given input code `X`, adversarial code `X_adv` must satisfy `X != X_adv`, perturbation budget ``||delta|| < theta``, similarity constraint ``Sim(X_adv, X) >= epsilon``, and output degradation ``Q(F(X)) - Q(F(X_adv)) >= phi`` under black-box access to output probabilities only.
- **Vulnerable-token scoring**: for each token `x_i`, replace it with `[MASK]` and compute influence ``I_{x_i} = sum_t o_t - sum_t o'_t`` from the drop in correct-token logits. Tokens are ranked by `I_{x_i}` and the top-`k` candidates are attacked first.
- **Substitute generation**: use masked [[codebert]] as adversarial generator. For each vulnerable token, mask its aligned WordPiece span and take the top `50` predictions as the initial substitute set.
- **Code-specific filtering**: keep substitutes only if operator count changes by at most `1`, i.e. ``|Op(v_i)| - 1 <= |Op(s_i)| <= |Op(v_i)| + 1``, and if code token class is preserved, i.e. ``C(v_i) = C(s_i)`` with matched class cardinality.
- **Edit operations**: allow both operator-level character insertion/replacement/deletion and token-level substitution. The greedy search iteratively tests candidates and returns once the quality drop threshold is met.
- **Attack budget and similarity**: attack at most `40%` of code tokens and maintain cosine similarity threshold `0.5` between original and perturbed code.
- **Evaluation setup**: code translation on Java<->C#, code repair on the Tufano small Java dataset, and code summarization on CodeSearchNet Python/Java/PHP; victim models are [[codet5]], [[codebert]], [[graphcodebert]], and [[roberta]].
- **Implementation details**: implemented in PyTorch with batch size `256`, using a `48 GiB RTX 8000` GPU; effectiveness measured by `CodeBLEU` or `BLEU`, plus success rate, average queries, perturbation count, and adversarial consistency `CodeBLEU_q = CodeBLEU(X, X_adv)`.

## Key Results

- On C#->Java translation, CodeAttack raises success rate to `89.3%` on [[codet5]], `97.7%` on [[codebert]], and `98.0%` on [[graphcodebert]], with `Delta_drop` values `12.27`, `17.03`, and `27.99`, respectively.
- On Java->C# translation, CodeAttack reaches `94.8%`, `91.1%`, and `90.8%` success on CodeT5, CodeBERT, and GraphCodeBERT, outperforming TextFooler in all three cases and usually using only about `1.64-2.03` perturbations.
- On code repair, CodeAttack achieves near-saturated success: `99.36%` on CodeT5, `99.39%` on CodeBERT, and `99.52%` on GraphCodeBERT, while requiring only `24.67-30.68` queries on average versus `174.1-262.5` for BERT-Attack.
- On PHP summarization, CodeAttack reduces BLEU from `20.06` to `11.06` on CodeT5, from `19.76` to `10.88` on CodeBERT, and from `19.06` to `10.98` on RoBERTa, with success rates `82.8%`, `88.32%`, and `87.51%`.
- Human syntax checks on sampled adversarial code report average syntactic correctness of `70%` for C#, `60%` for Java, and `76.19%` for Python, higher than the compared NLP baselines.
- In ablations, attacking vulnerable tokens alone is weaker; adding operator constraints and token-level substitutions improves attack strength, and the paper notes especially large drops on GraphCodeBERT because identifier perturbations disrupt data-flow relations.

## Limitations

- The evaluation is limited to sequence-to-sequence tasks; robustness for code classification, retrieval, or agentic code execution is not studied here.
- The datasets consist of code snippets that are not compiled or executed, so semantic preservation is approximated through similarity and token-class constraints rather than verified by program behavior.
- Even though syntax quality is better than NLP baselines, human-checked correctness is still only `60-76.19%`, so many adversarial examples remain imperfect code artifacts.
- The method assumes access to victim-model output probabilities/logits; stricter API settings that expose only decoded text would make this attack harder.
- The paper diagnoses vulnerabilities but does not provide a concrete defense beyond suggesting stronger structural modeling or compile/execute-aware preprocessing.
