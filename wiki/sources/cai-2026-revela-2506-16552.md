## Summary

REVELA proposes a self-supervised dense retriever training framework that reframes retrieval as language modeling over document chunks rather than over query-document pairs. The key idea is to augment a decoder-only LM with in-batch attention so that next-token prediction for one sequence can condition not only on its own prefix but also on other sequences in the batch, weighted by retriever-computed similarities. This couples retriever learning and language modeling in a single objective and removes the need for annotated or synthetic retrieval pairs. Across code retrieval, reasoning-heavy retrieval, and general-domain retrieval, the method consistently beats prior unsupervised baselines, reaches parity with much larger supervised or API models in some settings, and scales favorably with retriever size, LM size, and batch size.

## Problem & Motivation

Dense retrievers are usually trained from curated query-document pairs plus hard negatives, but such supervision is expensive and scarce in specialized domains like code or in retrieval settings that require reasoning. The paper argues that language models already learn token-level dependencies through self-supervised next-token prediction, so retrievers should analogously learn dependencies among larger text chunks directly from raw corpora. REVELA is motivated by turning cross-sequence dependencies inside language modeling into a retrieval signal, avoiding handcrafted pair construction, structural bias from weak supervision pipelines, and the reindexing-heavy training pattern used by some LM-guided retriever-learning methods.

## Method

- **Objective shift**: classical next-token prediction uses `P(x_l^i) = P_\Phi(x_l^i | x_{<l}^i)`, while REVELA trains `P_R(x_l^i) = P_{\Phi,\Theta}(x_l^i | x_{<l}^i, {D_j}_{j \ne i})`, so each sequence can condition on other documents in the batch.
- **In-batch attention**: each layer keeps standard self-attention output `e_i^l` and an additional in-batch branch `h_i^l`; the branch computes self-context `s_i^l` plus cross-document attention `b_{ij}^l = softmax(Q_i^h K_j^{eT} / sqrt(d_H)) V_j^e`, then aggregates `b_i^l = \sum_{j \ne i} Sim(D_i, D_j) b_{ij}^l` and combines them as `h_i^l = s_i^l + b_i^l`.
- **Retriever-weighted similarities**: a retriever encoder produces document embeddings `h_i = E_\Theta(D_i)`, normalizes them, computes cosine scores `S_{ij} = \tilde{h}_i^T \tilde{h}_j`, and turns them into attention weights with temperature-scaled softmax `Sim(D_i, D_j) = exp(S_{ij} / \tau) / \sum_{k \ne i} exp(S_{ik} / \tau)`.
- **Minimal transformer modification**: instead of redesigning the entire block, the implementation duplicates documents and adjusts the attention mask so the in-batch branch can attend across sequences while remaining compatible with standard transformer code paths.
- **Domain-specific chunking**: code corpora are segmented into chunks of at most `120` words, batch size is usually `16`, and Wikipedia chunks are interleaved across documents before batching; for Wikipedia similarity, only the first half of a chunk is used, while code retrieval keeps the full chunk.
- **Backbones and co-training setup**: the LM is `LLaMA-3.2-1B`; retriever backbones range from `135M` to `3B`, with `8B` used for BRIGHT comparisons. Sentence representation follows EOS pooling with prefixes `Query:` and `Passage:`.
- **Optimization**: both retriever and LM use LoRA with rank `256`; training uses `\tau = 1e-4`, learning rate `1e-4`, `100` warmup steps, WarmupDecayLR, gradient accumulation `8`, `bf16`, passage truncation `160`, and `1` epoch on `4 x A100 80GB` GPUs. Wikipedia training runs `10,000` steps in about `44` hours; code training runs around `11,000` steps in about `48` hours.
- **Implementation detail for Wikipedia**: when trained on Wikipedia, cross-document attention uses V-normalization `\tilde{b}_{ij}^l = b_{ij}^l / (N_{ij} + \epsilon)` with `\epsilon = 1e-6` to prevent single-token domination; the code-domain setting performs better without this normalization.
- **Inference**: query and document max length is `2048`; the method is designed to learn retrievers from raw text only, without annotated or synthetic query-document pairs.

## Key Results

- On CoIR, `Revela-3B` reaches mean `60.1` NDCG@10, beating `E5-Mistral-7B-Instruct` at `57.3` by `2.8` points and edging out `Voyage-Code-002` at `56.3`.
- On CoIR at comparable scale, `Revela-0.5B` scores `56.1`, outperforming weakly supervised `E5-PT` (`46.4`) by `9.7` points; `Revela-0.1B` beats supervised `UniXCoder` (`37.3`) by `11.1` points with `48.4`.
- On BRIGHT, `Revela-8B` obtains mean `17.8` nDCG@10, essentially matching `E5-Mistral` (`17.9`), `OpenAI` (`17.9`), and `Voyage` (`17.9`), while exceeding `Cohere` (`16.6`); `Revela-0.5B` also beats `E5-PT` by `1.1` points (`12.5` vs. `11.4`).
- On BEIR, `Revela-3B` reaches mean `45.2`, nearly matching `E5-PT` (`45.6`) while using about `1000x` less training data and more than `10x` less compute, and it exceeds `REPLUG-3B` (`36.3`) by `8.9` points.
- Under the same `LLaMA-3.2-1B` backbone, `Revela-code1B` scores `49.1` average over BEIR and CoIR versus `42.2` for `Contriever-code1B`; `Revela-wiki1B` scores `48.0` versus `46.4` for `Contriever-wiki1B`.
- Larger batches help: on CoIR, `Revela-1B` rises from `55.9` at batch size `4` to `58.6` at batch size `16`; on BEIR, the same model rises from `41.7` to `42.7`.

## Limitations

- The method depends on batch construction heuristics such as chunking, interleaving, and domain-specific similarity computation; these choices may materially affect performance and are not fully theory-backed.
- REVELA is more efficient than REPLUG in asymptotic batch scaling, but it still adds an extra in-batch attention branch and benefits from larger batches, so memory and training cost remain meaningful practical constraints.
- The evaluation focuses on text and code retrieval; multimodal retrieval is only proposed as future work rather than validated experimentally.
- The paper does not solve iterative indexing or dynamic grouping of chunks by evolving representations, which the authors identify as a more general but computationally expensive direction.
- Some architectural details, especially V-normalization and other attention variants, are explored only lightly and appear domain-dependent.
